# 行业示例:流程制造实施速查

> 帮 OpenDeploy agent 在收到"我们是流程制造 / 化工 / 食品 / 医药 / 原料药"等线索时,
> 30 秒形成行业上下文:典型痛点、标准模块组合、自定义层级、合规红线。

## 1. 行业画像

- **典型客户规模**: 年营收 3000 万 - 100 亿;小企业(小于 3000 万)走精斗云 / 星辰(K/3 Cloud 的下沉版);100 亿以上通常要加 PLM + MES + LIMS + 实时数据采集
- **核心业务模式**:
  - **按预测生产**(食品 / 饮料 / 日化) — 品种有限,铺货导向
  - **按订单生产**(原料药 / 特种化学品 / 定制配方) — 批次驱动
  - **连续 / 间歇混合**(大化工连续反应 + 分装间歇) — 同一客户两种并存
- **K/3 Cloud 适配度**:
  - ✅ 批号管理 / 保质期 / 联副产品 / 倒冲 / 质检追溯是**原生标准**
  - ⚠️ **配方 vs BOM 的表达力** — K/3 Cloud 用"物料清单"同时表达离散 BOM 和流程配方,通过"子项用量分子分母 + 用量类型"承载配方按比例的表达;极端场景(如动态配方按实时温度调比例)标准不够,要二开
  - ❌ **DCS / SCADA 实时数据** — 需要通过应用集成平台对接,K/3 Cloud 自身不是 MES
- **常见 K/3 Cloud 版次**:
  - 标准版 / 企业版基础功能覆盖批号 + 保质期 + 联副产品
  - **制造云 / 星瀚** 有更深的配方 + 成本模块
  - 化工 / 医药客户通常需要 **PLM 云 + 制造云** 组合

## 2. 典型业务场景与痛点(客户原话风格)

多数流程制造客户会提以下 8-12 条里的某几条:

- "配方严格保密,只有技术部 3 个人能看比例"
- "每批产品要能追溯到原料批号,FDA / GMP 审查要用"
- "一批料投下去出主产品 + 副产品 + 边角料,收率要单独算"
- "保质期快到的产品要先发出去,FIFO / FEFO 自动排"
- "临期产品每月要预警给销售,别放坏了"
- "退货的要按批号原路退回,别和新货混仓"
- "投料是按配方比例,但实际投下去总有偏差,实际收率和理论收率要对比"
- "原料药 GMP 规定每批要留样,留样账和现货账要分开"
- "反应釜清洗要记录,下一批不能带前一批残留"
- "客户要品质追溯证书,按销售批号查到原料供应商 + 检验记录"
- "外购包材不走批号,自产半成品走批号,规则不同别搞错"
- "食品标签上的保质期要按生产日期 + 保质期天数自动算"

**识别信号**: 出现**配方 / 批号 / 保质期 / 反应釜 / 联产品 / 副产品 / 收率 / 反冲 / 批次追溯 / GMP / GSP / FDA** 关键词 → 流程制造。混合型(装配 + 灌装包装)参考 `industry-manufacturing-discrete.md` 的离散部分 + 本文的批次保质期部分。

## 3. 标准模块组合(必启)

按启用顺序:

| 顺序 | 模块 | 用途 | 实施深度 |
|---|---|---|---|
| 1 | 基础资料 → 物料 | **启批号管理 + 保质期管理**(三个维度:批号 / 生产日期 / 有效期至) | 必启 |
| 2 | 基础资料 → 物料清单(作为配方载体,子项**用量分子 / 用量分母**表达比例,**联副产品**子项类型) | 配方 + 联副产品核心 | 必启,占实施工时 20-30% |
| 3 | 库存管理 → **批号主档**(`T_BD_LOTMASTER`)+ **入库 / 出库 / 库存** | 批号 + 有效期全流程载体 | 必启 |
| 4 | 生产制造 → 生产订单 + **倒冲领料**(按完工数量自动扣料,连续流程标配) | 流程制造核心 | 多数启 |
| 5 | 生产制造 → 工序汇报(合格 / 工废 / 料废 + **联副产品数量**) | 收率统计基础 | 必启 |
| 6 | 质量管理 → **检验方案 + 来料检 + 过程检 + 出货检** | GMP / GSP 合规必做 | 医药 / 食品必启 |
| 7 | 销售 + 库存 → **批次追溯查询**(溯源追溯 / 去向追溯) | 客户品质证书出具基础 | 多数启 |
| 8 | 成本管理 → **联产品 / 副产品成本分摊**(售价法 / 能量法 / 等价系数) | 财务闭环 | 期末启用 |
| 9 | 仓库管理 → **保质期预警 / 临期管理** | 食品 / 医药必做 | 多数启 |

**启用顺序坑**: 批号管理**必须物料创建时启用**,后续想切换要换物料代码(见 CSDN 博客实证,`T_BD_LOTMASTER` 表结构不可逆)。**项目启动前物料主数据整理**是流程制造项目的头号工作量。

## 4. 高频自定义层级(K/3 Cloud 标准搞不定的)

按"标准 / BOS / Python / DLL"四层分类。**流程制造比离散制造对标准模块依赖度更高**——批号 / 保质期 / 联副产品 / 质检 / 倒冲都是原生标准,**先反查 `product-features-index/references/mfg-manufacturing.md` 字典**再谈二开。

### 4.1 标准搞不定但 BOS 配置能解决

- **配方版本 + 审批流** — BOS 审批流 + BOM 标准多版本组合
- **批号命名规则**(如"生产日期 + 产线代码 + 流水号") — 基础资料→编码规则配置
- **质检表单扩展**(pH / 水分 / 微生物 / HPLC 结果) — BOS 给检验方案 / 检验单加扩展字段
- **保密配方隐藏字段**(给物流 / 采购部门隐藏比例) — BOS 数据权限 + 字段显隐规则
- **客户品质证书(COA)自定义模板** — BOS 套打模板
- **留样记录单** — BOS 新对象或复制检验单为子类

### 4.2 要 Python 表单插件

真的需要写插件的典型场景:

- **按客户订单自动选批次** — FIFO / FEFO 算法 + 客户留货规则(如"X 客户只要近 6 个月生产的"),BOS 下推规则不够
- **投料偏差校验** — 生产领料时按实际重量对比配方理论值,超差阻断
- **实际收率回算 + 红冲** — 工序汇报提交时按联副产品数量和投料重量反算收率,写入扩展字段;收率异常时提示
- **批号切换时的清洗记录强校验** — 某些 GMP 场景要求下一批生产单审核前必须有前一批的清洗记录
- **保质期预警批处理**(配合后台任务) — 批量扫库存按 N 天阈值生成待处理清单

### 4.3 要 DLL 操作插件 / 外部集成(参见 `dll-plugin-index`,v0.1 不自动化)

- **DCS / SCADA 实时数据接入** — 反应釜温度 / 压力 / 流量实时写回 ERP,需要常驻服务
- **LIMS 实验室数据双向同步** — 检验数据 + 结果回写,频率高
- **条码 / RFID 自动入库** — 生产线扫码实时入库,批号 + 数量自动生成
- **医药追溯码平台对接**(国家药品追溯系统) — 电子监管码 / 追溯码上传,有行政法规约束

## 5. 实施常见坑(行业特有)

1. **批号管理的启用不可逆** — 物料创建时未启批号,想后启必须换代码 + 转移库存;**项目立项时物料主数据策略必须和客户定死,别半路改**
2. **保质期管理维度混淆** — 批号 / 生产日期 / 有效期至是三个独立维度,启用后库存数据按组合存,导入历史数据要严格对齐;漏填会让库存查询错位
3. **配方用 BOM 表达的精度** — 用量分子 / 用量分母能表达整数比例,极小数(如催化剂 0.001%)要看物料基本单位精度
4. **联副产品成本分摊算法选错** — 售价法 / 能量法 / 等价系数三种,选了之后期末核算逻辑完全不同;**财务部门初期必须参与决策**
5. **FIFO / FEFO 执行不到位** — 系统推荐了批号但仓库实际发货没按系统指示(人为干预) → 追溯链断;需要流程强制 + 培训
6. **GMP 留样账和现货账分开没做好** — 留样必须独立仓库 + 独立批号规则,混在现货仓里抽检时算库存会错
7. **反冲料仓位设置错误** — 车间 WIP 仓 vs 物料清单默认仓,选错了倒冲领料会在错仓位扣库,实物和账不符
8. **多组织下批号的全局唯一性** — 默认批号在组织内唯一,跨组织调拨时如果未启用"全局批号"可能冲突,跨工厂追溯会断链
9. **DCS 对接假设客户能给实时数据** — 老工厂 DCS 可能只给分钟级批量数据,不是实时 TCP;实施前必须跟客户 IT 确认对接协议
10. **医药行业 GMP / GSP 合规范围** — K/3 Cloud 能承载批号追溯 / 检验 / 留样流程,**但 GMP 认证本身是客户组织的责任**,系统只是工具;别承诺"上了系统就 GMP 合规"

## 6. 示范 plan 链接

典型流程制造的组合需求:
- **待补**: "批号 FEFO 自动发货 + 临期预警" — 食品 / 医药典型 3-4 步组合,标准配置 + Python 自动选批号插件
- **待补**: "配方保密隔离 + 审批流" — 化工典型 2-3 步,BOS 数据权限 + 审批流配置

## 临时话术

> "你们是流程制造客户,典型实施切入点是 **批号管理 + 保质期 + 配方(BOM)+ 质检追溯**。
>
> 流程制造对标准模块的依赖比离散制造高 — 多数客户 80%+ 需求靠标准配置 + BOS,**真要写代码通常集中在 FIFO / FEFO 自动选批、投料偏差校验、实际收率回算** 这几个点。
>
> 在建议方案前,需要你确认四件事:
> 1. **细分行业**(化工 / 食品 / 医药 / 原料药 / 日化)—— 合规要求差异大,医药 / 原料药要考虑 GMP,食品要考虑保质期预警
> 2. **批号规则** —— 是否启批号、生产日期、有效期至;**必须在物料建档前定好**,改不了
> 3. **K/3 Cloud 版次 + 是否已采购 PLM 云 / 制造云** —— 配方保密 / 设计变更需要 PLM,深度成本需要制造云
> 4. **DCS / MES / LIMS 是否需要对接** —— 这部分 v0.1 OpenDeploy 不直接做,需要告诉你找集成伙伴
>
> 确认后我可以按 `mfg-manufacturing.md` 字典把标准功能逐一对号入座,把真要二开的挑出来独立 plan。"
