# 行业示例:服务业 / 工程项目 实施速查 🟡

> 帮 OpenDeploy agent 在收到 "我们是 [设计 / 咨询 / 工程 / IT 服务 / 安装 / 系统集成] 类客户" 或 "我们按项目核算" / "一单一项目" 等线索时形成行业上下文。
>
> **行业覆盖**:工程施工(建安 / 装饰 / 弱电 / 市政)、咨询(管理 / 财税 / 法律)、IT 服务(系统集成 / 外包开发 / 运维)、设计(建筑 / 工业 / 广告)、安装调试(设备 / 空调 / 消防)、专业服务(审计 / 测绘 / 教育)。
>
> 重点:服务业 / 项目类客户**没有标准库存或标准生产**,核心是**按项目核算**——收入 / 成本 / 费用 / 毛利全部落项目维度。

---

## 1. 行业画像

- **典型客户规模**:中小型 30-500 人居多;员工工时 = 主成本 ≥ 60%;年营收 500 万-5 亿
- **业务模式**:一单一项目(签合同即立项目),项目生命周期 1-24 个月,单项目规模几万到几千万
- **K/3 Cloud 适配度**:
  - ✅ **小-中型项目核算(单项目 < 2000 万)**——辅助核算 + 核算项目 + 项目云 即可
  - 🟡 **大型工程项目(单项目 > 5000 万,长周期,多层分包)**——K/3 Cloud 项目能力偏"核算 / 预算监控"方向,**专业 PMS (建筑 / 工程项目管理)**(广联达 / 鲁班 / 新中大)在进度计划 / 工程量 / 变更签证 / 结算单的深度上更强,通常要走**集成方案**(PMS 管业务 + K/3 Cloud 管财务)
  - ❌ **超大型 EPC / 建筑央企**——一般直接走苍穹或行业专用 ERP,K/3 Cloud V9 难覆盖
- **常见版次**:
  - **旗舰版** 带"项目云"(项目 WBS + 甘特图 + 项目预算 + 项目分析),服务型项目推荐
  - **标准版 / 专业版** 靠**辅助核算 + 核算项目 + 项目档案基础资料** 做最小化项目核算,适合咨询 / IT 服务类"轻项目"
  - **工程项目行业方案**——基于 BOS 的行业定制包,大工程项目企业要问金蝶原厂
- **第一句话诚实提醒**:K/3 Cloud 能不能做你这个项目,取决于**项目规模 + 是否要工程量计算 + 是否要分包管理**——不是所有 "按项目算" 都一个方案

## 2. 典型业务场景与痛点(客户原话)

1. **"我们要按项目算成本"** — 最高频诉求,本质是启用辅助核算 + 核算项目维度
2. **"工时怎么记到项目里"** — 员工日 / 周报工时,归集到项目;数据来源可能 OA / HR / 钉钉 / 人工登记
3. **"项目收入按进度确认"** — 完工百分比法 vs 完工合同法 vs 里程碑法,选型看合同条款 + 会计政策
4. **"材料领用要绑项目"** — 领料单必带项目字段;出库成本落到项目
5. **"项目预算超支预警"** — 合同预算 ≈ 项目天花板,实际成本接近预算要提醒 / 审批 / 拦截
6. **"客户分期付款,我们要按节点开票"** — 合同里程碑 → 开票计划 → 收款跟进
7. **"项目结项 / 归档 / 转 PMC"** — 完工验收 + 结算单 + 决算分析 + 禁用项目
8. **"项目变更签证怎么算进预算"** — 甲方追加工作 / 减项,需要变更单修正预算基数
9. **"外协 / 分包款怎么绑项目"** — 外协合同 + 进度款 + 质保金管理
10. **"一张销售单含多项目怎么办"** — 表体每一行挂不同项目,分摊毛利
11. **"人工成本怎么摊进项目"** — 工资不直接按项目发,要按**工时 × 费率**归集
12. **"报表要按项目看"** — 项目损益表 / 项目成本汇总 / 项目应收账龄 / 项目现金流

## 3. 标准模块组合(必启)

核心一句话:**辅助核算 + 核算项目维度 是服务业 / 项目类客户的命脉**。任何写代码前先确认这两项已经配对。

| 模块 | 用途 | 启用顺序 |
|---|---|---|
| 基础资料 → **核算项目** / **项目档案** | 项目维度字典(所有业务单据共用) | 🟢 第 1 优先级,建账前必定 |
| 总账 → **辅助核算**(项目维度挂科目) | 凭证 / 账簿 / 报表按项目透视 | 🟢 第 1 优先级,和上面一起 |
| 应收 / 应付 → 单据表体 **项目字段** | 开票 / 收款 / 付款全部带项目 | 🟢 启用初期必做 |
| **销售管理**(部分启用) | 销售合同 + 销售订单 + 开票申请,单据带项目字段 | 第 2 步 |
| **采购管理** + 仓存(极简用法) | 采购合同 + 采购订单 + 入库 / 领料带项目 | 第 2 步 |
| **费用报销** / 费用管理 | 差旅 / 报销单带项目维度 | 第 2-3 步 |
| **合同管理**(旗舰版 / 行业版) | 销售合同 / 采购合同 / 外协合同统一管理 | 第 3 步 |
| **项目云 / 项目管理**(旗舰版) | 项目 WBS / 甘特图 / 进度 / 项目预算 / 里程碑 | 第 3 步,有就用,没有靠辅助核算也能跑 |
| **智能会计平台** + 凭证模板 | 业务单据 → 凭证(项目维度自动传递) | 第 4 步,AR/AP 开始立账后 |
| **报表**(总账账龄 + 账表设计器) | 项目损益 / 项目成本汇总表 | 最后 |

> **注意**:大部分客户认为"项目管理"要买项目云,其实**只要辅助核算 + 核算项目 + 销售 / 采购 / 费用单据表体的项目字段标配**,就能跑最小化的项目核算闭环;项目云是"加分项",不是必需。

## 4. 高频自定义层级(标准 → BOS → Python → DLL)

### 标准配置层(80% 需求落在这里)

- **辅助核算挂科目** — 主营业务收入 / 主营业务成本 / 管理费用 / 应收应付科目全部挂"项目"维度
- **凭证模板** — 业务单据的项目字段自动传递到凭证的辅助核算,不需插件
- **编码规则** — 项目编码按年份 + 客户缩写 + 流水自动生成
- **预算控制方案** — 费用申请 / 报销 / 采购申请按项目预算拦截超支
- **审批流** — 项目立项 / 变更 / 结项走工作流

### BOS 配置层(扩展字段 + 过滤)

服务业典型场景:
- **项目档案扩展字段**:项目阶段(立项 / 执行 / 收尾 / 归档) / 项目类型(咨询 / 实施 / 运维) / 客户经理 / 实施顾问 / 行业分类 / 合同金额 / 开工 / 竣工日期 — BOS 直接加
- **销售订单扩展**:项目里程碑(首付 / 中期 / 尾款) / 开票计划
- **费用申请扩展**:外勤 / 驻场标记 / 客户签字附件
- **过滤方案** — 销售员只看自己负责的项目

### Python 插件层(BOS 做不到的校验 / 联动)

常见:
- **项目预算超支硬校验** — 费用申请 / 采购申请 BeforeSave 累加同项目实际 + 在途,超预算拦截(标准预算控制方案不够精细时补)
- **工时录入校验** — 每日工时 ≤ 8 小时 / 同人同日同项目唯一 / 非活动项目禁录
- **项目结项联动** — 项目状态改 "归档" 时反查未核销应收 / 未完工工时,有就拦
- **签证自动挂项目** — 从基础资料带出项目经理 / 默认审批人

### DLL 插件层(v0.1 不支持,需外部二开)

仅在以下场景考虑:
- **复杂收入确认算法** — 多阶段加权完工百分比 + 汇率折算 + 回调冲销
- **工程量结算自动化** — 从 CAD / BIM 数据源拉工程量明细拼结算单
- **多项目资金池调度** — 集团资金调剂跨项目算利息

## 5. 实施常见坑

### 5.1 辅助核算未启用 → 后期补无法重算历史

- **症状**:实施中期客户要求"按项目看成本",翻凭证发现项目维度全空
- **根因**:启用账簿时没勾"项目"维度到相关科目
- **对策**:**启用前必须设计辅助核算方案**——哪些科目挂项目 / 挂部门 / 挂客户。历史已录凭证**只能从启用期重新做**,旧凭证回填成本极高
- **来源**:[辅助核算 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E7%A7%91%E7%9B%AE) · fetched 2026-04-23

### 5.2 项目档案分类设计错误 → 报表透视困难

- **症状**:半年后客户要 "按行业看项目毛利",发现项目档案没行业字段,只能手工分类
- **根因**:项目档案基础资料只有编码 + 名称,未做行业 / 类型 / 客户经理分组
- **对策**:**立项前就把 BOS 扩展字段规划好**——常用维度:项目阶段 / 项目类型 / 客户经理 / 行业 / 结算方式 / 预计毛利率
- 典型误判:用户以为"行业字段加字段"要写插件 → **BOS 扩展字段就能搞**

### 5.3 工时数据来源不清(自录 vs OA / HR 集成)

- **症状**:员工抱怨"每天登两次,K/3 Cloud 也登 OA 也登"
- **根因**:没规划工时数据源——K/3 Cloud 项目云支持工时归集,但**员工自录门槛高**;客户有 OA / 钉钉 / 企微 的话**先集成**再说
- **对策**:
  - 员工少(< 30 人) → K/3 Cloud 项目云 "工时汇报" 单据自录
  - 员工多 / 有 OA → **工时走 OA,日终 / 周终批量推送到 K/3 Cloud 费用分摊**(集成方案)
  - 钉钉 / 企微钟点工 → 第三方 iPaaS(每刻 / 轻易云)推送
- **来源**:[服务型项目解决方案 · zhuanlan.zhihu.com](https://zhuanlan.zhihu.com/p/1900953050824307843) · fetched 2026-04-23

### 5.4 项目收入确认时点的会计政策选择

- **症状**:IPO 审计问"你们怎么算新准则收入?",顾问不知所措
- **根因**:项目收入 5 种常见确认法——**按合同条款里的"履约义务完成"判**:
  - **完工百分比**(累计投入 / 工时 / 进度)——**大型项目最常用**,符合新收入准则
  - **完工合同法**(老准则,新准则极少允许)
  - **里程碑法**(分阶段验收,每段独立确认)——**IT 实施 / 咨询最常见**
  - **发货 / 到货确认**——设备安装类
  - **到期应收**(租赁性质)——运维 / SaaS
- **对策**:**立项前让客户财务 + 审计协商确认方法**,不要等半年后才定;对应配**凭证模板 + 收入凭证自动化方案**
- **来源**:[项目四算管理 · kingdee.com](https://www.kingdee.com/article/1834458907595051009.html) · fetched 2026-04-23

### 5.5 多项目串单据(一张销售单含多项目)

- **症状**:一张集成项目销售订单里,硬件 + 软件 + 服务要拆 3 个子项目核算
- **根因**:客户想一张单开 + 一次收款,但成本要分摊
- **对策**:
  - **单据表体每行挂不同项目字段**(标准支持)——推荐
  - 不要强拆成 3 张销售订单(客户签字痛苦)
  - 凭证模板按表体项目字段汇总借贷分录即可

### 5.6 项目变更签证未闭环

- **症状**:甲方追加工作两个月后才想起来补变更单,项目预算早超支
- **根因**:没设"变更必须走变更单 → 审批 → 更新合同预算基数"的硬流程
- **对策**:**变更单 BOS 配置审批流 + 预算刷新**;Python 辅助:变更单审核时自动刷新项目预算基数
- **前置**:客户有项目经理 + 商务 **双岗分工**;一人兼两岗时变更常被漏

### 5.7 外协分包款的挂项目

- **症状**:外协发票来了,采购会计找不到对应项目
- **根因**:外协订单 / 入库单没强制挂项目
- **对策**:**采购订单表体项目字段设"必录"**;付款单自动带出对应项目;应付账龄按项目维度看

### 5.8 项目归档后误录单据

- **症状**:去年结项的项目今年还能开费用报销
- **根因**:项目档案只有"启用 / 禁用",没有"归档态"
- **对策**:BOS 扩展字段 `F_ProjectStatus`(立项 / 执行 / 收尾 / 归档) + Python 插件 BeforeSave 拦截"归档态项目不让选"

## 临时话术

> **"我们是服务业 / 工程项目类客户,要按项目算账"**
>
> "K/3 Cloud 做项目核算的基础能力很完整,但**不是所有项目需求一套方案**——要先问几个问题:
>
> 1. **单项目规模多大?** 小项目(< 500 万,IT 服务 / 咨询)辅助核算 + 核算项目 就够;大项目(> 5000 万工程类)可能要 PMS 集成
> 2. **要不要工时管理?** 要的话先决定工时来源——K/3 Cloud 自录 / OA 集成 / 钉钉推送
> 3. **收入怎么确认?** 完工百分比 / 里程碑 / 发货 — 选错的话后期审计很痛
> 4. **工程量 / 变更签证 / 分包有吗?** 有的话要上项目云或行业方案,辅助核算不够用
>
> 回答这四个问题后,我再给具体的启用路径和 plan。**顺序永远是:辅助核算 + 核算项目 → 业务单据带项目字段 → 凭证模板 → 报表**,先把根基做对。"

## 已知不确定(明确标注)

- **项目云 V9 版本实际可用性**——"项目云"是旗舰版模块,标准版 / 专业版不含;具体功能点(WBS / 甘特图 / 预算控制)**未在客户环境实测**;实施前要确认客户采购版次
- **工时单据标准字段**——不同版次下的"工时汇报单"字段有差异,实施时反查手册
- **BOS 行业方案**(工程项目行业方案)——金蝶原厂包,**未接触过实物**,需要向金蝶索取 ISV 方案详情
- **与 PMS 集成**(广联达 / 鲁班)——已知有集成案例,但接口形态 / 数据点 **未实证**,是客户选型时才展开的专项
- **新收入准则凭证模板**——"下载凭证云模板" 有服务业 / 建筑业模板,但具体模板字段未实测

> 遇到以上场景,不得编造菜单路径或字段名,明示用户"这块需要现场实证 + 反查金蝶原厂手册"。
