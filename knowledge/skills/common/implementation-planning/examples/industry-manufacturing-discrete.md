# 行业示例:离散制造实施速查

> 帮 OpenDeploy agent 在收到"我们是离散制造客户"或类似线索(装配、机械、电子、家电、玩具、家具、汽配)时,
> 30 秒形成行业上下文:典型痛点、标准模块组合、自定义层级、风险点。

## 1. 行业画像

- **典型客户规模**: 年营收 5000 万 - 50 亿,员工 100 - 5000 人,1-10 个工厂;小于这个量级的客户通常走标准产品配置能搞定,大于这个量级可能需要 K/3 Cloud 旗舰版 + 智造云 MES + APS 组合
- **核心业务模式**:
  - **MTS(备货生产)** — 家电 / 玩具 / 标品电子,按预测铺货
  - **MTO(按单生产)** — 非标机械 / 电子 OEM,客户下单才投产
  - **ETO(按单设计)** — 装备制造 / 定制家具,订单驱动设计→工艺→生产
  - 多数客户是**混合模式**(标品 MTS + 大客户 MTO),别默认单一模式
- **K/3 Cloud 适配度**: ✅ 原生匹配 — MFG 模块是金蝶覆盖最深的之一,BOM / 工艺路线 / 工序汇报 / 委外 / MRP 全闭环
- **常见 K/3 Cloud 版次**:
  - 标准版够中小装配类客户(功能全但并发有限)
  - 企业版 / 旗舰版加多组织 + 多视图 BOM + APS 排产
  - 智造云 MES / PDA 报工要**单独购买**,不在基础 MFG 模块里 — 实施前必问客户是否已采购

## 2. 典型业务场景与痛点(客户原话风格)

多数离散制造客户会提以下 8-12 条里的某几条:

- "同一个产品客户定颜色 / 尺寸 / 配件,BOM 能不能一码管多版本"
- "销售订单直接下推生产订单,别让计划员手动录"
- "A 料没了自动用 B,别让 PMC 去现场问仓库"
- "工程师改了 BOM,在制订单要不要跟着变,怎么处理"
- "一张生产订单太大,要分几批投产"
- "工序 A 跟工序 B 能不能并行,别串行等着"
- "现场工人用 PDA 扫码报工,别再填纸单"
- "电镀 / 表面处理 / SMT 这道工序外协,别人做完回来继续"
- "开工后字段不让改,怕财务结账前被人偷改数量"
- "返工流程怎么走,返修数量要单独统计"
- "成本要精确到订单 / 批次,别按月平摊"
- "多工厂 / 多车间协同,一个物料在不同厂代码不一样怎么办"

**识别信号**: 出现 BOM / 工艺 / 工序 / 装配 / 车间 / 工人 / 报工 / 外协关键词 → 离散制造。出现**配方 / 批号 / 保质期 / 反应釜 / 联产品 / 收率** 关键词 → 流程制造(见 `industry-manufacturing-process.md`)。混合型(如食品包装 + 自动化装配线)需具体问。

## 3. 标准模块组合(必启)

按启用顺序:

| 顺序 | 模块 | 用途 | 实施深度 |
|---|---|---|---|
| 1 | 基础资料 → 物料 / 客户 / 供应商 | 主数据底座 | 必启 |
| 2 | 基础资料 → **物料清单**(BOM) | 工程数据核心,多版本 + 子项类型 + 用量分子分母 | 必启,占实施工时 15-25% |
| 3 | 工程数据 → **工艺路线** | 工序 + 工作中心 + 6 个基本活动(工时定额) | 必启 |
| 4 | 生产制造 → **生产订单** | 四种来源(手工/MRP/MPS/拆分)+ 状态机 | 必启 |
| 5 | 生产制造 → **工序汇报** | 合格 / 工废 / 料废 / 损耗字段齐全 | 必启 |
| 6 | 生产制造 → **生产领料 / 倒冲**(车间管理下,具体菜单按版本实证) | 按完工数量自动扣料 | 多数启用 |
| 7 | 计划管理 → **MRP 运算** | 净需求计算 + 计划订单转生产单/采购单 | 看客户意愿,部分客户初期仍手工 |
| 8 | 委外管理 / 工序委外 | 整单委外 vs 单工序外协(两种独立流程) | 有外协业务的启 |
| 9 | 成本管理 → 生产成本核算 | 按订单 / 批次归集,实际成本 vs 标准成本 | 财务闭环期启 |

**别踩的顺序坑**: 先上 BOM 和工艺路线,再上生产订单。若 BOM 还没理清直接开生产订单,基础资料会反复重建,返工严重。

## 4. 高频自定义层级(K/3 Cloud 标准搞不定的)

按"标准 / BOS / Python / DLL"四层分类。**注意**: 很多"以为要二开"的场景其实标准就有,先反查 `product-features-index/references/mfg-manufacturing.md` 字典再谈二开。

### 4.1 标准搞不定但 BOS 配置能解决(多数场景)

- **客户定制化字段**(颜色 / 尺寸 / 特殊标识 / QA 等级) → BOM / 生产订单加扩展字段,别新建对象
- **工程变更单审批流** → BOS 审批流(系统参数里开审批流,不用代码)
- **按订单类型限制可见字段** → BOS 表单控件配置(按单据类型显隐字段)
- **车间 / 工厂维度的数据权限** → BOS 数据规则 + 标准多组织
- **报表 / 套打格式调整**(不同客户要不同送货单格式) → 套打模板配置

### 4.2 要 Python 表单插件

以下是**真的需要写插件**的场景(通常不超过 3-5 个需求):

- **BeforeSave 级联校验** — 多字段交叉校验(如"订单数量必须是 BOM 基本批量的整数倍",单字段 BOS 搞不定)
- **生产订单下达前检查齐套** — 扫当前库存 + 在途采购 + 已下达生产用料,齐套率 <N% 阻断下达
- **ECN 工程变更时回写在制订单** — 变更审批通过后,按规则回写下达中的生产订单 BOM 引用版本
- **自动按客户规则生成物料编码** — 客户定制化场景的"一物一码"自动编码(家具行业典型需求)
- **工序汇报数量的二次加工**(如"按合格率自动回算工时奖金") — 字段计算公式 BOS 搞不定时

### 4.3 要 DLL 操作插件(参见 `dll-plugin-index`,v0.1 不自动化)

- **APS 排产集成 / 外部 MES 实时数据推送** — 需要后台常驻服务或高频 TCP 通信
- **复杂成本分摊算法**(联产品 / 副产品 + 加权平均) — 财务期末跑批的逻辑太重
- **PLM 双向同步** — 设计 BOM ↔ 制造 BOM,需要队列 / 调度机制

## 5. 实施常见坑(行业特有)

1. **BOM 多版本启用后不能回头** — 启用"按 BOM 版本扣料"策略后,历史订单已引用版本 1,后期想切"按日期扣料"会让历史数据错位;**启用前全员对齐一次**
2. **工艺路线的"基本批量"没配** → 工时按订单数量线性放大会算错,标准 6 个活动(准备 / 机加 / 移动 / 等待 / 排队 / 拆卸)各有独立定额 + 基本批量折算
3. **生产订单单据类型没拆** — 自制单 / 返修单 / 试产单混用一个单据类型,后续要分别管控规则时要改流水数据,很痛;**项目初期就按业务类型拆好**
4. **替代料用 Python 做而不是 BOM 配置** — 见 mfg 字典"替代料"条目,BOM `子项类型=替代件` + 替代策略(混用 / 整批 / 整批+混用)原生支持,写插件是重复造轮子
5. **工序委外做成两次整单委外** — 工序委外是独立模块,有 5 种转移类型 + 末序入库点配置,拼两次整单会少掉质量字段 + 结算字段
6. **PDA 报工没确认客户买了智慧车间 MES** — 标准 MFG 模块工序汇报是 Web 界面,PDA 是智慧车间套件(独立 License);**项目立项时必须问清**
7. **开工后不让改字段误用 BOS 锁字段** — 状态机(计划→下达→开工→完工→结案)已经控制了大部分操作权限,先看默认行为再决定是否叠加 BOS
8. **拆单后的子订单成本口径** — 拆分行后各子订单独立归集还是共享母单成本,财务部门需要提前定口径

## 6. 示范 plan 链接

典型离散制造的组合需求多数是"启用标准 + 加扩展字段 + Python 校验 + 培训"四步:
- 信用管控类(销售/生产通用): `examples/credit-management.md` 已有完整 plan
- **待补**: "定制化 BOM 自动编码 + 一码多 BOM" — 家具 / 玩具行业典型 3-4 步组合,可参照 `examples/credit-management.md` 的丰满度按需起草
- **待补**: "工序委外末序直接入库" — 电镀 / 涂装 / SMT 场景,2-3 步标准配置组合

## 临时话术

> "你们是离散制造客户(装配型),典型实施切入点是 **BOM(物料清单) + 工艺路线 + 生产订单 + 工序汇报** 四个核心模块。
>
> 多数客户 70-80% 需求标准配置能覆盖,真的需要写 Python 插件的通常 3-5 个(齐套检查 / ECN 回写 / 自动编码等)。
>
> 在建议方案前,需要你确认三件事:
> 1. **业务模式**(MTS 备货 / MTO 按单 / ETO 按单设计)—— 影响 BOM 设计策略
> 2. **K/3 Cloud 版次**(标准 / 企业 / 旗舰)+ 是否已采购**智造云 MES** —— 影响 PDA 报工 / APS 排产能不能谈
> 3. **工厂 / 车间数量** + 是否多组织 —— 影响多组织权限 / 预留网络配置
>
> 确认后我可以按 `mfg-manufacturing.md` 字典把标准功能逐一对号入座,把真要二开的挑出来独立 plan。"
