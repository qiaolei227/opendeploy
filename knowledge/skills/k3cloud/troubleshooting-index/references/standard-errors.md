# K/3 Cloud 标准功能报错诊断 🟡

<!-- 来源:help.open.kingdee.com / vip.kingdee.com / m.heshuyun.com / blog.csdn.net 公开内容综合;实证状态:🟡 常见场景索引,具体解决方案以客户环境为准 -->

覆盖登录 / 账套 / 启用模块 / 关账 / 业务参数 / 单据类型启用这类"**系统没跑起来 / 跑不到下一步**"的症状。**不覆盖**数据不一致(看 `data-inconsistency`)、不覆盖具体操作被拦截(看 `operation-failures`)。

**使用纪律**:每个场景的"解决方向"段都要让 agent 走 `common/systematic-debugging` 五步——先钉症状,再形成假设,再查证据,再锁根因。不要跳步。

## 1. 登录失败

### 场景 1.1:输入账号密码后无反应或提示失败

- **症状**:客户端输入用户名 / 密码 / 账套后,点"登录"无响应或弹"登录失败"
- **常见原因**:
  1. 服务端 IIS 应用池停了(最常见,**先排除基础设施**)
  2. 数据中心未启动或 `K3CloudManager` 服务异常
  3. 账号被禁用 / 密码过期 / 密码输错超限被锁
  4. IP 白名单限制(客户启用了协同开发平台或安全组的 IP 限制策略)
  5. 客户端版本与服务端补丁不匹配(客户端自动更新失败)
- **诊断步骤**:
  1. 让用户在服务器上访问 `http://localhost/K3Cloud/Kingdee.BOS.ServiceFacade.ServicesStub.Account.AccountService.ValidateUser.common.kdsvc`(仅诊断用,看 IIS 是否响应)——无响应 = 应用池 / 服务问题
  2. 让用户检查 **K3Cloud 管理中心**(http://服务器/K3Cloud/Manager/)是否能打开
  3. 让用户用**另一个管理员账号**试登,排除账号级别问题
  4. 查客户端日志:`%USERPROFILE%\Documents\Kingdee\K3Cloud\Log`,搜 `Exception`
- **解决方向**:
  - 服务问题 → 联系客户 IT 重启 IIS + K3CloudManager 服务
  - 账号问题 → 让 admin 登录系统管理中心解锁 / 重置密码
  - 版本问题 → 重装客户端,从服务端最新版本下载安装包
- **链到**:如果登进去了但很慢,看 `performance-issues` §4
- **来源**:[金蝶云星空报错信息及解决办法](https://blog.csdn.net/weixin_42236617/article/details/106936989) · [星空企业版公有云常见问题汇总](https://vip.kingdee.com/article/469893721664889088) fetched 2026-04-23

### 场景 1.2:多账套切换后登不回原账套

- **症状**:用户在多账套环境切换账套后,回不到原账套或提示"该用户在此数据中心无权限"
- **常见原因**:
  1. 目标账套的用户分配被 admin 移除了
  2. 账套所在的数据中心停用
  3. 客户端 session 状态错乱(并行登录冲突)
- **诊断步骤**:用 `kingdee_*` 只读工具查该账套的数据中心状态;让用户完全退出客户端(任务管理器杀进程)后重新打开
- **解决方向**:admin 在**系统管理中心**(不是业务界面)重分配用户到账套;客户端清缓存重登
- **来源**:经验归纳 + 社区帖子综合

## 2. 账套 / 数据中心问题

### 场景 2.1:账套切换按钮灰掉 / 列表为空

- **症状**:登录成功但账套下拉框为空,或切换账套按钮点不动
- **常见原因**:
  1. 该用户未被分配到任何账套
  2. 数据中心服务未启动
  3. 用户角色被禁用
- **诊断步骤**:admin 登系统管理中心看用户分配
- **解决方向**:admin 补分配;联系客户 IT 检查 `K3CloudManager` 服务
- **来源**:经验归纳

### 场景 2.2:创建 / 启用账套失败

- **症状**:管理员尝试创建或启用新账套,卡在某一步或报错
- **常见原因**:
  1. SQL Server 实例未启动或磁盘空间不够
  2. 账套 SA 密码错(K/3 Cloud 初始化需要 sa 权限)
  3. 管理中心站点配置错(IIS 绑定 / 应用池标识问题)
- **诊断步骤**:看管理中心日志 `<K3Cloud 安装目录>\WebSite\App_Data\Log`
- **解决方向**:这类基础设施问题不是 agent 能单独解决的,**建议用户找金蝶技服或熟悉部署的 IT**,附上日志
- **来源**:[金蝶云星空补丁无法安装的失败原因及解决方法大全](https://m.heshuyun.com/873.html) fetched 2026-04-23

## 3. 启用模块 / 初始化失败

### 场景 3.1:启用某模块(库存 / 销售 / 应收)失败

- **症状**:在"**启用**"或"初始化"界面点启用,报"未完成必要配置 / 存在未审核单据 / 前置模块未启用"
- **常见原因**:
  1. **模块启用有顺序依赖**——先启基础资料 → 总账 → 应收应付 / 库存 / 销售 / 成本
  2. 前一模块的期初单据未审核完
  3. 业务参数没填全(如"核算维度"没指定)
  4. 组织架构没配置完(该模块要求的核算组织未挂上)
- **诊断步骤**:让用户查**各模块启用状态**页,看哪些已启、哪些待启
- **解决方向**:
  - 按金蝶推荐的顺序重走,**不要跳步**——顺序:系统 → 基础资料 → 总账 → 其他
  - 用 `kingdee_*` 工具查模块的启用标志位是否正确
  - 期初单据没审核完 → 先去单据列表批量审核
- **链到**:如果卡在"关账失败"阶段的前置,看下面 §4
- **来源**:[金蝶云星空开发手册](https://www.cnblogs.com/lanrenka/collections/5745?page=1) · 经验归纳 fetched 2026-04-23

### 场景 3.2:初始化数据导入失败

- **症状**:用 Excel 模板批量导入期初数据(客户 / 物料 / 期初库存)失败,报"引入数据失败"
- **常见原因**:
  1. 模板字段不匹配(Excel 列顺序 / 名称与系统模板不同)
  2. 必填字段为空或基础资料未分配到业务组织
  3. 物料 / 客户编码重复
  4. 日期格式错(系统要 yyyy-MM-dd)
- **诊断步骤**:让用户**先导入 1 条记录**看具体报错,不要一次批量几千条
- **解决方向**:逐字段对照系统模板,按报错提示修
- **来源**:[金蝶云星空引入数据失败](https://blog.csdn.net/weixin_42236617/article/details/106936989) fetched 2026-04-23

## 4. 期末关账失败

### 场景 4.1:应付 / 应收 / 库存关账被拦

- **症状**:点"期末结账"按钮报"存在未审核凭证 / 未结清单据 / 期初未平"
- **常见原因**(按出现频率):
  1. **有未审核的单据**(采购入库单 / 销售出库单 / 收款单 / 付款单)
  2. **凭证未审核**或**凭证与单据不平**
  3. 负库存告警未处理(库存模块关账时)
  4. 应收应付未核销完
  5. **月结顺序错了**——正确顺序:应付 → 库存 → (应收 / 现金管理 / 存货核算 / 固定资产无序) → 总账
- **诊断步骤**:
  1. 让用户查**关账检查报告**(关账按钮旁边通常有"**期末结账合规检查**"入口)——金蝶会列出具体哪几张单 / 哪些科目阻塞
  2. 按报告逐项处理:未审核的审掉 / 未核销的核销掉 / 负库存调平
- **解决方向**:
  - 让用户**按顺序关**:应付 → 库存 → 总账(其他无序)
  - 负库存用"盘盈盘亏"或"其他出入库"在当期调平
  - 如果"凭证与单据不平"且差额很小可能是**汇率差 / 进位差** → 看账面差额后用凭证调
- **链到**:如果是应收应付数据本身对不上,跳 `data-inconsistency` §3
- **来源**:[金蝶云星空系统月结顺序](https://www.zjgkd.cn/?p=1408) · [星空系统结账手册](https://vip.kingdee.com/article/372379712616026112) · [金蝶云星空存货核算结账操作流程](https://m.heshuyun.com/509.html) fetched 2026-04-23

### 场景 4.2:总账关账报"凭证未过账"

- **症状**:前面模块都关了,总账关账提示"存在未过账凭证"
- **常见原因**:凭证审核了但没过账(审核 ≠ 过账,是两步)
- **诊断步骤**:让用户查"**凭证过账**"菜单,批量过账
- **解决方向**:走**凭证 → 批量过账**
- **来源**:[凭证反过账 金蝶](https://blog.csdn.net/weixin_32184669/article/details/112331854) fetched 2026-04-23

### 场景 4.3:反关账失败

- **症状**:用户想反关账(回到前期改数据),提示"反关账失败 / 被凭证锁定 / 后期已关账"
- **常见原因**:
  1. 后续期间已经关账——必须**先反关后期**再反关前期
  2. 凭证已归档 / 已锁定
  3. 用户权限不够(反关账通常是 admin 或专门的财务主管角色)
- **诊断步骤**:看当前各期间状态
- **解决方向**:按顺序从最后一个已关期间往前反;权限不够让 admin 授权
- **来源**:[云星空反过账失败提示凭证锁定](https://ask.csdn.net/questions/8882538) fetched 2026-04-23

## 5. 业务参数 / 系统参数缺失

### 场景 5.1:操作单据报"业务参数未配置"

- **症状**:保存或审核单据时报"系统参数 XX 未设置 / 业务参数缺失"
- **常见原因**:
  1. 该模块的**系统参数**(核算科目 / 默认仓库 / 默认客户 / 期间规则)没配
  2. 组织级别参数没设(集团启用了多组织但某个子公司没配)
  3. 特定单据类型参数缺失(如销售订单要"价目表策略"但没选)
- **诊断步骤**:让用户进**系统设置 → 系统参数**看对应模块,或**财务会计 → 总账 → 基本参数**
- **解决方向**:按报错提示定位到具体参数项,填上
- **来源**:经验归纳 + 社区帖子综合

## 6. 单据类型未启用 / 不可见

### 场景 6.1:用户找不到某类单据入口

- **症状**:菜单里看不到"销售订单 / 采购订单 / 调拨单"等预期单据的入口
- **常见原因**:
  1. 模块未启用(最常见)
  2. 用户**角色没授权**这个单据功能
  3. **组织权限**不在——用户被分到了没这个单据权限的组织
  4. 单据类型本身被禁用(admin 在"**单据类型管理**"里禁了)
- **诊断步骤**:
  1. 让用户切到 admin 试看能不能看到——能看到 = 权限问题
  2. 查**权限分配 → 功能权限**
- **解决方向**:admin 授权;模块启用;组织调整
- **来源**:经验归纳

## 临时话术(排障完成 / 卡住时的交付语)

### 排障到位但要用户动手

> "定位到了:**症状** XX,**根因** YY。请按下面动作:
> 1. [具体步骤 1]
> 2. [具体步骤 2]
> 做完后在系统里重新试一次 [触发操作],把结果(成功/仍报错+错误文本)回复我,我再判断下一步。"

### 排障走完没定位(诚实)

> "按 `systematic-debugging` 我列了假设 A / B / C 都查过了都不是。建议你按以下顺序升级:
> 1. 让客户 IT 把**服务端日志**发给我(`<K3Cloud 安装目录>\WebSite\App_Data\Log`,按日期找最近一个 .log 文件)
> 2. 客户端报错时按 **Ctrl+F12** 调出性能面板截图
> 3. 如果都没线索,去 [vip.kingdee.com](https://vip.kingdee.com/) 提工单(附完整报错文本 + K/3 Cloud 版本号 + 补丁号)
> 基础问题我继续看,系统层面的需要金蝶技服。"

### 判断是客户现场二开干扰(最常被忽略)

> "这个报错提示文本里有中文业务描述——看起来像你们现场的**二开插件**拦下的,不是金蝶原厂行为。可以让技术方查一下这个单据是否有 BOS 扩展,扩展上挂了什么 Python / DLL 插件。如果能登后台,我可以用 `kingdee_list_extensions('<baseObjectId>')` 查一眼。"
