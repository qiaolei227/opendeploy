## 1. 移动表单设计器入口

### 移动平台设计器
- **能做什么**:在 BOS 集成开发平台里"所见即所得"地拖拽创建移动业务对象,设计完发布到移动端主控台(支持金蝶云移动工作台 / 云之家 / 企业微信 / 钉钉)
- **配置入口**:BOS 集成开发平台 → 【新建】→ 选择"移动业务对象"类型(移动表单 / 移动单据)
- **典型场景**:把 PC 端某个高频单据(如外勤拜访登记 / 出差申请 / 报销单)做一个移动版本,让一线员工手机填
- **OpenDeploy 工具覆盖**: ❌
- **来源**: https://help.open.kingdee.com/dokuwiki/doku.php?id=移动平台设计器 · fetched 2026-04-23

### 移动表单 vs 移动单据(两类业务对象)
- **能做什么**:
  - **移动表单**:类似 PC 的"动态表单",单一界面,常用于过滤条件 / 导航栏 / 系统参数 / 轻交互页
  - **移动单据**:类似 PC 的"普通单据",分"单据"和"列表"两套界面,适合基础资料 / 业务单据
- **配置入口**:同上,在新建时按用途二选一
- **典型场景**:做"移动审批意见录入"用移动表单;做"移动销售订单"用移动单据
- **OpenDeploy 工具覆盖**: ❌
- **来源**: https://help.open.kingdee.com/dokuwiki/doku.php?id=移动平台设计器 · fetched 2026-04-23

## 2. PC 表单 → 移动表单的字段映射

### 来源单据绑定
- **能做什么**:移动单据可在"来源单据"属性里绑定一个 PC 端基础资料 / 单据,绑定后 PC 端字段直接可在移动单据上拖拽使用,不用重定义
- **配置入口**:移动单据属性面板 → 来源单据 → 选 PC 端业务对象
- **典型场景**:做移动版销售订单,绑定 `SAL_SaleOrder` 后字段(客户 / 物料 / 数量 / 金额)直接复用
- **OpenDeploy 工具覆盖**: ❌
- **来源**: https://help.open.kingdee.com/dokuwiki/doku.php?id=移动平台设计器 · fetched 2026-04-23

### 字段类型映射(常规字段)
- **能做什么**:移动端支持的常规字段类型——文本 / 小数 / 整数 / 多行文本 / 日期 / 时间 / 复选框 / 下拉列表 / 多选下拉 / 基础资料 / 多选基础资料 / 移动列表(多行卡片样式)
- **配置入口**:设计器左侧字段面板拖入
- **典型场景**:PC 端字段大多能直接对应,**没有 1:1 对应的(如 PC 的复杂下推字段)需要在移动单据里用基础资料 / 文本兜底**
- **OpenDeploy 工具覆盖**: ❌
- **来源**: https://help.open.kingdee.com/dokuwiki_std/doku.php?id=公有云移动bos · fetched 2026-04-23

## 3. 移动专属控件(扫码 / 拍照 / 定位 / 签名)

> 🔴 **公开手册资料不充分**。下列条目以 BOS 通用控件列表里**与移动端能搭得上**的项为准,但官方文档**没有专门的"移动专属控件"清单**,也没有"拍照"和"GPS 定位"控件的明确条目。**真正落地建议:让懂 PDA / 移动端实施的二开人员在 BOS Designer 里实测确认**。

### 二维码控件
- **能做什么**:把字段值转成二维码图片显示
- **配置入口**:BOS Designer → 通用控件 → 二维码
- **典型场景**:移动端展示物料 / 单据二维码供其他人扫
- **OpenDeploy 工具覆盖**: ❌
- **来源**: https://help.open.kingdee.com/dokuwiki_std/doku.php?id=bos通用控件列表 · fetched 2026-04-23

### 附件控件 / 附件列表
- **能做什么**:上传 / 删除附件;附件列表支持批量新增 / 查看 / 修改 / 下载 / 在线预览
- **配置入口**:通用控件 → 附件控件 / 附件列表
- **典型场景**:**移动端"拍照"在没有专属拍照控件的情况下,通常借附件控件实现**——客户端唤起相机,拍完作为附件上传(具体行为依移动客户端版本,需实测)
- **OpenDeploy 工具覆盖**: ❌
- **来源**: https://help.open.kingdee.com/dokuwiki_std/doku.php?id=bos通用控件列表 · fetched 2026-04-23

### 电子签章
- **能做什么**:存放流程审批人的电子签章
- **配置入口**:通用控件 → 电子签章
- **典型场景**:移动审批时显示 / 落审批人签名图;**不是手写签名板**
- **OpenDeploy 工具覆盖**: ❌
- **来源**: https://help.open.kingdee.com/dokuwiki_std/doku.php?id=bos通用控件列表 · fetched 2026-04-23

### 音频 / 串口控件
- **能做什么**:音频播放;串口对接外接 USB / 串口设备(扫码枪 / 称重设备)
- **配置入口**:通用控件 → 音频 / 串口
- **典型场景**:PDA 接外置扫码枪走串口;车间称重设备读数
- **OpenDeploy 工具覆盖**: ❌
- **来源**: https://help.open.kingdee.com/dokuwiki_std/doku.php?id=bos通用控件列表 · fetched 2026-04-23

### 拍照 / 定位 / 手写签名
- 🔴 **官方手册无明确条目**。建议:
  - 拍照:借附件控件 + 移动客户端的相机能力
  - 定位:可能在移动主控台 / 表单服务里通过插件取 GPS,需找二开实测
  - 手写签名:无开箱即用控件,通常二开嵌 H5

## 4. 移动审批 / 移动消息推送

### 移动审批
- **能做什么**:在移动客户端处理工作流审批任务,与 PC 端工作流共用审批流定义
- **配置入口**:工作流 / 审批流设计器配置审批节点 → 用户在移动客户端(金蝶云移动工作台 / 云之家)收任务处理
- **典型场景**:出差 / 报销 / 采购单审批走移动端
- **OpenDeploy 工具覆盖**: ❌
- **来源**: https://help.open.kingdee.com/dokuwiki_std/doku.php?id=审批流设计 · fetched 2026-04-23

### 移动消息推送
- 🔴 **公开手册未给出独立配置入口**。审批 / 消息推送通常依托云之家或企业微信 / 钉钉的消息通道,具体推送规则在云之家管理后台 / 第三方 IM 后台配置
- **建议**:查云之家管理后台的"消息中心" / "应用消息配置";K/3 Cloud 侧主要负责把审批任务和业务事件丢给消息通道,不直接管推送规则

## 5. 云之家 / 金蝶云 APP 集成

### 云之家集成配置
- **能做什么**:把 K/3 Cloud 用户和云之家用户做映射,审批 / 消息可在云之家收发,业务对象可在云之家轻应用里打开
- **配置入口**:administrator 登录 → 【基础管理】→【移动设置】→【云之家集成】→【云之家集成配置】(另有【查询用户】→【云之家用户同步】路径);后台每天自动执行同步
- **典型场景**:企业已用云之家做 IM,把 K/3 Cloud 审批 / 消息接进来不用再装一个 APP
- **OpenDeploy 工具覆盖**: ❌
- **来源**: https://help.open.kingdee.com/dokuwiki/doku.php?id=云之家集成配置 · fetched 2026-04-23

### 金蝶云移动工作台(原生 APP)
- **能做什么**:金蝶官方移动客户端,Android / iOS 双平台,访问移动端主控台发布的所有移动应用 / 移动单据 / 工作流任务
- **配置入口**:用户手机浏览器访问 `http://3g.k3cloud.kingdee.com/m/AppDowload.htm` 下载;管理员在【移动设置】→【移动主控台】发布应用
- **典型场景**:不用云之家 / 企业微信的客户,直接装金蝶官方 APP
- **OpenDeploy 工具覆盖**: ❌
- **来源**: https://help.open.kingdee.com/dokuwiki_std/doku.php?id=移动客户端 · fetched 2026-04-23

### 微信 / 企业微信 / 钉钉发布
- **能做什么**:移动平台设计器里设计完的移动业务对象可发布到云之家 / 微信 / 企业微信 / 钉钉,以轻应用形式打开
- **配置入口**:移动主控台发布时选目标渠道
- **典型场景**:客户已重度使用钉钉 / 企业微信,不想多装 APP
- **OpenDeploy 工具覆盖**: ❌
- **来源**: https://help.open.kingdee.com/dokuwiki_std/doku.php?id=移动平台 · fetched 2026-04-23

## 6. PDA 应用(车间 / 仓库)

> 🔴 **公开手册资料不足**。dokuwiki 上没有独立的"PDA 应用"主题页,PDA 在 K/3 Cloud 的形态通常是:
>
> - **手段 A**:用移动平台设计器做移动单据,部署到运行 Android 的 PDA 上的"金蝶云移动工作台"APP
> - **手段 B**:车间 / 仓库专属轻应用(如工序派工的移动版),官方在生产模块里有相关页面
> - **手段 C**:扫码枪通过 BOS 通用控件中的"串口控件"接入 PDA
>
> **配置入口**:无统一入口,按上面 3 条路分别走
>
> **典型场景**:仓库收货扫码 / 车间报工扫码 / 工序派工
>
> **OpenDeploy 工具覆盖**: ❌
>
> **建议**:让客户的 IT / 二开实施人员在 BOS Designer + 移动主控台里实测;OpenDeploy v0.1 不介入 PDA 配置
>
> **来源**: https://help.open.kingdee.com/dokuwiki/doku.php?id=工序派工 · fetched 2026-04-23(只是关联性参考,非 PDA 专题)
