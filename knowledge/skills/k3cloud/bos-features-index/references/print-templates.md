# 套打模板(BOS) · 主流程

> K/3 Cloud 的"打印"分两个体系:**套打模板**(像素级精确排版,常用于纸质单据 / 标签 / 凭证)和**标准报表 / Excel 模板**(轻量,数据网格式)。本文聚焦套打。
>
> 套打模板的设计、绑定、运行时切换都在 BOS 体系内,不是单据本身的字段。OpenDeploy v0.1 **不工具化套打模板**——元数据形态是 XML + 二进制,设计器是 Windows 客户端,自动化收益低。

---

## 1. 套打模板设计器入口与版本差异

### 套打设计器(独立工具)
- **能做什么**: 像素级设计单据 / 标签 / 凭证打印模板,支持一维码 / 二维码控件、合计大写、分录区循环、子单据体嵌套
- **配置入口**: 客户端"下载中心" → 下载并安装"套打设计器"(独立 EXE,不在 BOS Designer 里);打开后顶部工具栏【新建】→ 选子系统 → 选业务对象(如"采购订单")→ 进入设计画布
- **典型场景**: 客户要求把销售出库单打成定制纸张(税票格式 / 行业模板 / 含 logo 抬头);标签机直连打印
- **OpenDeploy 工具覆盖**: ❌
- **来源**: [模板设计与应用](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E6%A8%A1%E6%9D%BF%E8%AE%BE%E8%AE%A1%E4%B8%8E%E5%BA%94%E7%94%A8) · fetched 2026-04-23

### BOS Designer vs 套打设计器
- **BOS Designer**: 设计单据**结构**(字段、布局、业务规则)
- **套打设计器**: 设计单据**纸面打印输出**(版面、坐标、字体、条码)
- 两者**完全独立工具**,数据通过"业务对象"绑定关联

### 标签模板配置
- **能做什么**: PRN 文件 + 变量映射方式打标签,支持柔性产线 HMI 直连斑马打印机
- **配置入口**: PC 端"标签模板配置"菜单 → 新建 → 选分组 → 设定"适用单据"+ "标签变量"(变量名必须 = PRN 文件占位符名)
- **典型场景**: 工序汇报后打产品标签;箱签随包装动作触发
- **OpenDeploy 工具覆盖**: ❌
- **来源**: [标签模板配置](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%A0%87%E7%AD%BE%E6%A8%A1%E6%9D%BF%E9%85%8D%E7%BD%AE) · fetched 2026-04-23

## 2. 模板与单据的绑定

### 默认套打模板(开发期绑定)
- **能做什么**: 在 BOS Designer 里把某个套打模板设为单据**默认**,所有用户进打印界面都先看到它
- **配置入口**: BOS Designer → 打开目标单据(如采购订单)→ 单据属性面板 → 【默认套打模板】下拉 → 选已设计好的模板 → 保存单据
- **典型场景**: 实施期统一全公司打印模板;新建单据类型后绑模板
- **OpenDeploy 工具覆盖**: ❌
- **来源**: [套打设置与应用](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%A5%97%E6%89%93%E8%AE%BE%E7%BD%AE%E4%B8%8E%E5%BA%94%E7%94%A8) · fetched 2026-04-23

### 套打设置(运行期切换)
- **能做什么**: 用户级 / 公司级覆盖默认模板,按条件切换模板,指定打印机
- **配置入口**: 单据列表 / 编辑界面工具栏 → 【套打设置】按钮 → 三个页签:
  - **默认页签**: 设当前用户该单据类型的默认模板
  - **高级页签**: 配多个打印方案,基于"单据头字段"做条件判断(如"销售组织 = A 公司用模板甲,= B 公司用模板乙"),"打印机名称"支持 `*` 模糊匹配,"套打排序"按"单据体分录字段"排序
  - **参数设置页签**: 调批次打印页数 / 模板显示
- **典型场景**: 多组织环境不同公司打不同模板;特定打印机走专用纸张
- **OpenDeploy 工具覆盖**: ❌
- **来源**: [套打设置与应用](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%A5%97%E6%89%93%E8%AE%BE%E7%BD%AE%E4%B8%8E%E5%BA%94%E7%94%A8) · fetched 2026-04-23

### 三种模板形态并存
- 单据可以同时有套打模板 / 标准报表模板 / Excel 模板,运行时由用户在打印对话框里选择走哪种;套打模板优先级最高(实证待补)
- **来源**: [套打应用](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%A5%97%E6%89%93%E5%BA%94%E7%94%A8) · fetched 2026-04-23

## 3. 数据源绑定(头 / 明细 / 子单据体 / 基础资料联表)

🟡 **主流程**——官方手册分散在多页,核心机制确认,字段级细节请到设计器内 F1 / 实证。

### 业务对象 = 数据源根
- 设计器新建模板时**第一步选业务对象**(如 `SAL_SaleOrder`),整个画布的数据源树以这个对象为根
- 头字段、单据体、子单据体、基础资料的下钻字段(如客户的"信用额度")都在数据源树展开
- **配置入口**: 设计器左侧"数据源"导航树
- **来源**: [模板设计与应用](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E6%A8%A1%E6%9D%BF%E8%AE%BE%E8%AE%A1%E4%B8%8E%E5%BA%94%E7%94%A8) · fetched 2026-04-23

### 头字段 vs 明细字段
- **头字段**:拖到画布任意位置,打印时取当前单据的标量值
- **明细 / 单据体字段**:必须放在"分录区"(repeater)内,设计器自动按行循环
- **子单据体**(如生产订单的"工序明细" 嵌在"产品明细" 下):需嵌套分录区
- 实证字段绑定方式(双击字段 vs 拖拽 vs 表达式)请在设计器内核实

### 基础资料联表字段
- 单据上的基础资料字段(`FCustId`)在数据源树里可下钻取该客户档案的任意字段(`FCustomer.FCreditLimit`)
- **典型场景**: 销售订单打印里展示"客户当前信用额度";物料明细行里展示物料的"规格型号"
- **OpenDeploy 工具覆盖**: ❌

### 计算字段 / 公式
- 设计器支持表达式 / 内置函数(合计、大写金额、当前日期);具体函数列表请在设计器"插入公式" 里看
- **典型场景**: 发票打印必须有"金额大写";销售单需"折扣前 - 折扣 = 折扣后" 表达式
- **来源**: 推断自 [套打应用](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%A5%97%E6%89%93%E5%BA%94%E7%94%A8) · fetched 2026-04-23(手册未列函数表)

## 4. 条码 / 二维码 / 图片 / 公式

### 一维码 / 二维码控件
- **能做什么**: 套打设计平台内置**一维码模板和二维码模板**控件,字段绑定后运行时自动生成图形
- **配置入口**: 套打设计器画布 → 工具箱 → 选条码 / 二维码控件 → 拖入画布 → 属性面板绑数据源字段(如"单据编号" `FBillNo`)
- **典型场景**: 出库单打条码供仓库扫描收货;标签条码扫描下推下游单据
- **OpenDeploy 工具覆盖**: ❌
- **来源**: [条码管理](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%9D%A1%E7%A0%81%E7%AE%A1%E7%90%86) · fetched 2026-04-23

### 条码生成与条码主档
- 条码生成可"逐行"或"按规则分组合并",由"条码规则"决定;生成的条码进"条码主档",条码主档自身可按标签模板批量打印
- **配置入口**: 条码云 → 条码规则 / 条码主档 / 条码生成菜单
- **典型场景**: 仓储出库批量生成箱码;追溯码贴产品
- **来源**: [条码生成](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%9D%A1%E7%A0%81%E7%94%9F%E6%88%90) · fetched 2026-04-23

### 图片 / Logo
- 套打设计器支持插入静态图片(公司 logo) 和动态图片字段(如签章字段);具体控件名 / 支持格式手册未明列,实证待补 🔴
- **典型场景**: 销售单抬头公司 logo;电子签章

## 5. 凭证打印 vs 单据打印的区别

🔴 **资料不足**——官方"凭证模板"页讲的是凭证**生成规则**(借贷方向 / 摘要取值)而非**打印模板**,与本文 1-4 节的套打体系不是一回事。

### 已知部分
- **凭证模板**(`【财务会计】→【智能会计平台】→【基础资料】→【凭证模板】`)定义的是"业务单据如何生成会计凭证"——凭证日期、凭证字、摘要、科目、借贷方向、金额取值规则,**不是打印模板**
- 凭证**打印**是否走套打设计器、有无独立的"凭证套打模板"体系,手册未明确
- **建议**: 顾问遇到凭证打印需求,先到客户端打开一张凭证 → 看打印对话框里有哪几种模板形态(套打 / 标准报表 / Excel) → 再去对应模块找模板设计入口
- **来源**: [凭证模板](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%87%AD%E8%AF%81%E6%A8%A1%E6%9D%BF) · fetched 2026-04-23

## 6. 打印插件(`PrintPlugins` 节点)

### AbstractPrintPlugIn
- **能做什么**: 在打印数据生成阶段拦截,定制打印数据(如生成动态二维码、追加合计大写、按状态切换模板)
- **配置入口**: BOS Designer → 单据扩展 → 注册打印插件;元数据节点 `<PrintPlugins>`(类比 `<FormPlugins>`,挂在扩展对象的 `FKERNELXML` 里)
- **典型场景**:
  - 标签内容无法直接从单据字段取(需联表 / 计算)→ 写打印插件补数据
  - 根据单据状态(已审核 / 草稿)选不同模板
  - 打印时实时生成二维码字符串(含时间戳 / 用户)
- **技术形态**: C# / IronPython,基类 `AbstractPrintPlugIn`
- **OpenDeploy 工具覆盖**: ❌(v0.1 只覆盖 `<FormPlugins>`,见 `plugin-types.md`)
- **来源**: [标签模板配置](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%A0%87%E7%AD%BE%E6%A8%A1%E6%9D%BF%E9%85%8D%E7%BD%AE)("二开插件的自定义方式实现") · fetched 2026-04-23

### 与表单插件的区别
- 表单插件触发在 UI 层(保存前 / 字段变),打印插件触发在打印数据装配阶段
- 表单插件改不了打印输出——同一字段在表单层显示和打印输出可以不一样,要改打印必须走打印插件
- **识别话术**: 用户说"打印时多一行 / 打印时这个字段要变 / 打印时算个总价大写",大概率是打印插件,**不是**表单插件

## 7. 套打配置常见坑

🟡 **下面是基于手册描述 + 通用打印实施经验的归纳,客户环境未实测,使用时请验证**。

### 7.1 模板设计器和 BOS Designer 是两个独立工具
- 新顾问常以为套打模板在 BOS Designer 里改——**不是**。要装独立的"套打设计器"(下载中心)。
- BOS Designer 里只能**绑定**已有模板(单据属性 → 默认套打模板下拉),不能编辑模板内容。

### 7.2 默认模板 vs 套打设置覆盖
- BOS Designer 设的是**全局默认**;每个用户在客户端"套打设置"里设的是**用户级覆盖**——用户级优先于全局。
- 实施期改了默认模板,某些用户客户端里看不到改动 → 大概率被该用户的"套打设置"覆盖了,要清掉用户级配置。

### 7.3 标签变量名 = PRN 占位符名
- 标签模板的"标签变量名" **必须**与 PRN 文件里的占位符同名,大小写敏感(实证待补)。
- 改了 PRN 没改变量名 → 打出来空白 / 占位符原文。
- **来源**: [标签模板配置](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%A0%87%E7%AD%BE%E6%A8%A1%E6%9D%BF%E9%85%8D%E7%BD%AE) · fetched 2026-04-23

### 7.4 高级方案条件字段只支持单据头
- "套打设置 → 高级"的条件字段只能取**单据头字段**——按"分录上某物料"切模板做不了(只能写打印插件)。

### 7.5 打印机名称模糊匹配
- 高级方案绑打印机名支持 `*` 通配,运行时按客户端实际打印机名匹配;客户端打印机名变了(驱动重装 / 共享路径变)→ 走默认而不是预期模板,排错时先看打印机名实际值。

### 7.6 子单据体嵌套循环易错
- 多层嵌套(产品明细下嵌工序明细)的分录区放错层级会导致打印行重复或丢失;设计时务必预览实际数据,不要光看设计器示意。

### 7.7 凭证打印 ≠ 凭证模板
- "凭证模板"配的是凭证**生成规则**,不是打印输出。打印格式问题不要去改凭证模板。

### 7.8 套打模板格式不可跨版本随便迁移
- 不同 K/3 Cloud 版本(V8 / V9 / V9.x SP)套打模板格式可能不兼容,跨版本升级要重新验证模板,**实证待补** 🔴。

## OpenDeploy v0.1 工具覆盖

❌ **不工具化套打模板**。原因:
1. 模板元数据是 XML + 二进制混合,设计器深度依赖 Windows 客户端
2. 实施需求绝大多数是"调整版面 / 加字段 / 加 logo",不是结构性二开
3. 写打印插件(`<PrintPlugins>`)虽然类比表单插件可做,但 v0.1 只覆盖 `<FormPlugins>`(见 `plugin-types.md`)

未来计划见 `python-plugin-index` 路线图(暂未排期)。

## 临时话术

> "套打模板 OpenDeploy 暂不自动化。K/3 Cloud 套打分两层:
>
> - **模板设计**:用独立的'套打设计器'(下载中心装),按业务对象新建 → 拖字段 / 加条码 / 设公式 → 保存
> - **绑定**:在 BOS Designer 的单据属性里设'默认套打模板';用户级在客户端工具栏'套打设置'里覆盖
>
> 如果是'打印时多一行 / 字段要算 / 二维码要动态生成',那是**打印插件**(`PrintPlugIn`),OpenDeploy v0.1 也不覆盖,需要你或开发同事手工写。
>
> 如果只是改版面 / 加 logo / 改字段位置,直接套打设计器里调,不需要二开。"
