# 转换规则 / 下推配置

**下推** = 从 A 单据按规则生成 B 单据(带字段映射、按行勾选、带汇总)。

---

## 典型下推链

```
销售订单 → 发货通知单 → 销售出库单 → 应收单 → 收款单
采购申请 → 采购订单 → 采购收料通知 → 采购入库单 → 应付单 → 付款单
库存调拨申请 → 调拨发货 → 调拨收货
```

每一步都是标准转换规则配置。

---

## 转换规则配置路径

1. BOS Designer → **单据转换规则**
2. 新建:源单据 `SAL_SaleOrder` → 目标单据 `SAL_DELIVERYNOTICE`
3. 配置:
   - **字段映射**:源字段 → 目标字段(可多对一、可带公式转换)
   - **分组策略**:按什么字段分组合并(比如按"客户 + 交货日期"合并多张订单到一张发货通知)
   - **选单策略**:用户下推时能看到哪些源单、必选 / 可选
   - **过滤策略**:哪些源单据允许下推
4. 保存 → 刷新

---

## 常见需求 → 转换规则方案

### "销售订单下推发货通知时要带上客户地址"

- 默认下推带了客户 ID,但**客户地址**是客户档案的字段,不在订单上
- **方案**:在转换规则里加字段映射:`目标.FReceiveAddress = 源.FCustId.FAddress`(基础资料联表)

### "按区域合并多张订单到一张发货通知"

- **方案**:分组策略,按 `FCustRegion` 分组

### "扩展字段要跟着下推"

- 如果 A / B 单据都扩展了同名字段,**需要在转换规则里手工加映射**——BOS 不自动传扩展字段
- 这是常见坑:顾问扩展了字段,发现下推没带,以为是 bug

### "下推时金额要按比例重算"

- 字段映射可带公式:`目标.F1 = 源.F1 * 0.9`

---

## 转换插件(进阶)

当字段映射 + 分组 + 过滤不够时,可挂**转换插件**做复杂逻辑:

- 插件类型:`DLL` 或 `Python`(IronPython)
- 事件:`BeforeConvert` / `AfterConvert`
- 用途:下推时校验源单状态、调用外部 API、写自定义日志

**OpenDeploy v0.1 不支持转换插件**(只支持表单插件)。要做转换插件,手工 BOS Designer。
