# 业务规则 / 公式

BOS 的**公式引擎**——字段之间的自动计算,不用写代码。

---

## 适用场景

- **`F金额 = F数量 * F单价`** —— 最经典
- **`F折扣额 = F原价 * (1 - F折扣率)`**
- **`F交货日 = F下单日 + 7`**
- **`F税额 = F金额 * F税率`**
- **头字段汇总明细**:`F总金额 = SUM(明细.F金额)`

---

## 配置路径

1. BOS Designer → 目标单据扩展 → **业务规则** 标签(或属性页)
2. **新建规则**:
   - 规则名:`FAmount 自动计算`
   - 触发字段:`FQty, FPrice` 任一变化时
   - 目标字段:`FAmount`
   - 公式:`FQty * FPrice`
3. 保存扩展 → 刷新

---

## 公式语法

BOS 公式是**限制版**表达式语言,**不是 Python 也不是 SQL**:

| 类型 | 示例 |
|---|---|
| 四则运算 | `F1 + F2 * F3 / F4` |
| 条件 | `IIF(F1 > 100, F1 * 0.9, F1)` |
| 字符串 | `CONCAT(F1, "-", F2)` |
| 日期 | `DATEADD(F1, 7, "d")` |
| 数学函数 | `ROUND(F1, 2)`, `ABS(F1)` |
| 聚合(头里引明细) | `SUM(EntryKey.F1)` |

详细函数表要反编译 `Kingdee.BOS.Formula` 或看 Designer 里的公式编辑器。

---

## 业务规则 vs Python 插件

| 用业务规则 | 用 Python 插件 |
|---|---|
| 字段间**纯计算** | 需要**校验 / 阻断保存** |
| 值可以由函数表达 | 逻辑含 if-else 分支多 |
| 不需读其他单据 / 基础资料 | 需跨查数据(客户档案 / 库存) |
| 维护者是实施顾问 / 非程序员 | 需要程序员维护 |

**判断标准**:如果你能一行公式写完,就用业务规则;写着写着发现需要 `if A then B else C` 嵌套,就升到 Python 插件。

---

## 常见坑

1. **循环依赖**:`F1 依赖 F2`,`F2 依赖 F1` → BOS 会报环。重新梳理依赖方向
2. **聚合性能**:明细 100 行以上,头字段 `SUM(...)` 实时算可能卡。考虑改成"保存前 Python 计算一次"
3. **精度丢失**:小数位计算用 `ROUND(..., 2)` 显式指定,否则不同版本 BOS 精度行为不一致
4. **触发时机**:业务规则是**字段变化后**算,不是保存前。要保存前强制重算,补一个 `BeforeSave` 插件
