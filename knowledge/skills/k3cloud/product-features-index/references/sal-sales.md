# 销售管理(SAL)标准功能字典

实施顾问最常接的模块。下面按**典型需求关键词**反查标准功能。

---

## 1. 信用管理

### 客户信用额度控制
- **需求关键词**:"超额禁止下单"、"额度预警"、"信用冻结"
- **标准功能**:启用 **信用管理** 后,K/3 Cloud 自动校验销售订单金额 vs 客户信用额度,超额可配置为"阻止"、"仅提醒"、"审批通过"
- **启用路径**:应收管理 → 基础资料 → **信用策略** + 客户档案 → "信用管理"页签填 `FCreditLimit`
- **典型误判**:顾问上来就写 Python 校验插件 — **先问客户启用了信用策略没**

### 应收账龄分析 / 预警
- **需求关键词**:"看客户欠款多久"、"快到期的应收"
- **标准功能**:应收管理 → 账龄分析表(内置报表)
- **启用路径**:应收管理 → 报表 → 应收账龄分析
- **典型误判**:被问"能不能给我一张按客户欠款天数列的表" → 不是二开,是内置报表

---

## 2. 价格管理

### 销售价目表 / 折扣
- **需求关键词**:"不同客户不同价"、"阶梯价"、"老客户 95 折"
- **标准功能**:**销售价目表** + **价格组**——按客户 / 物料 / 数量段配置不同价格
- **启用路径**:销售管理 → 基础资料 → **价目表** + **价格策略**
- **典型误判**:客户说"每个大客户价格不一样" → 不是要二开,是价目表功能

### 手工调价权限
- **需求关键词**:"销售员不能随便改价"、"保护最低价"
- **标准功能**:**价格权限方案**——控制特定角色能否修改销售单价、折扣率
- **启用路径**:销售管理 → 基础资料 → 价格权限方案
- **典型误判**:顾问写 Python 拦截价格字段修改 — **标准有,别重写**

---

## 3. 订单流程

### 订单审批流
- **需求关键词**:"XX 元以上要主管批"、"大金额走多级审批"
- **标准功能**:**审批流** + **条件表达式**(Python 条件)——按单据字段组合触发不同审批路径
- **启用路径**:系统服务 → 审批流设计器(需切到"审批模式")
- **典型误判**:用户说"要审批" → 不是一定要做插件,**先走审批流设计器**。真要做复杂条件判断再在审批流里加 Python 表达式
- **OpenDeploy 工具覆盖**:❌ 审批流设计暂未工具化,让用户手工在 BOS Designer 改

### 订单状态机(审核 / 反审核 / 关闭 / 作废)
- **需求关键词**:"订单需要关闭"、"作废需审批"
- **标准功能**:K/3 Cloud 单据状态流是标准功能(`未审核 → 已审核 → 关闭`),不需二开
- **启用路径**:自动,配合审批流
- **典型误判**:"我需要一个关单按钮" → 标准已有

---

## 4. 销售员 / 组织

### 销售员维护 + 字段联动
- **需求关键词**:"选客户自动带销售员"、"按组织隔离销售员"
- **标准功能**:**客户档案 → 业务信息页签 → 默认销售员** 字段。保存订单时选客户会自动携带
- **启用路径**:基础资料 → 客户档案 → 业务信息
- **典型误判**:Python 联动写 `DataChanged("FCustId")` 自动填销售员 → **客户档案配了默认销售员就自动带,不用插件**

### 销售区域 / 销售组
- **需求关键词**:"按区域统计销售"、"销售组独立核算"
- **标准功能**:**销售区域** 基础资料 + **销售员所属销售组** 字段
- **启用路径**:销售管理 → 基础资料 → 销售区域 / 销售组

---

## 5. 发货 / 退货

### 分批发货
- **需求关键词**:"订单分几次发"、"部分发货"
- **标准功能**:销售订单 → 下推 → 发货通知单(可部分下推)→ 销售出库单
- **启用路径**:自动,单据流支持

### 销售退货
- **需求关键词**:"客户退货"、"退款流程"
- **标准功能**:销售退货单 → 销售退货出库单(其实是入库)
- **启用路径**:销售管理 → 销售退货

---

## 6. 核销 / 结算

### 销售收款
- **需求关键词**:"收钱和订单挂钩"、"预收款"
- **标准功能**:应收单 ↔ 收款单,可手工 / 自动核销
- **启用路径**:应收管理 → 收款单 + 核销
- **典型误判**:用户说"收款了要扣对应订单" → 核销就是做这个的

---

## 常见"以为要二开实则标准"的总结

| 顾问描述 | 真实答案 |
|---|---|
| "加个信用额度校验" | 启用信用管理 + 客户档案填额度,不用插件 |
| "选客户带销售员" | 客户档案默认销售员字段,不用联动插件 |
| "销售员不能随便改价" | 价格权限方案,不用 BOS 锁字段 |
| "按客户给不同价" | 销售价目表 + 价格组,不用 Python |
| "订单分批发货" | 下推机制标准支持 |
| "大金额走审批" | 审批流 + 条件,不用工作流二开 |
| "看账龄" | 应收账龄报表标准 |
