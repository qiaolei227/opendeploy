# 现金管理 / 出纳(CB)标准功能字典 🟡

> CB 模块覆盖银行账户 / 现金 + 银行日记账 / 票据 / 资金计划 / 银企互联 / 现金流量项目。本字典基于金蝶官方手册抽取主流程,**未在客户环境逐条实测**;具体子场景(对账方案细节、银企互联渠道差异)需要实施时反查手册或现场验证。
>
> 模块归属:**财务会计 → 出纳管理**(主)+ **财务会计 → 网上银行**(银企互联) + **财务会计 → 票据管理**(票据,V9 部分版本独立菜单) + **财务会计 → 资金管理**(集团资金)。

## 1. 银行账户管理

### 银行账号(我方账户)
- **需求关键词**:"我们公司有十几个银行户要维护"、"开户行 / 账号"、"销户"
- **标准功能**:**银行账号** 基础资料——管理本企业的银行账户,支持 **开户申请** / **销户申请** 流程
- **启用路径**:财务会计 → 出纳管理 → 基础资料 → 银行账号
- **典型误判**:顾问以为"客户 / 供应商的银行账号也在这里维护" → **不是**,客户 / 供应商账户在 **客户档案 / 供应商档案** 的"银行信息"页签,**银行账号** 主表只放"我方"账户
- **OpenDeploy 工具覆盖**:❌
- **来源**:[出纳管理 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%87%BA%E7%BA%B3%E7%AE%A1%E7%90%86) · fetched 2026-04-23

### 开户行
- **需求关键词**:"工行 XX 支行"、"开户银行档案"
- **标准功能**:**开户银行** 基础资料——独立维护银行机构 + 联行号
- **启用路径**:财务会计 → 出纳管理 → 基础资料 → 开户银行
- **典型误判**:把开户行直接写在银行账号的文本字段里 → 失去 F7 关联,后续银企互联 / 报表分类报错
- **OpenDeploy 工具覆盖**:❌

## 2. 现金 / 银行日记账(收 / 付 / 转)

### 收款单(出纳收款)
- **需求关键词**:"客户回款记一笔"、"销售收款"
- **标准功能**:**收款单**——处理销售业务及其他对外收款,可手工录入或由销售单 / 应收单下推
- **启用路径**:财务会计 → 出纳管理 → 日常处理 → 收款单
- **典型误判**:把"应收款管理"的收款单和"出纳"的收款单混为一谈 → 实际两侧同一张单据,看视角不同;**先在应收侧做核销,再在出纳侧确认收款** 是常规流程
- **OpenDeploy 工具覆盖**:❌
- **来源**:[出纳管理 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%87%BA%E7%BA%B3%E7%AE%A1%E7%90%86) · fetched 2026-04-23

### 付款单(出纳付款)
- **需求关键词**:"付供应商货款"、"采购付款"
- **标准功能**:**付款单**——处理采购及其他对外付款,可由付款申请单 / 应付单下推
- **启用路径**:财务会计 → 出纳管理 → 日常处理 → 付款单
- **典型误判**:用户说"要审批后才能付" → 标准走 **付款申请单 → 审批 → 下推付款单** 流程,不是二开
- **OpenDeploy 工具覆盖**:❌
- **来源**:[出纳管理 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%87%BA%E7%BA%B3%E7%AE%A1%E7%90%86) · fetched 2026-04-23

### 现金存取单
- **需求关键词**:"去银行取现"、"现金存银行"
- **标准功能**:**现金存取单**——处理日常存现 / 取现,自动同步现金 + 银行存款两侧日记账
- **启用路径**:财务会计 → 出纳管理 → 日常处理 → 现金存取单
- **典型误判**:用收款单 / 付款单去模拟存取现 → 流量项目和报表分类都错;**专用单据** 才会在现金日报正确分类

### 银行转账单
- **需求关键词**:"账户间内部划转"、"购汇"
- **标准功能**:**银行转账单**——处理本企业银行账号间转账以及购汇业务
- **启用路径**:财务会计 → 出纳管理 → 日常处理 → 银行转账单
- **典型误判**:做内部转账时一边付款单一边收款单 → 重复记账;**银行转账单** 是单一单据双侧落账

### 付款申请单
- **需求关键词**:"付款审批流"、"用款申请"
- **标准功能**:**付款申请单**——业务部门发起付款申请,经审批后下推付款单;可挂工作流
- **启用路径**:财务会计 → 出纳管理 → 日常处理 → 付款申请单

### 现金 / 银行日记账(查询)
- **需求关键词**:"出纳每天看的流水"、"银行存款日报"
- **标准功能**:**现金日记账** + **银行存款日记账**——记录库存现金 / 银行存款日常收付,**根据上述各类出纳单据自动生成**,出纳无需重复录入
- **启用路径**:财务会计 → 出纳管理 → 日记账 → 现金日记账 / 银行存款日记账
- **典型误判**:用户说"我要再录一遍流水" → **不需要**,日记账由收款 / 付款 / 存取 / 转账单自动汇总

## 3. 票据管理(应收 / 应付票据)

> V9 票据管理可以走 **出纳管理 → 票据处理** 子菜单,也可通过独立的 **财务会计 → 票据管理** 主菜单进入,功能等价。

### 应收票据
- **需求关键词**:"客户开承兑过来"、"商业汇票收款"、"票据贴现"
- **标准功能**:**应收票据**——支持票据登记 + **贴现** / **背书** / **背书退回** / **到期收款** / **退票** 全生命周期;含 基本 / 收付款银行 / 背书 / 收款单 等页签
- **启用路径**:财务会计 → 出纳管理 → 票据处理 → 应收票据 **或** 财务会计 → 票据管理 → 票据处理 → 应收票据
- **典型误判**:用户说"票据流转过程要可追溯" → **不是二开**,标准应收票据已经把背书 / 贴现链路串起来,可在 **应收票据使用流水** 报表查看
- **OpenDeploy 工具覆盖**:❌
- **来源**:[票据管理 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E7%A5%A8%E6%8D%AE%E7%AE%A1%E7%90%86) / [应收票据 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%BA%94%E6%94%B6%E7%A5%A8%E6%8D%AE) · fetched 2026-04-23

### 应付票据
- **需求关键词**:"开承兑给供应商"、"票据付款"
- **标准功能**:**应付票据**——可被付款单引用确认"以票付款",支持 **到期付款** / **退票** 等结算业务
- **启用路径**:财务会计 → 出纳管理 → 票据处理 → 应付票据
- **典型误判**:用户问"为什么付款单选了票据后银行余额没动" → 应付票据是负债性凭证,**承兑到期日** 才冲银行;不是 bug
- **OpenDeploy 工具覆盖**:❌
- **来源**:[应付票据 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%BA%94%E4%BB%98%E7%A5%A8%E6%8D%AE) · fetched 2026-04-23

### 电子票据(网银侧)
- **需求关键词**:"电票签收"、"ECDS"、"电子承兑汇票"
- **标准功能**:**电子票据**——网上银行模块下集成 **签收 / 开票 / 背书 / 贴现** 操作,与人民银行 ECDS 系统互通(具体走 BOS 银企平台)
- **启用路径**:财务会计 → 网上银行 → 票据管理
- **典型误判**:把电票当纸票录 → 缺少 ECDS 票号、签收链路;走网银侧才能和银行系统对齐
- **OpenDeploy 工具覆盖**:❌
- **来源**:[网上银行 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E7%BD%91%E4%B8%8A%E9%93%B6%E8%A1%8C) · fetched 2026-04-23

## 4. 资金计划 / 资金日报

> 资金管理是 **集团 / 多组织** 场景的进阶模块;单体小客户通常不启用。

### 资金管理(收支两条线)
- **需求关键词**:"集团资金集中"、"成员单位收支不能坐支"
- **标准功能**:**资金管理**——支持结算中心 **收支两条线** 模式,严禁坐支,资金组织根据资金计划或成员请款 **下拨** 到成员单位的支出账户
- **启用路径**:财务会计 → 资金管理
- **典型误判**:把"资金管理"和"出纳管理"混为一谈 → 出纳是单组织日常账,资金管理是 **多组织资金集中调度**;两者数据 + 流程都不同
- **OpenDeploy 工具覆盖**:❌
- **来源**:[资金管理 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E8%B5%84%E9%87%91%E7%AE%A1%E7%90%86) · fetched 2026-04-23

### 资金计划 / 上划下拨
- **需求关键词**:"成员公司每月报资金计划"、"母公司下拨"、"上划归集"
- **标准功能**:**资金计划** + **上划下拨**——计划申报 → 审批 → 上划 / 下拨执行,支持内部活期计息
- **启用路径**:财务会计 → 资金管理 → 日常处理 → 上划 / 下拨 / 个别报表
- **典型误判**:写 Python 自定义"内部利息算法" → 标准 **内部活期计息** 已经支持;先看是否够用再考虑二开
- **OpenDeploy 工具覆盖**:❌
- **来源**:[资金管理 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E8%B5%84%E9%87%91%E7%AE%A1%E7%90%86) · fetched 2026-04-23

### 资金日报 / 头寸表
- **需求关键词**:"每天看资金头寸"、"现金日报表"
- **标准功能**:**现金日报表** / **银行存款日报表** / **资金头寸表**——出纳模块自带,无需自建
- **启用路径**:财务会计 → 出纳管理 → 报表分析
- **典型误判**:顾问被要求"做一张资金日报"就开始写 BI / 二开 → **报表分析** 子菜单内置;先确认是否覆盖
- **OpenDeploy 工具覆盖**:❌
- **来源**:[出纳管理 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%87%BA%E7%BA%B3%E7%AE%A1%E7%90%86) · fetched 2026-04-23

## 5. 银企互联 / 银行对账单导入

### 银企直联(BOS 银企平台)
- **需求关键词**:"和银行直连"、"自动拉对账单"、"系统直接付款"
- **标准功能**:**银企互联** —— 通过金蝶 BOS 银企平台对接 **国内一百多家银行** 及支付宝直联,涵盖人民币 + 境内外币
- **启用路径**:财务会计 → 网上银行 → 基础设置 → 银企互联设置(配 BOS 银企平台地址 + 测试连接)
- **典型误判**:用户问"接个工行接口要多少钱" → **银企互联** 是配置题不是开发题;接入靠 BOS 银企平台 + 银行端开通,不需要二开写 socket
- **OpenDeploy 工具覆盖**:❌
- **来源**:[网上银行 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E7%BD%91%E4%B8%8A%E9%93%B6%E8%A1%8C) · fetched 2026-04-23

### 网上银行付款 / 接收交易明细
- **需求关键词**:"系统直接发起付款"、"自动收银行流水"
- **标准功能**:**银行付款单**(向银行系统提交付款指令)+ **接收银行交易明细**(自动拉流水生成业务单据)+ **电子回单** 下载

- **典型误判**:用户说"付款指令要复核 / 限额控制" → 标准 **网上银行参数设置** 里有付款验证 / 动态密码 / 额度控制配置项,不需要二开
- **OpenDeploy 工具覆盖**:❌
- **来源**:[网上银行 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E7%BD%91%E4%B8%8A%E9%93%B6%E8%A1%8C) · fetched 2026-04-23

### 银行对账(智能对账)

## 6. 现金流量项目设置

### 现金流量项目(基础资料)
- **需求关键词**:"做现金流量表"、"经营 / 投资 / 筹资 流量分类"
- **标准功能**:**现金流量项目** 基础资料——**系统已预设**适用于中国大陆新会计准则的现金流量项目表;**只能新增 / 维护最明细级**,中间级次不能编辑
- **启用路径**:基础资料 → 财务 → 现金流量项目
- **典型误判**:用户要"加自定义流量分类" → 直接新增 **明细级** 即可,**不要去改预设的中间层级**(系统不让改也不该改)
- **OpenDeploy 工具覆盖**:❌

### 科目挂"现金类"标签

## 7. 典型"以为要二开"汇总表

| 顾问 / 客户描述 | 真实答案 |
|---|---|
| "维护我们公司十几个银行户" | **银行账号** 基础资料(含开 / 销户申请) |
| "客户银行信息也在银行账号里维护吧" | **不是**——客户 / 供应商档案下的"银行信息"页签 |
| "出纳要再录一遍流水" | 不需要,**收 / 付 / 存取 / 转账单** 自动汇总到日记账 |
| "付款要先审批再付" | **付款申请单 + 工作流**,不是二开 |
| "内部账户互转要做两张单" | **银行转账单** 一张搞定 |
| "客户开承兑过来要管全流程" | **应收票据**(背书 / 贴现 / 到期收款全在标准里) |
| "电子承兑要走 ECDS" | **网上银行 → 票据管理** 的电子票据签收 / 开票 / 背书 |
| "要和工行直连付款 / 拉流水" | **银企互联**(BOS 银企平台) + **网上银行** 模块,配置题不是开发题 |
| "我们没银企直连,只有 Excel 对账单" | **银行对账单导入** + **导入方案设置**(每家银行配一个方案) |
| "对账自动找未达账项" | **对账方案** + **银行存款智能对账报告** |
| "集团多公司资金集中调度" | **资金管理**(收支两条线 + 资金计划 + 上划下拨) |

| "每张凭证手选流量项目太烦" | **科目预设流量项目**,凭证自动带出 |
| "做一张资金日报" | 出纳模块 **报表分析** 子菜单已有 |
| "自定义经营 / 投资 / 筹资 三类外的流量分类" | 现金流量项目 **新增最明细级** 即可 |
