# 库存管理(STK)标准功能字典

> 🟡 **主流程级别**。以金蝶官方公开手册(`help.open.kingdee.com/dokuwiki` 与 `dokuwiki_std`)为线索源,从**实施顾问视角**重写为"误判提醒+功能反查"字典。
>
> **未在客户环境逐条实测**——具体子参数 / 字段位置 / 版本差异(V8 / V9 各版次)以现场为准。带 `<!-- 待实证 -->` 的条目表示官方页面没明说,**不要凭印象补**。
>
> 库存(STK)是 K/3 Cloud 启用项最密集的模块——批号 / 序列号 / 保质期 / 仓位 / 货主 / 库存状态全在物料档案 + 仓库参数里勾选。**绝大多数"库存特殊需求"是启用项问题不是二开**。先反查这份字典再考虑 BOS / Python。

---

## 主流程

```
其他入库单 / 采购入库单 / 生产入库单
        ↓ (库存增加)
   即时库存(T_STK_INVENTORY)
        ↓ (库存消耗)
其他出库单 / 销售出库单 / 生产领料单 / 调拨发货单
```

库存核算路径独立:**单据更新即时库存** → 月末 **存货核算** → 出库成本核算 → 关账。

## 1. 多组织 / 多仓库 / 多仓位

### 库存组织
- **需求关键词**:"按组织隔离库存"、"分公司各自管自己的货"、"集团下多家工厂"
- **标准功能**:**库存组织**(具有库存职能的业务组织)独立核算库存数量和金额;是单据的主业务组织,必须先维护再录其他字段
- **启用路径**:基础资料 → 组织机构 → 业务组织 → 勾选"库存"职能;[供应链] → [库存管理] → [初始化] → [启用库存管理] 按组织启用
- **典型误判**:被问"分公司库存独立" 想做扩展字段过滤 → 库存组织本身就是隔离粒度
- **OpenDeploy 工具覆盖**:✅ 元数据可读
- **来源**:[启用库存管理](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%90%AF%E7%94%A8%E5%BA%93%E5%AD%98%E7%AE%A1%E7%90%86) · fetched 2026-04-23

### 货主(库存所有权)
- **需求关键词**:"代管别人的货"、"客户的货放我仓库"、"外部货主"
- **标准功能**:**货主** 字段表示所有权归属,可以是**业务组织 / 客户 / 供应商**三类;同一仓库可放多货主的货,即时库存按货主维度独立汇总
- **启用路径**:基础资料 → 客户 / 供应商 → 业务信息 → 勾选"作为货主";单据上"货主类型 + 货主"字段
- **典型误判**:被问"代管客户的货" 想做扩展库存表 → 标准货主维度已支持
- **来源**:[发货通知单](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%8F%91%E8%B4%A7%E9%80%9A%E7%9F%A5%E5%8D%95) · fetched 2026-04-23

### 仓库 / 仓位 / 货架
- **需求关键词**:"一个仓库下多个货位"、"货架编号"、"巷道-货架-层"
- **标准功能**:**仓库** → **仓位**(货位)两级;仓位通过 **仓位值集** 定义维度组合(例:巷道/货架/层/位);**货架是仓位值集的一个维度**,可独立禁用
- **启用路径**:基础资料 → 仓库 → 勾选"启用仓位管理"(勾了才需录仓位);基础资料 → 仓位值集 → 维度;基础资料 → 仓位
- **典型误判**:
  - "按货架管理" 想做扩展字段 → 仓位值集 N 维笛卡尔积
  - 启用仓位前**先确认是否勾选了"管易仓"**——勾了管易仓不允许启用仓位管理 <!-- 待实证 -->
- **OpenDeploy 工具覆盖**:✅ 元数据可读
- **来源**:[仓库](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E4%BB%93%E5%BA%93) · [仓位值集](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E4%BB%93%E4%BD%8D%E5%80%BC%E9%9B%86) · fetched 2026-04-23

## 2. 物料启用项(批号 / 序列号 / 保质期 / 辅助单位)

> **STK 模块最容易踩坑的一段**——以下都是物料档案上的勾选,启用后对应字段和流程**自动亮**,不是二开。

### 批号管理
- **需求关键词**:"按批次管理"、"按生产批次"、"按进货批次出库"、"批号追溯"
- **标准功能**:物料勾 **启用批号管理** 后,所有出入库单据自动出现 `批号` 字段;即时库存按批号汇总;支持**批号调整单**改批号 / 保质期 / 仓位 / 库存状态
- **启用路径**:基础资料 → 物料 → 库存属性页签 → **启用批号管理** + **批号编码规则**;[供应链] → [库存管理] → [批号调整单]
- **典型误判**:被问"按批次管理" 想加扩展字段记批次 → **直接勾启用项**,批号字段 + 即时库存汇总 + 调整单全套
- **来源**:[批号调整单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%89%B9%E5%8F%B7%E8%B0%83%E6%95%B4%E5%8D%95) · fetched 2026-04-23

### 序列号管理
- **需求关键词**:"每个产品一个唯一码"、"按机身号管理"、"序列号扫码"
- **标准功能**:物料勾 **启用序列号管理** 后,出入库按个录 SN;**未启用时 SN 按钮全隐藏**;**启用后有 SN 数据则不允许取消启用**(必须先清空)
- **启用路径**:基础资料 → 物料 → 库存属性页签 → 启用序列号管理 + 编码规则;系统参数 → 序列号系统参数
- **典型误判**:被要求"按 SN 管理" 想加 BOS 字段 → 启用项就有完整 SN 流程(发货 / 入库 / 盘点 / 出库逐个扫)
- **来源**:[序列号系统参数](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%BA%8F%E5%88%97%E5%8F%B7%E7%B3%BB%E7%BB%9F%E5%8F%82%E6%95%B0%E5%92%8C%E8%AE%BE%E7%BD%AE) · fetched 2026-04-23

### 保质期管理
- **需求关键词**:"保质期"、"近效期先出"、"过期预警"、"生产日期"
- **标准功能**:物料勾 **启用保质期管理** 后,出入库录 **生产日期 + 保质期**(单位 = 天 / 月 / 年),系统自动算 **有效期至**;支持 **近效期先出** 拣货规则(例:生产 2011.01.01,保质期 2 年 → 有效期至 2013.01.01)
- **启用路径**:基础资料 → 物料 → 库存属性页签 → 启用保质期管理 + 保质期单位 + 数值;通常配合批号启用
- **典型误判**:"过期预警" → 即时库存查询本身就有有效期至字段,条件查询 / 计划任务即可,**不需 Python**
- **来源**:[批号序列号关系](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%89%B9%E5%8F%B7%E5%BA%8F%E5%88%97%E5%8F%B7%E5%85%B3%E7%B3%BB) · fetched 2026-04-23

### 辅助单位 / 多计量单位
- **需求关键词**:"按箱按件"、"采购按吨入库按斤"、"换算率"
- **标准功能**:物料一个 **基本单位** + 多个 **辅助单位**;用 **换算分子/分母** 表达(例:1 箱 = 12 件 → 分子 12 / 分母 1);单据 `数量+计量单位` 与 `基本单位数量+基本计量单位` 双轨自动换算
- **启用路径**:基础资料 → 计量单位组;物料 → 单位换算 → 辅助单位 + 换算关系
- **典型误判**:被问"既能录箱又能录件" 想做计算插件 → 辅助单位标准双向换算
- **来源**:[单位换算处理](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%8D%95%E4%BD%8D%E6%8D%A2%E7%AE%97%E5%A4%84%E7%90%86) · [计量单位](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E8%AE%A1%E9%87%8F%E5%8D%95%E4%BD%8D) · fetched 2026-04-23

## 3. 调拨

### 直接调拨(组织内 / 跨组织一张单)
- **需求关键词**:"仓库 A 的货搬到仓库 B"、"跨组织调货简单流程"
- **标准功能**:**直接调拨单** 一张单完成调拨,**组织内**(库存组织+货主都不变)和**跨组织**(任一变);跨组织时调出/调入货主在同一核算体系下上级核算组织不同,支持**组织间结算**
- **启用路径**:[供应链] → [库存管理] → [库存调拨] → [直接调拨单]
- **典型误判**:被问"跨组织调货走采购+销售" → **直接调拨单更简洁**,有组织间结算闭环
- **OpenDeploy 工具覆盖**:✅ 元数据可读
- **来源**:[直接调拨单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%9B%B4%E6%8E%A5%E8%B0%83%E6%8B%A8%E5%8D%95) · [调拨](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E8%B0%83%E6%8B%A8) · fetched 2026-04-23

### 分步式调拨(申请 → 调出 → 调入)
- **需求关键词**:"调拨要审批"、"调出和调入分开做"、"在途库存"
- **标准功能**:**调拨申请单 → 分步式调出单(发货)→ 分步式调入单(收货)** 三段式,两端各自审单,期间数量进 **在途库存**
- **启用路径**:[供应链] → [库存管理] → [库存调拨] → 调拨申请单 / 分步式调出单 / 分步式调入单
- **典型误判**:被问"调拨要走审批+在途" 想自定义流程 → 标准分步式调拨就是这个
- **来源**:[调拨申请单](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E8%B0%83%E6%8B%A8%E7%94%B3%E8%AF%B7%E5%8D%95) · [分步式调出单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%88%86%E6%AD%A5%E5%BC%8F%E8%B0%83%E5%87%BA%E5%8D%95) · fetched 2026-04-23

### 部门间调拨
- **需求关键词**:"两个车间互相借料"、"领用部门换了"
- **标准功能**:**部门间调拨单** 处理部门维度调拨,不动仓库;[供应链] → [库存管理] → [库存调拨] → 部门间调拨单
- **来源**:[部门间调拨单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E9%83%A8%E9%97%B4%E8%B0%83%E6%8B%A8%E5%8D%95) · fetched 2026-04-23

## 4. 盘点

### 盘点方案 → 盘点单 → 盘盈 / 盘亏
- **需求关键词**:"盘库"、"盘盈盘亏"、"做实盘"
- **标准功能**:**库存盘点方案**(范围+规则)→ 生成 **盘点单**(填实盘)→ 自动生成 **盘盈单/盘亏单**;**盘盈数量 = 盘点数 - 账存数**(正盘盈,负盘亏)
- **启用路径**:[供应链] → [库存管理] → [盘点管理] → 库存盘点方案 / 盘点单 / 盘盈单 / 盘亏单
- **典型误判**:被要求"盘点生盈亏单" 想做转换插件 → 标准流程**自动**生成,方案里配开关
- **OpenDeploy 工具覆盖**:✅ 元数据可读
- **来源**:[库存盘点方案](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%BA%93%E5%AD%98%E7%9B%98%E7%82%B9%E6%96%B9%E6%A1%88) · [盘盈单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%9B%98%E7%9B%88%E5%8D%95) · fetched 2026-04-23

### 周期盘点 / 抽盘
- **需求关键词**:"按月分批盘"、"重要物料天天盘"、"循环盘点"、"ABC 盘点"
- **标准功能**:**周期盘点计划** 按物料重要度(ABC)按月/季循环盘点;支持**容差范围**——超容差自动纳入下次盘点
- **启用路径**:[供应链] → [库存管理] → [盘点管理] → 周期盘点计划;查询:物料周期盘点表
- **典型误判**:被问"分批循环盘" 想写排期 → 周期盘点计划标准,带 ABC 分类+容差重盘
- **来源**:[周期盘点计划](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%91%A8%E6%9C%9F%E7%9B%98%E7%82%B9%E8%AE%A1%E5%88%92) · fetched 2026-04-23

### 盘点方案高级参数
- **常见参数**(盘点方案上勾选):是否生成盘盈/盘亏单 / 实盘数默认值(0 / 账存数 / 空)/ 序列号默认为盘亏 / 禁用物料是否参与盘点 / 忽略库存单据审核状态检查
- **典型误判**:被问"序列号没扫到算什么" → 方案参数已配,**不需写插件**

## 5. 库存预警 / 库龄 / 呆滞物料

### 库存预警(安全库存 / 上下限)
- **需求关键词**:"低于最低库存提醒"、"安全库存"、"自动生成补货申请"
- **标准功能**:物料配 **安全库存 / 最低库存 / 最高库存**;**库存预警报表** 统计低于安全 / 加在途仍低于最小 / 高于最大的物料;预警可**按差异数量直接生成采购申请单**
- **启用路径**:基础资料 → 物料 → 库存属性 → 安全/最低/最高库存;[供应链] → [库存管理] → [库存预警] → 库存预警报表
- **典型误判**:
  - "预警生成采购单" 想做 Python → 报表上**直接有"生成采购申请"操作**
  - "实时弹窗预警" 想做插件 → 报表是查询,实时拦下单看"销售订单锁库"
- **来源**:[库存预警报表](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%BA%93%E5%AD%98%E9%A2%84%E8%AD%A6%E6%8A%A5%E8%A1%A8) · fetched 2026-04-23

### 库存账龄分析(库龄)
- **需求关键词**:"看库存压了多久"、"库龄分布"、"账龄段统计"
- **标准功能**:**库存账龄分析表** 按账龄段(0-30/31-60/...) 统计库存数量/金额分布;[供应链] → [库存管理] → [报表] → 库存账龄分析表
- **典型误判**:"做库龄报表" 想 SQL 写 → 标准报表已有,自定义账龄段即可
- **来源**:[库存账龄分析表](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%BA%93%E5%AD%98%E8%B4%A6%E9%BE%84%E5%88%86%E6%9E%90%E8%A1%A8) · fetched 2026-04-23

### 呆滞物料
- **需求关键词**:"找出呆滞料"、"长期不动的库存"
- **标准功能**:**呆滞料分析报表** 按呆滞天数阈值识别未动物料;处理走**其他出库单**(报损/转其他业务)
- **典型误判**:顾问要建独立"呆滞物料"扩展表 → 报表 + 现有出库单足够 <!-- 启用路径待用户实证 -->

- **需求关键词**:"不让库存变负数"
- **标准功能**:**库存负结余预警** 检测出库后库存为负;系统参数控制 允许 / 警告 / 阻止;库存管理系统参数 → 控制策略 → 负库存控制 <!-- 待实证 -->

## 6. 库存核算 / 出库成本

### 计价方法(加权平均 / 移动平均 / 先进先出 / 个别计价)
- **需求关键词**:"算成本"、"FIFO"、"加权平均"、"按批次成本"
- **标准功能**:**核算政策** 配置计价方法,支持 **加权平均 / 移动平均 / 先进先出 / 个别计价**;个别计价支持按 **批号 / 仓库 / 仓位 / BOM编号 / 辅助属性** 等维度
- **启用路径**:[供应链] → [存货核算] → [核算政策];核算组织+核算体系+会计政策三件套
- **典型误判**:被问"按批次算成本" 想写脚本 → 个别计价 + 核算维度勾批号即可
- **来源**:[出库成本核算](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%87%BA%E5%BA%93%E6%88%90%E6%9C%AC%E6%A0%B8%E7%AE%97) · [存货核算](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%AD%98%E8%B4%A7%E6%A0%B8%E7%AE%97) · fetched 2026-04-23

### 月末关账 / 反关账
- **需求关键词**:"月底封账"、"封了不让改"、"反关账"
- **标准功能**:**库存关账** 禁止录入/修改/反审核/作废**关账日期前**的出入库单,按**库存组织**关账;关账前**自动检查未审核未作废单据**,有则失败
- **启用路径**:[供应链] → [库存管理] → [期末处理] → 关账 / 反关账
- **典型误判**:"封账后改不了" 想加 BOS 字段控制 → 关账机制数据库层锁住,不需扩展
- **来源**:[关账_反关账](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%85%B3%E8%B4%A6_%E5%8F%8D%E5%85%B3%E8%B4%A6) · fetched 2026-04-23

## 7. 寄售业务(委托代销)

### 寄售类客户(出寄售给客户)
- **需求关键词**:"先发货后结算"、"代销"、"寄售给客户"、"客户卖了再算钱"
- **标准功能**:**寄售类销售订单** → 发货通知单 → **寄售类直接调拨单**(货发客户仓但所有权未转移)→ **寄售结算单**(客户卖了通知)→ 销售出库扣客户仓库存 → 应收单
- **启用路径**:基础资料 → 客户档案 → 客户类别 = **寄售类客户**(关键!);销售订单类型选寄售类
- **典型误判**:
  - "代销业务" 想做扩展字段标识 → 客户类别寄售 + 寄售订单 + 寄售结算单一整套标准
  - **客户类别错了走不通流程**——先确认是不是寄售类客户
- **来源**:[寄售业务](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%AF%84%E5%94%AE%E4%B8%9A%E5%8A%A1) · [寄售结算单](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%AF%84%E5%94%AE%E7%BB%93%E7%AE%97%E5%8D%95) · fetched 2026-04-23

### VMI(供应商寄售在我仓)
- **需求关键词**:"供应商把货放我仓"、"VMI 供应商管理库存"、"卖了再付钱"
- **标准功能**:**VMI 销售订单** + 货主 = 供应商;企业向外部客户销售后定期与供应商结算
- **启用路径**:基础资料 → 供应商 → 业务信息 → 作为货主;销售订单类型 = VMI
- **典型误判**:顾问被问"VMI" 想做扩展库存表 → 货主 = 供应商已支持

## 8. 直运 / 不入库直发

### 直运销售
- **需求关键词**:"供应商直接发给客户"、"我不进货"、"不过我仓库"、"三角贸易"
- **标准功能**:**销售订单(直运)→ 直运采购订单 → 供应商直接发客户**;**不产生入库/出库单据**,直接产生应收/应付;采购销售在企业内闭环结算
- **启用路径**:销售订单类型选 **直运销售**;订单审核后下推 **直运采购订单**
- **典型误判**:被问"不入库直发" 想做特殊流程 → 直运订单类型标准支持
- **来源**:[直运销售业务](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%9B%B4%E8%BF%90%E9%94%80%E5%94%AE%E4%B8%9A%E5%8A%A1) · [直运采购订单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%9B%B4%E8%BF%90%E9%87%87%E8%B4%AD%E8%AE%A2%E5%8D%95) · fetched 2026-04-23

### 生产直送(车间直发)
- **需求关键词**:"生产完不入库直接发"、"车间直发"、"省一步入库"
- **标准功能**:**生产直送** 处理生产完工产品**不经成品仓**直接转销售出库/下道工序;[生产制造] → [车间管理] → 生产直送 <!-- 启用路径待实证 -->
- **典型误判**:被要求"生产完直接发" 想自己拼流程 → 生产直送标准模块
- **来源**:[生产直送](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E7%94%9F%E4%BA%A7%E7%9B%B4%E9%80%81) · fetched 2026-04-23

## 9. 出入库类型 / 业务类型

### 单据类型(决定凭证模板 / 流程规则)
- **需求关键词**:"两种入库走不同账"、"特定业务要单独凭证"、"分类型管控"
- **标准功能**:出入库单(其他入库/出库/采购入库/销售出库/生产入库等)都有 **单据类型** 字段;**类型决定凭证模板+流程规则+业务行为**(例:退货类型的其他出库单**反向**更新即时库存——增加而不是减少)
- **启用路径**:基础资料 → 单据类型;凭证模板:[财务会计] → [总账] → [凭证模板]
- **典型误判**:
  - "两种业务不同凭证" 想 BOS 拆两对象 → **新增一个单据类型 + 一套凭证模板** 即可,不拆对象
  - **退货类型 vs 普通类型** 同对象不同行为——`其他出库单(退货)` 实际是入库;**搞错类型即时库存方向反**
- **来源**:[单据类型](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%8D%95%E6%8D%AE%E7%B1%BB%E5%9E%8B) · [其他出库单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%85%B6%E4%BB%96%E5%87%BA%E5%BA%93%E5%8D%95) · [凭证模板](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%87%AD%E8%AF%81%E6%A8%A1%E6%9D%BF) · fetched 2026-04-23

## 10. 即时库存 / 库存状态 / 锁库

### 即时库存维度
- **需求关键词**:"看现在多少货"、"实时库存"
- **标准功能**:**即时库存表** 按 **物料 + 库存组织 + 仓库 + 仓位 + 货主 + 保管者 + 批号 + 保质期 + 库存状态 + BOM编号 + 辅助属性** 多维汇总;**即时库存查询** 自由组合筛选
- **启用路径**:[供应链] → [库存管理] → 即时库存查询 / 即时库存汇总数据查询
- **典型误判**:被问"按某维度看库存" 想做新报表 → 即时库存查询筛选/分组已是 N 维数据立方体
- **来源**:[即时库存](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%8D%B3%E6%97%B6%E5%BA%93%E5%AD%98) · fetched 2026-04-23

### 库存状态(可用 / 待检 / 冻结 / 退货冻结 / 在途 / 收货冻结)
- **需求关键词**:"待检验的货不能发"、"冻结一批货"、"质量异常先扣住"
- **标准功能**:仓库定义可存放的库存状态类型;出库默认只取"可用"状态库存;基础资料 → 仓库 → 可用库存状态(多选)
- **典型误判**:被问"质检中的货不发" 想做扩展字段 → 库存状态=待检 + 出库默认过滤即可

### 锁库 / 销售订单预留
- **需求关键词**:"VIP 客户的货留着"、"预留库存"、"先锁住别给别人"
- **标准功能**:**销售订单锁库**——销售订单/即时库存执行锁定/解锁;**仓库+库存状态都标"可锁"才能锁**;锁库后不参与其他单据下推匹配
- **启用路径**:销售订单 → 操作 → 锁库/解锁;[供应链] → 即时库存查询 → 选记录 → 锁定
- **典型误判**:"给某客户留货" 想做 BOS 字段 → 销售订单锁库 + 物料预留网络(MFG 字典里)标准支持
- **来源**:[销售订单锁库和预留](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E9%94%80%E5%94%AE%E8%AE%A2%E5%8D%95%E9%94%81%E5%BA%93%E5%92%8C%E9%A2%84%E7%95%99) · fetched 2026-04-23

### 业务冻结
- **需求关键词**:"冻结记录不让操作"、"暂停业务处理"
- **标准功能**:销售订单等单据支持**业务冻结**,冻结后记录不进入后续业务匹配;单据列表 → 选记录 → 冻结 / 解冻

## 11. 组装 / 拆卸单

### 组装 / 拆卸(库存层套件)
- **需求关键词**:"散件组成套件"、"成品拆成零件"、"重新打包"、"礼盒套装"
- **标准功能**:**组装拆卸单** 处理库存层组装/拆卸:
  - **组装**:子件库存减,套件库存增,成本 = 子件成本之和+组装费
  - **拆卸**:套件库存减,子件库存增,套件成本按比例分摊到子件
- **启用路径**:[供应链] → [库存管理] → [组装拆卸] → 组装拆卸单
- **典型误判**:被问"散件包套件" 想走生产订单 → **不进车间** 的组装拆卸用本单据,轻量
- **来源**:[组装拆卸单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%BB%84%E8%A3%85%E6%8B%86%E5%8D%B8%E5%8D%95) · fetched 2026-04-23

## 12. PDA / WMS 集成(谨慎)

> 🔴 **资料不足**——以下基于公开手册片段,客户环境是否能开箱即用 / 需要单独购买 / 走第三方 WMS 都待**用户实证**。

### 条码管理
- **需求关键词**:"扫码出入库"、"条码标签"
- **标准功能**:**条码管理** 贯穿原材料入库 → 生产领用 → 工序跟踪 → 成品入库 → 销售出库 → 盘点全流程 <!-- 具体启用路径待实证 -->
- **典型误判**:不要承诺"开箱即用条码"——**先确认客户买了条码模块 + 配了打印机/扫码枪**

### PDA 端(扫码 + 报工 + 出入库)
- **需求关键词**:"PDA 扫码"、"现场移动操作"、"工业 PDA"
- **标准功能**:PDA 工业级扫描设备走 WIFI 实时数据交互;支持 PDA 报工 / 出入库 / 盘点等;**PDA 是单独客户端** + 服务端 PDA 界面配置 <!-- V9 是否标准内置待实证,通常配套智慧车间 MES/仓储模块,可能需单独购买 -->
- **典型误判**:
  - 不承诺"PDA 开箱即用"——**先确认客户 PDA 硬件 + K/3 Cloud PDA 模块授权**
  - "自研 PDA"通常走 Web API 集成而不是标准 PDA,**问清是哪一种**
- **来源**:[pda界面配置](https://help.open.kingdee.com/dokuwiki/doku.php?id=pda%E7%95%8C%E9%9D%A2%E9%85%8D%E7%BD%AE) · fetched 2026-04-23

### WMS 集成
- **需求关键词**:"对接 WMS"、"第三方仓储"、"立库 / AGV"
- **标准功能**:K/3 Cloud 没有自带"WMS"模块名,但**库存管理 + 仓位 + 条码 + PDA + 拣货策略** 组合起来是事实上的 WMS 能力;第三方 WMS 集成靠开放平台 OSF / Webapi <!-- 具体方案待客户场景定 -->
- **典型误判**:不要说"K/3 Cloud 内置 WMS"——表述应是"标准库存模块 + 配套 PDA/条码模块组合实现 WMS 场景"
