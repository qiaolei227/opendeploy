# 报表平台 / 智能分析 标准功能字典

> 报表领域是"动手前先分类"的重灾区——客户一句"给我做张报表"可能对应**5 种完全不同**的产品形态。本字典帮顾问识别"客户要的是 [内置标准报表 / 账表设计 / 自定义报表 / 智能分析(BI) / 二开] 哪一类",**多数情况下不需要写 SQL,更不需要二开**。

## 决策骨架(先看这里再往下读)

| 客户描述 | 99% 是这个 | 启用代价 |
|---|---|---|
| "我要看资产负债表 / 利润表 / 现金流量表" | **报表系统**(预置模板) | 编制方案配一次,自动出表 |
| "看销售/采购/库存/应收账龄汇总" | **模块内置账表**(每个模块菜单下都有) | 点开就能看 |
| "已有的报表多一列 / 改个过滤条件" | **过滤方案**(用户自己存) 或 **账表设计**(BOS) | 顾问 0~半天 |
| "我要一张全新维度的明细 + 汇总表" | **账表设计**(简单账表 / 透视表 / SQL 账表) | 顾问 0.5~2 天,**不写代码** |
| "老板要看大盘 / 多维度交叉分析 / 钻取" | **经营分析平台**(财务分析 8 张预置 BI) 或 **看板定义** | 配置类 |
| "报表里要根据用户角色动态改数据 / 调外部接口" | **报表插件**(C# DLL,IReportPlugIn) | 真二开 |

> 默认假设:**用户描述里不出现"按钮"、"动态计算"、"调用 XX 系统"这种词,基本就不用插件**。先按上面的表对号入座。

## 1. 内置业务报表(零成本,先确认有没有)

K/3 Cloud 每个业务模块自带一套**账表(Account Reports)**,菜单基本固定在 `<模块> → 报表` 下。**先让客户在系统里看一眼,大概率已经有了**。

### 财务三大表 / 五大表
- **需求关键词**:"资产负债表"、"利润表"、"现金流量表"、"所有者权益变动表"、"附注"
- **标准功能**:**报表系统** 预置三大表 / 五大表模板;通过 **报表自动生成方案** 按期取数
- **启用路径**:总账 → 报表 → 报表管理 → 报表模板;首次需配 **报表自动生成方案**(批量生成多期)
- **典型误判**:被问"能不能给我一张资产负债表" → 不是要二开,是预置模板里就有,**只需配一次取数公式参数**(账簿、币别、期间、ACCT 公式参数)
- **OpenDeploy 工具覆盖**:❌ 报表模板配置走客户端 BOS / 报表设计器,产品不参与
- **来源**:[报表系统 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E6%8A%A5%E8%A1%A8%E7%B3%BB%E7%BB%9F)

### 销售业务报表
- **需求关键词**:"销售出库汇总"、"销售订单跟踪"、"销售明细 by 客户/物料/销售员"
- **标准功能**:销售模块内置销售出库汇总表、销售订单执行情况表、销售明细表等;支持按维度切换
- **启用路径**:销售管理 → 销售报表
- **典型误判**:"我要按销售员看月度回款" → 应收账龄 + 销售明细两张报表都已有,过滤方案搞定
- **OpenDeploy 工具覆盖**:❌

### 采购 / 库存 / 应收应付业务报表
- **需求关键词**:"采购入库明细"、"在途/在库 / 收发存"、"应收/应付明细 / 账龄"
- **标准功能**:各模块均有标配账表(收发存汇总表、采购明细表、应收账龄分析表 等)
- **启用路径**:对应模块 → 报表
- **典型误判**:报表的列不全 → 优先看 **账表设计**(下文),不要直接 SQL
- **OpenDeploy 工具覆盖**:❌

## 2. 账表设计(BOS 设计器内,**不写代码也能新建报表**)

`账表设计 = K/3 Cloud 的"低代码报表"`。**90% 的"我要张新报表"用这个解决,不用 SQL,不用插件**。

### 简单账表 / 简单报表 🟡
- **需求关键词**:"以基础资料/单据为来源,加几列汇总"
- **标准功能**:**简单账表**(BOS 设计器新建)直接显示报表数据;**简单报表** 三种子类型:明细 / 汇总 / 交叉,以基础资料、业务单据为数据源,通过向导配字段
- **启用路径**:BOS 设计器 → 工具栏 **【新建】** → 业务对象新建向导 → 选 **简单账表 / 简单报表**;或在单据右键 **【新建报表】**
- **典型误判**:"我要张新报表" → 优先简单账表,设计器里拖几下,不用动 SQL
- **OpenDeploy 工具覆盖**:❌(BOS 报表元数据写入不在 v0.1 范围)
- **来源**:[账表设计 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E8%B4%A6%E8%A1%A8%E8%AE%BE%E8%AE%A1)

### 树形账表 / 分页账表 🟡
- **需求关键词**:"按部门/组织层级展开"、"上万条要分页"
- **标准功能**:**树形账表** 支持以树形结构显示;**分页账表** 大数据量分页加载
- **启用路径**:同上,业务对象新建向导里选对应子类型
- **典型误判**:"组织树展开看下属数据" → 树形账表标配,不用插件
- **OpenDeploy 工具覆盖**:❌
- **来源**:[账表设计 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E8%B4%A6%E8%A1%A8%E8%AE%BE%E8%AE%A1)

### 直接 SQL 账表 🟡
- **需求关键词**:"跨多个单据汇总,关系复杂"、"已有 SQL 想直接出报表"
- **标准功能**:**直接 SQL 账表** —— 写 SQL(可带关键字 / 参数化过滤)直接渲染为账表
- **启用路径**:BOS 设计器 → 新建账表 → 选 **直接 SQL 账表** → 维护 SQL + 关键字
- **典型误判**:写 Python 插件去拼数据 → SQL 账表能搞定就别插件
- **架构红线**:产品 agent **不主动写客户业务 SQL**(数据红线在 CLAUDE.md);告诉用户"自己/IT 写 SQL,贴进 BOS 直接 SQL 账表"
- **OpenDeploy 工具覆盖**:❌
- **来源**:[账表设计 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E8%B4%A6%E8%A1%A8%E8%AE%BE%E8%AE%A1)

### 透视表 🟡
- **需求关键词**:"按客户 × 月份交叉看销量"、"Excel 数据透视图那种"
- **标准功能**:**透视表** —— 类似 Excel 透视图的交叉分析报表
- **启用路径**:BOS 设计器 → 新建账表 → 选 **透视表**
- **典型误判**:导出 Excel 自己拉透视 → 直接做透视账表,客户登录就能看
- **OpenDeploy 工具覆盖**:❌
- **来源**:[账表设计 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E8%B4%A6%E8%A1%A8%E8%AE%BE%E8%AE%A1)

## 3. 报表取数公式(传统报表系统的核心)

报表系统(资产负债表 / 利润表那一套)靠**取数公式**取数,**不是数据集 / SQL**。配置在单元格里,类似 Excel 的 `=ACCT(...)`。

### ACCT 系列 —— 总账取数主力
- **需求关键词**:"资产负债表里某科目的期末数 / 当期发生额"
- **标准功能**:`ACCT` 公式从科目余额表取数;支持账簿 / 科目 / 取数类型(`C` 期初 / `Y` 期末 / `JF` 借方 / `DF` 贷方 / `BG` 预算 / `BD` 预算差异)/ 年度 / 期间(相对值 `0` 当期、`-1` 上期)/ 币别
- **启用路径**:报表设计器 → 单元格 → 公式编辑 → 选 ACCT;参数向导填
- **典型误判**:"我要从凭证按月汇总数据" → ACCTEXT 公式是"按日从凭证取数",不用 SQL
- **OpenDeploy 工具覆盖**:❌
- **来源**:[报表取数公式 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%8A%A5%E8%A1%A8%E5%8F%96%E6%95%B0%E5%85%AC%E5%BC%8F)

### 现金流 / 阿米巴 / 项目类公式 🟡
- **公式名**:`ACCTCASH`(现金流量项目取数)/ `ACCTITEM`(按项目取数)/ `AMBAcct` `AMBJSJM`(阿米巴报表)
- **典型误判**:"现金流量表能不能自动出" → 启用现金流量项目 + ACCTCASH 公式后自动出,不需要二开
- **OpenDeploy 工具覆盖**:❌
- **来源**:[报表取数公式 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%8A%A5%E8%A1%A8%E5%8F%96%E6%95%B0%E5%85%AC%E5%BC%8F)

### 跨表 / 跨报表 / 元数据 公式 🟡
- **公式名**:`REF`(跨表页取数)/ `REF_F`(跨报表取数,合并 / 汇总报表常用)/ `RPTDATE`(报表日期)/ `RPTINFO`(报表元数据,页眉页脚)
- **典型误判**:"合并多个分公司的报表" → REF_F 跨报表取数标配
- **OpenDeploy 工具覆盖**:❌
- **来源**:[报表取数公式 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%8A%A5%E8%A1%A8%E5%8F%96%E6%95%B0%E5%85%AC%E5%BC%8F)

### 数字聚合 / 单元格四则运算
- **功能**:`SUM` / `MAX` / `MIN` / `AVERAGE` / `COUNT`,以及单元格直接 `B5+D6` 这种四则运算
- **启用路径**:单元格里直接录公式,完全 Excel 体验
- **OpenDeploy 工具覆盖**:❌
- **来源**:[报表取数公式 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%8A%A5%E8%A1%A8%E5%8F%96%E6%95%B0%E5%85%AC%E5%BC%8F)

## 4. 自定义报表 / 过滤方案(用户自己玩,顾问指路)

### 自定义报表 🟡
- **需求关键词**:"用户自己改报表的列 / 排序 / 分组"
- **标准功能**:**自定义报表** 在报表菜单下,用户基于已有账表自定义视图(列 / 排序 / 过滤),保存后多人复用
- **启用路径**:报表 → **自定义报表** 入口;或在任一账表界面 → 工具栏自定义 → 保存为方案
- **典型误判**:"用户希望默认就看到他自己关心的列" → 让用户存自己的方案,不用顾问做
- **OpenDeploy 工具覆盖**:❌
- **来源**:[报表系统 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E6%8A%A5%E8%A1%A8%E7%B3%BB%E7%BB%9F)

### 过滤方案 🟡
- **需求关键词**:"每次打开报表想自动带某个组织 / 物料分类"、"销售员只看自己的客户"
- **标准功能**:K/3 Cloud 通用 **过滤方案**(每个账表 / 报表都支持),保存为方案后可设为默认 / 共享给角色
- **启用路径**:报表界面 → 过滤条件 → 保存为方案 → 可勾"默认方案"或"共享给角色"
- **典型误判**:"销售员看不到别人客户" → 过滤方案 + 数据权限组合,不用插件
- **OpenDeploy 工具覆盖**:❌
- **来源**:[报表系统 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E6%8A%A5%E8%A1%A8%E7%B3%BB%E7%BB%9F)

### 项目数据查询 / 动态项目数据查询 🟡
- **需求关键词**:"按项目核算维度看数据"
- **标准功能**:报表系统的 **项目数据查询** + **动态项目数据查询**,基于报表项目维度
- **启用路径**:报表 → 项目数据查询
- **OpenDeploy 工具覆盖**:❌
- **来源**:[报表系统 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E6%8A%A5%E8%A1%A8%E7%B3%BB%E7%BB%9F)

## 5. 智能分析平台 / BI / 看板

### 财务分析(8 张预置 BI 主题)🟡
- **需求关键词**:"老板要看经营大盘"、"毛利率 / 净利润率 / 资产负债率仪表盘"
- **标准功能**:经营分析 → 财务分析 模块预置 **8 张分析主题**:
  | 主题 | 内容 |
  |---|---|
  | 经营大盘 | 毛利率、净利润率、资产负债率等 KPI |
  | 损益分析 | 营收 / 成本 / 利润穿透 |
  | 资产负债分析 | 资产 / 负债 / 权益结构 + 偿债能力 |
  | 经营效率 | 现金周转天数、存货周转天数 |
  | 资金分析 | 资金流量与余额 |
  | 应收分析 | 客户欠款 + 信用 |
  | 费用分析 | 期间费用结构 |
  | 科目多维度分析 | 科目 × 维度交叉钻取 |
- **启用路径**:经营分析 → 财务分析 → 财务主题
- **典型误判**:"老板要看大盘" → 8 张已经预置,二开做仪表盘是浪费;企业可基于预置做"个性化定制"
- **OpenDeploy 工具覆盖**:❌
- **来源**:[财务分析 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E8%B4%A2%E5%8A%A1%E5%88%86%E6%9E%90)

### 指标设计平台(BI 建模)🟡
- **需求关键词**:"我要个跨多个单据的实时指标"、"指标体系 / 数据中台轻量版"
- **标准功能**:**指标设计平台** 以星空单据 + 报表为数据源,通过执行计划 **每天更新写入宽表**;基于宽表建模(维度 + 指标)做主题配置
- **启用路径**:经营分析 → 指标设计平台
- **典型误判**:"我们要做数据中台" → 先看指标设计平台够不够,够就别上外部 BI
- **架构注意**:数据是**异步 T+1**(宽表每天执行计划更新),不是实时;实时类需求别选这个
- **OpenDeploy 工具覆盖**:❌
- **来源**:[指标设计平台 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%8C%87%E6%A0%87%E8%AE%BE%E8%AE%A1%E5%B9%B3%E5%8F%B0)

### 看板定义(车间 / 监控类可视化)🟡
- **需求关键词**:"车间大屏"、"实时生产看板"、"异常监控大屏"
- **标准功能**:**看板定义** 三种数据获取方式可选:**自定义 SQL / 存储过程 / 自定义插件**;BOSIDE 建立结果元数据定义中间结构
- **启用路径**:智慧车间 MES → 智慧车间 → 电子看板 → **看板定义**(预置 7 类:订单工序进度 / 发料 / 异常 / 质量 等)
- **典型误判**:"做个生产大屏" → 预置 7 类先看一遍,不够再选 SQL / 存储过程,**最后才考虑插件**
- **架构红线**:看板"自定义 SQL"是客户 IT 自己写,不是 OpenDeploy agent 写
- **OpenDeploy 工具覆盖**:❌
- **来源**:[看板定义 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E7%9C%8B%E6%9D%BF%E5%AE%9A%E4%B9%89)

### 数据智能服务(公有云专属)🔴
- **需求关键词**:"风险预警"、"销售预测"、"AI 决策"
- **范围说明**:金蝶云星空**公有云客户**专属,融合大数据 + 模型,提供业务洞察 / 风险预警 / 预测 / 智能决策

- **OpenDeploy 工具覆盖**:❌

## 6. 报表导出 / 邮件 / 调度

### 报表导出
- **功能**:所有账表 / 报表内置 **导出 Excel / PDF**(工具栏标准按钮)
- **典型误判**:"能不能导出" → 永远能,不用问

### 报表邮件发送 🟡
- **需求关键词**:"日报 / 月报自动发邮箱"
- **标准功能**:报表系统支持**邮件发送 + 日志**(`报表邮件发送日志`);依赖 **邮件服务器设置** 和 **邮件虚拟账户**
- **启用路径**:
  - 邮件服务器:基础管理 → 公共设置 → 邮件设置 → **邮件服务器设置**
  - 虚拟账户:同上 → **邮件虚拟账户设置**(不配则需每个用户用自己邮箱发)
  - 报表推送:报表 → 基础资料 → **报表推送邮件设置**
- **典型误判**:"老板要每天早上自动收日报" → 配邮件 + 推送方案,不写代码
- **OpenDeploy 工具覆盖**:❌
- **来源**:[报表系统 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E6%8A%A5%E8%A1%A8%E7%B3%BB%E7%BB%9F) · [邮件服务器设置 · fetched 2026-04-23](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E9%82%AE%E4%BB%B6%E6%9C%8D%E5%8A%A1%E5%99%A8%E8%AE%BE%E7%BD%AE)

### 报表自动生成方案 🟡
- **需求关键词**:"每月初自动跑出所有期间的资产负债表"
- **标准功能**:**报表自动生成方案** —— 批量生成多账簿 / 多期间 / 多报表
- **启用路径**:报表 → 报表管理 → **报表自动生成方案**

## 7. 报表插件(C# DLL,真二开,**不是 Python**)

> ⚠️ K/3 Cloud 报表插件是 **C# (`IReportPlugIn` / `AbstractDynamicFormPlugIn`),不是 Python 表单插件**。OpenDeploy v0.1 **不支持 DLL 二开**,只能口头指路;Python 插件覆盖的是**表单**,不是报表。

### 报表插件适用场景 🟡
- **需求关键词**:"报表里某列要根据当前用户角色改算法"、"过滤后调外部 HTTP 接口校验数据"、"报表显示前数据要做复杂业务转换"
- **标准功能**:报表插件覆盖事件:`FilterChanged`(过滤改变)/ `BeforeQuery`(查询前)/ `AfterQuery`(查询后)/ `BarItemClick`(工具栏按钮)等
- **启用路径**:BOS 设计器 → 账表元数据 → 注册 C# 插件类(.dll 上传服务器)
- **典型误判**:"报表多个动态列" → 优先 **直接 SQL 账表** + 关键字参数;真做不到再插件
- **决策红线**(参考 `solution-decision-framework`):
  - ✅ 用户是否描述"按钮 / 动作 / 调用外部"? → 才考虑插件
  - ❌ 仅仅"列不够 / 过滤要变" → 账表设计 / 过滤方案能搞定
- **OpenDeploy 工具覆盖**:❌(C# DLL 二开非 v0.1 范围)
- **跨主题**:细节参见 `bos-features-index/references/plugin-types`(若有 report 类条目)
