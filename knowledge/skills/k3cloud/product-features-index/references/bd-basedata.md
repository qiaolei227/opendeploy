# 基础资料(BD)标准功能字典

基础资料是所有业务单据的前置依赖。**先把基础资料的标准字段查清楚,很多"加字段"需求其实标准已有**。

---

## 主要基础资料

### 客户(`T_BD_CUSTOMER`)
- **标准字段**(常见):客户编码、名称、简称、类别、行业、客户分组、销售员、价目表、信用额度、税率、发票抬头、税号、开户行、账号、地址、联系人、电话、邮箱、备注、核算属性(外币币别 / 结算方式)、应收科目、控制策略
- **"以为要扩展字段"的典型**:
  - "客户信用额度" → `FCreditLimit` 标准有
  - "默认销售员" → `FSellerId` 标准有
  - "客户分类" → `FGroup` 标准有,层级多的可以加分组
  - "客户业务员电话" → 联系人子单据体已有
- **扩展入口**:客户档案 → 业务信息 / 财务信息 / 其他页签。**字段齐全,先查再说扩展**

### 物料(`T_BD_MATERIAL`)
- **标准字段**:物料编码、名称、规格型号、计量单位、物料属性、物料分组、仓库、默认供应商、安全库存、采购周期、物料分类、采购单位、销售单位、辅助单位、批号管理、序列号管理、保质期管理、启用仓位、价格(计划价 / 标准价)、税率
- **"以为要加的字段" vs 标准启用**:
  - "按批次管理" → 启用 `FIsBatchManage`
  - "按序列号管理" → 启用 `FIsSNManage`
  - "保质期" → 启用 `FIsKFPeriod` + 保质期天数
  - "辅助单位 / 换算" → 辅助属性字段
- **扩展入口**:物料档案 → 10 多个页签,字段丰富

### 供应商(`T_BD_SUPPLIER`)
- 对称客户的结构:编码、名称、类别、默认采购员、信用额度、付款条件、税率、发票信息、银行、联系人
- **"以为要加的"**:付款条件、税率、币别 → 都标准有

### 仓库(`T_BD_STOCK`)
- 仓库编码、名称、组织、类型(原材料 / 成品 / 半成品 …)、启用仓位、保管员

### 部门 / 员工
- 多组织支持,员工可归属多部门

## 组织架构

K/3 Cloud 有 **6 种组织类型**,一个业务实体可以兼任:

| 组织类型 | 含义 |
|---|---|
| 法人组织 | 独立法人,有工商注册 |
| 核算组织 | 独立核算财务 |
| 利润中心 | 独立核算利润 |
| 成本中心 | 独立核算成本 |
| 销售组织 | 独立签销售合同 |
| 采购组织 | 独立签采购合同 |
| 库存组织 | 独立管理库存 |

**"以为要定制多公司架构"** → 99% 是标准组织配置能搞定。

---

## 基础资料公共特性

以下每个基础资料都支持,**不用二开**:

1. **多组织分配**:基础资料创建时归属"创建组织",通过"分配"让其他组织能用
2. **多级分组 / 分类**(FGroup 字段)
3. **审批流**(配置)
4. **禁用 / 启用**(FForbidStatus)
5. **自定义字段**(BOS 扩展)——这个才是"加字段"应该走的路径
6. **基础资料过滤方案**:不同用户看到的列表可以不同

---

## 典型"以为要二开"

| 顾问描述 | 真实答案 |
|---|---|
| "客户要分等级" | 客户分组 / 分类标准有 |
| "物料按厂家分组" | 物料分组支持多层级 |
| "仓库按区域分组" | 仓库属性 + 分组 |
| "基础资料审批" | 审批流配置 |
| "不同人看不同客户" | 基础资料权限 + 数据权限 |
| "客户编码按规则自动生成" | **编码规则** 基础资料,可配自动编码 |

---

## 编码规则

- **需求关键词**:"客户编码自动生成"、"订单号按日期+流水"
- **标准功能**:**编码规则** 基础资料——按前缀 + 日期 + 流水 + 分类规则自动生成单据号 / 基础资料编码
- **启用路径**:系统服务 → 编码规则

**这是标准实施必会的基础配置**,顾问写 Python 生成编码 = 浪费时间。

---

## 多级审批

基础资料的新建 / 修改 / 禁用 / 删除 都可以走审批流,开启后会多一个"已提交"状态。

**启用路径**:单个基础资料类型 → 系统参数 → 审批设置

---

## 小结

如果用户需求是"给 XXX 基础资料加字段 / 加分类 / 加审批",**90% 是标准配置 + BOS 扩展 + 审批流设计**,不是写插件。
