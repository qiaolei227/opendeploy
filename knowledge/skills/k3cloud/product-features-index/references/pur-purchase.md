# 采购管理(PUR)标准功能字典 🟡

> **定位**:实施顾问最常被"二开冲动"骗到的模块之一。客户一开口"想要个采购订单按钮 / 加个供应商比价 / 自动下采购单 / 三单匹配",九成情况答案是"K/3 Cloud 标准就有"。
>
> 本文按金蝶官方公开手册(`help.open.kingdee.com/dokuwiki` 与 `dokuwiki_std`)从**实施顾问视角**反查"误判提醒+功能定位"。**未在客户环境逐条实测**——版本差异(V8 / V9 各版次)以现场为准。带 `<!-- 待实证 -->` 的条目不要凭印象补。

## 0. 主流程速览(读其他章节前先扫一眼)

```
采购申请 → 询价单 → 比价单 → 采购合同 → 采购订单 → 收料通知单 → 采购入库单 → 应付单 → 付款申请 → 付款单
                                                  ↘ 来料检验单(可选)
```

每步可独立启用 / 关闭 / 跳过,由 **采购管理系统参数**(供应链 → 采购管理 → 系统设置 → 系统参数)控制(典型开关:是否强制走收料通知 / 是否启用三单匹配 / 询比价是否必需)。

来源:[采购管理系统参数](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E9%87%87%E8%B4%AD%E7%AE%A1%E7%90%86%E7%B3%BB%E7%BB%9F%E5%8F%82%E6%95%B0) · fetched 2026-04-23

## 1. 采购合同

### 三种合同类型(普通 / 框架 / 单价)
- **需求关键词**:"和供应商签框架"、"年度框架协议"、"长期单价锁定"、"按合同分批采购"
- **标准功能**:K/3 Cloud 采购合同支持三类型,**类型决定了控制力度**
  - **普通合同** — 录入物料明细,按数量控制,累计订单数量达合同数量自动业务关闭
  - **框架合同** — **无物料明细**,只指定合同总金额,典型为年度战略框架
  - **单价合同** — 录物料明细 + 单价,允许累计超额(柔性控制),长期采购首选
- **启用路径**:供应链 → 采购管理 → 合同管理 → **采购合同**(单据头有"合同类型"字段)
- **典型误判**:顾问被问"年度签的框架协议怎么管" 想 BOS 加扩展字段 — **直接选框架合同类型**,系统已经按"无明细 + 总金额"运作
- **OpenDeploy 工具覆盖**:✅ 元数据可读(`kingdee_describe_object` 查 `PUR_Contract` <!-- 对象代码待实证 -->)
- **来源**:[采购合同](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E9%87%87%E8%B4%AD%E5%90%88%E5%90%8C) · fetched 2026-04-23

### 合同累计数量 / 金额回写
- **需求关键词**:"合同执行了多少"、"还能下多少订单"、"合同剩余额度"
- **标准功能**:采购订单审核时,订单数量自动累加至 **采购合同 → 累计订单数量** 字段;反审核扣减;累计达合同数量时自动业务关闭。普通合同控数量,框架合同控金额
- **启用路径**:自动联动,无需配置;查询走 **采购合同执行明细表**
- **典型误判**:写 SQL 自己算"合同剩余" → 标准回写字段 + 内置报表已经覆盖
- **来源**:[采购合同执行明细表](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E9%87%87%E8%B4%AD%E5%90%88%E5%90%8C%E6%89%A7%E8%A1%8C%E6%98%8E%E7%BB%86%E8%A1%A8) · fetched 2026-04-23

### 合同状态机
- **需求关键词**:"合同到期关闭"、"提前终止"
- **标准功能**:**业务关闭**(自动 / 手动 + 反关闭)和 **业务终止**(手动 + 反终止)两条独立状态;关闭后下游不可再下推
- **启用路径**:采购合同 → 操作菜单 → 业务关闭 / 业务终止
- **典型误判**:被问"合同关单按钮" → 标准已有,不需要二开

## 2. 采购申请 → 询价 → 比价 → 选源

### 采购申请单(申请来源)
- **需求关键词**:"使用部门提需求"、"采购申请"、"计划申请采购"
- **标准功能**:由使用 / 计划部门发起,经审核后转采购订单 / 询价单。来源可手工 / MRP 计划订单下推 / **配套材料采购申请**(按销售单一键算缺料)
- **启用路径**:供应链 → 采购管理 → **采购申请单**;批量算缺料走 **配套材料采购申请**
- **典型误判**:顾问被问"按销售订单算缺料并提采购申请" 上来想 SQL — 配套材料采购申请就是干这个,**不用 MRP 全跑**
- **OpenDeploy 工具覆盖**:✅ 元数据可读(`PUR_Requisition` <!-- 对象代码待实证 -->)
- **来源**:[采购申请单](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E9%87%87%E8%B4%AD%E7%94%B3%E8%AF%B7%E5%8D%95) · [配套材料采购申请](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E9%85%8D%E5%A5%97%E6%9D%90%E6%96%99%E9%87%87%E8%B4%AD%E7%94%B3%E8%AF%B7) · fetched 2026-04-23

### 询价单 → 报价单 → 比价单
- **需求关键词**:"多供应商比价"、"询价流程"、"招标式采购"
- **标准功能**:三张单据闭环
  - **询价单** — 询比价起点,设定适用供应商;手工录或采购申请下推
  - **供应商报价单** — 供应商响应(可走供应商协同门户由供应商在线填)
  - **比价单** — 系统按含税单价从低到高排序,选出采纳价格 + 供应商;比价单审核后**自动回写询价单和报价单**
- **启用路径**:供应链 → 采购管理 → **询价单 / 报价单 / 比价单**
- **典型误判**:被问"想要个'比价表'看哪家便宜" 想报表二开 — 比价单本身就是这个表 + 还能直接选源
- **OpenDeploy 工具覆盖**:✅ 元数据可读
- **来源**:[询价单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E8%AF%A2%E4%BB%B7%E5%8D%95) · [比价单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%AF%94%E4%BB%B7%E5%8D%95) · fetched 2026-04-23

### 货源清单 / 供应商物料价格
- **需求关键词**:"哪些供应商能供这个料"、"物料指定供应商"、"准入物料"
- **标准功能**:**货源清单管理** 维护"物料 ↔ 供应商"准入关系;**采购价目表** 按 物料 / 供应商 / 数量段 / 时间段 维护标准价
- **启用路径**:供应链 → 采购管理 → 基础资料 → **货源清单管理** / **采购价目表**
- **典型误判**:顾问看到"客户某些料只能这家供" 想 Python 校验 — **货源清单已经做了这层准入约束**
- **来源**:[货源清单管理](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E8%B4%A7%E6%BA%90%E6%B8%85%E5%8D%95%E7%AE%A1%E7%90%86) · [采购价目表](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E9%87%87%E8%B4%AD%E4%BB%B7%E7%9B%AE%E8%A1%A8) · fetched 2026-04-23

## 3. 采购订单

### 订单类型(普通 / 寄售 / 直运 / VMI / 资产)
- **需求关键词**:"我们有几种采购怎么办"、"寄售订单"、"直运给客户"、"采固定资产"
- **标准功能**:订单**类型**字段切换,同一对象覆盖多场景
  - **普通采购订单** — 入企业仓库
  - **VMI 采购订单** — 入库 = 暂存,所有权仍在供应商,**物权转移单** 才确认应付
  - **直运采购订单** — 货直接发给最终客户,不进企业仓
  - **资产采购订单** — 走资产模块的固定资产采购流
- **启用路径**:供应链 → 采购管理 → 订单处理 → **采购订单**(单据头"单据类型"切换);资产专用走 **资产采购订单**
- **典型误判**:顾问见"寄售场景" 想新建一套自定义单据 — VMI 是订单类型分支,**底层同一个对象**
- **OpenDeploy 工具覆盖**:✅ 元数据可读(`PUR_PurchaseOrder` <!-- 对象代码待实证 -->)
- **来源**:[采购订单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E9%87%87%E8%B4%AD%E8%AE%A2%E5%8D%95) · [vmi采购订单](https://help.open.kingdee.com/dokuwiki/doku.php?id=vmi%E9%87%87%E8%B4%AD%E8%AE%A2%E5%8D%95) · [资产采购订单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E8%B5%84%E4%BA%A7%E9%87%87%E8%B4%AD%E8%AE%A2%E5%8D%95) · fetched 2026-04-23

### 订单变更(变更单 + 反向调价)
- **需求关键词**:"订单签了又改"、"加量 / 减量"、"调价记录留痕"
- **标准功能**:**采购订单变更单** 三种变更类型 — **新增 / 修改 / 取消**;变更审核成功后回写原订单字段并重算单价 / 金额 / 税额;**采购调价表** 处理批量单价调整
- **启用路径**:供应链 → 采购管理 → 订单处理 → **采购订单变更单**
- **典型误判**:顾问被要求"订单价格能改但要审批" 想 BOS 加审批 — **变更单本身走单据审批,主订单不动**;留痕天然
- **来源**:[采购订单变更单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E9%87%87%E8%B4%AD%E8%AE%A2%E5%8D%95%E5%8F%98%E6%9B%B4%E5%8D%95) · fetched 2026-04-23

### 状态机(创建 → 审核 → 关闭 / 终止)
- **需求关键词**:"订单要关单"、"反审核限制"、"关联付款不能反审"
- **标准功能**:状态流 **新增 → 保存 → 审核 → 已审核**;支持反审核,**但关联付款单后的订单不能反审核**(系统硬规则);业务关闭 / 终止两条独立状态线
- **启用路径**:自动状态机 + 操作菜单的关闭 / 终止
- **典型误判**:被问"加个订单关单按钮" → 标准已有;被问"为什么这单不能反审" → **看下游有没有付款单**,常见踩坑

### 交货控制(上下限 / 早晚限 / 累计收料)
- **需求关键词**:"允许超收 5%"、"提前到货拒收"、"看订单交货完成度"
- **标准功能**:订单表体含 `交货上限` / `交货下限` / `控制交货数量` / `最早交货日` / `最晚交货日` / `累计收料数量` / `累计入库数量` 字段;系统按这些字段在收料 / 入库时校验
- **启用路径**:订单 → 表体 → 交货安排页签
- **典型误判**:顾问写 Python 算"超收 5%" — **交货上下限字段就是干这个**

## 4. 收料 / 入库 / 退料 / 退货

### 收料通知单 vs 采购入库单(关键差异)
- **需求关键词**:"收到货还没入账"、"待入库"、"质检通过才入库"
- **标准功能**:两张单据**职责泾渭分明**
  - **收料通知单** — 记收货事实(单据类型 / 供应商 / 收料组织 / 物料);**不更新即时库存**、不影响库存报表
  - **采购入库单** — 真正更新即时库存;入库单"实收数量必须 ≥ 退货 + 关联收料数量"
- **启用路径**:供应链 → 采购管理 → **收料通知单** / **采购入库单**;系统参数控制是否强制先走收料通知
- **典型误判**:顾问看见"收到 + 入库 都要登记两次" 想合并 — **是设计如此**:仓库人员收 ≠ 财务认账,中间留质检 / 待入库环节
- **OpenDeploy 工具覆盖**:✅ 元数据可读
- **来源**:[收料通知单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%94%B6%E6%96%99%E9%80%9A%E7%9F%A5%E5%8D%95) · [采购入库单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E9%87%87%E8%B4%AD%E5%85%A5%E5%BA%93%E5%8D%95) · fetched 2026-04-23

### 退料申请单 → 采购退料单
- **需求关键词**:"质量不合格退给供应商"、"先申请再退"、"退料流程"
- **标准功能**:两张单据闭环
  - **退料申请单** — 因质量 / 价格 / 不符合订单条款发起的申请,走审批
  - **采购退料单** — 真正库存出库,更新即时库存;由收料 / 入库下推时**退料类型只能是"库存退料"**
- **启用路径**:供应链 → 采购管理 → **退料处理** → 退料申请单 / **采购退料单**
- **典型误判**:被问"退货要走流程" → 标准已分两步,不需要 BOS 自定义
- **来源**:[采购退料单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E9%87%87%E8%B4%AD%E9%80%80%E6%96%99%E5%8D%95) · [退料申请单](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E9%80%80%E6%96%99%E7%94%B3%E8%AF%B7%E5%8D%95) · fetched 2026-04-23

## 5. 采购质检(来料检验)

### 来料检验单
- **需求关键词**:"到货必须先检验"、"不合格品隔离"、"按抽样方案"
- **标准功能**:质量管理模块的**来料检验单**与采购流程联动 — 入库前(或入库后,看参数)触发检验,质检状态流 **计划 → 质检开始 → 质检完成**;支持多种**抽样方案**、**定量/定性分析**、**指标自动判定**
- **启用路径**:供应链 → 质量管理 → **来料检验单**;质量管理参数控制"是否启用来料检验" + "是否阻塞入库"
- **典型误判**:顾问被问"质检不通过禁止入库" 想写 Python 拦截 — **质量管理参数 + 检验单状态**已经能挡;不通过走 **不良品处理单** / **MRB 评审单**
- **OpenDeploy 工具覆盖**:✅ 元数据可读
- **来源**:[来料检验单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%9D%A5%E6%96%99%E6%A3%80%E9%AA%8C%E5%8D%95) · [质量管理参数](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E8%B4%A8%E9%87%8F%E7%AE%A1%E7%90%86%E5%8F%82%E6%95%B0) · fetched 2026-04-23

### 不合格处理(MRB 评审 + 不良品处理)
- **需求关键词**:"不合格怎么办"、"让步接收"、"返工 / 报废 / 退货"
- **标准功能**:**MRB 评审单**(Material Review Board)裁定不合格品的去向 — 让步接收 / 退货 / 返工 / 报废;裁定后下推对应单据
- **启用路径**:供应链 → 质量管理 → **MRB 评审单**
- **典型误判**:顾问被问"不合格要走多人评审"想 BOS 工作流 — **MRB 单 + 审批流**就是干这个
- **来源**:[mrb评审单](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=mrb%E8%AF%84%E5%AE%A1%E5%8D%95) · fetched 2026-04-23

## 6. 采购发票校验(三单匹配)

### 应付单 ← 入库 → 发票(三单匹配)
- **需求关键词**:"订单+入库+发票对得上"、"三单匹配"、"发票金额对不上"
- **标准功能**:K/3 Cloud 标准链 **合同 → 订单 → 入库单 → 应付单 → 付款**;采购订单 / 收料 / 入库 / 退料都可下推或自动生成应付单(参数控制)。**采购发票** 来源:应付单下推(自动带字段)/ 应付单列表勾选下推。**应付开票核销** 调和应付单 vs 发票金额差异(确认应付时点 ≠ 收发票时点)。下推时可选**按明细分录合并 / 不合并**(支持"多入库合一发票")
- **启用路径**:财务会计 → 应付款管理 → **应付单 / 采购发票 / 应付开票核销**
- **典型误判**:顾问被问"三单匹配" 上来就 BOS 写校验 — **下推关系 + 开票核销**已经是标准三单匹配;严格"金额不一致禁审" 看应付参数和审批流
- **OpenDeploy 工具覆盖**:✅ 元数据可读
- **来源**:[应付款管理](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%BA%94%E4%BB%98%E6%AC%BE%E7%AE%A1%E7%90%86) · [应付单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%BA%94%E4%BB%98%E5%8D%95) · [采购普通发票](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E9%87%87%E8%B4%AD%E6%99%AE%E9%80%9A%E5%8F%91%E7%A5%A8) · fetched 2026-04-23

## 7. 付款申请 + 付款单

### 付款申请单
- **需求关键词**:"付款要先申请"、"分期付款"、"预付款流程"
- **标准功能**:付款申请单作为请款流程入口,审批后下推付款单;**审核通过的费用报销单可自动下推付款申请**;通过付款单 → 下推 → 选单可关联**正数应付单 / 正数其他应付 / 负数应收 / 负数其他应收**
- **启用路径**:财务会计 → 应付款管理 → **付款申请单 / 付款单**

## 9. VMI / 寄售采购

### VMI 采购订单 + 物权转移
- **需求关键词**:"供应商管理库存"、"先入库再结算"、"消耗后才付款"、"VMI"
- **标准功能**:VMI 业务专属链路

- **启用路径**:供应链 → 采购管理 → **VMI 采购订单**;查询走 **供应商协同 VMI 物料出入库明细表** 等

## 12. 什么时候**真的**需要二开

筛掉以上 11 节后,剩下还可能要 BOS / Python 的常见场景:

- **跨账套合同**:集团总部签框架 + 多账套子公司分别下单要二开拉通(单账套内标准合同够)
- **特殊三单容差规则**:"金额差异 <1% 自动通过、>1% 财务复核" — 应付参数 / 审批流不够细 → BOS 加字段 + Python 校验
- **供应商门户外的渠道集成**:供应商不愿用协同门户而要 EDI / API → 集成开发 + Python 转换
- **复杂选源算法**:超出"价格最低 / 历史最优" — 质量评分 + 交期 + 配额综合 → BOS 选源工作台 + Python
- **审批后强制二次签字**:电子签 / 数字证书 → BOS 加二次确认 + Python 调签名服务

> 进入这五类前,**先把上面 1-10 章节的字段 / 单据类型 / 系统参数跟客户对一遍**。
