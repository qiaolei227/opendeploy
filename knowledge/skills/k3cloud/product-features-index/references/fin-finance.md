# 财务管理(AR / AP / GL)标准功能字典 🟡

> 财务模块是 K/3 Cloud 实施 80% 项目都要碰的核心。本字典覆盖 **应收 / 应付 / 凭证 / 总账 / 报表勾稽 / 月结年结 / 多币别 / 税金 / 预算 / 合并** 主干,**未在客户环境逐条实测**;具体子场景(智能会计平台模板细节、合并报表抵消顺序)需要实施时反查手册或现场验证。
>
> 模块归属:**财务会计 → 应收 / 应付 / 总账 / 报表 / 期末处理 / 智能会计平台 / 合并报表 / 预算管理**。
>
> **CB(出纳 / 银行账户 / 票据 / 资金计划 / 现金流量项目)** 在 [`cb-cash.md`](./cb-cash.md) 单独覆盖;本字典涉及 CB 的部分只一句话引用,不重复内容。

## 1. 应收单(AR)— 业务发起 + 立账模式

### 应收单(债权确立单据)
- **需求关键词**:"出库就要确认应收"、"销售应收"、"其他应收(罚款 / 押金)"
- **标准功能**:**应收单**——和销售发票彻底分离,**是债权成立的标志**。销售应收 + 其他应收(罚款 / 押金 / 借款回收)两类
- **来源单据**:手工录入 **或** 销售出库单审核自动下推(必录字段全 → 创建态;不全 → 暂存态)
- **启用路径**:财务会计 → 应收款管理 → 日常处理 → 应收单;参数配置 → "出库单审核生成应收单"
- **典型误判**:顾问被问"出库自动算应收" → 标准参数已经支持,不用 Python AfterSave 拼数据
- **来源**:[应收款管理 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%BA%94%E6%94%B6%E6%AC%BE%E7%AE%A1%E7%90%86) · fetched 2026-04-23

### 立账模式(业务应收 vs 暂估应收)
- **需求关键词**:"按发票确认收入"、"先暂估后开票冲回"、"以票控收"
- **标准功能**:**核算组织级配置**——
  - **业务应收**——业务单据(出库 / 服务确认)即立账
  - **暂估应收**——按发票立账;**没开票** 走暂估科目,**开票后** 自动红冲暂估 + 转财务应收
- **启用路径**:基础设置 → 应收应付参数 → 立账模式;业务应收单 vs 财务应收单两张单据取数走向不同
- **典型误判**:用户说"开票才算收入" → **不是二开**,切 **暂估应收** 立账模式,凭证模板自动配套
- **来源**:[应收单 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%BA%94%E6%94%B6%E5%8D%95) · fetched 2026-04-23

### 应收单状态机 + 反核销顺序
- **状态**:`暂存 → 创建 → 已审核 → 已核销 / 已开票`,各阶段独立操作权限
- **典型误判**:"反审已核销的应收单" → 必须 **先反核销 → 再反审核**,顺序硬约束;别写插件绕过
- **来源**:[应收款管理 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%BA%94%E6%94%B6%E6%AC%BE%E7%AE%A1%E7%90%86) · fetched 2026-04-23

## 2. 应付单(AP)— 业务发起 + 立账模式

### 应付单 + 付款计划
- **需求关键词**:"采购入库自动应付"、"按合同分期付款"、"其他应付(押金 / 临时挂账)"
- **标准功能**:**应付单**——应收对称债务凭证;**付款计划页签** 设多个到期日 + 金额按计划分批付款。可由 **采购入库单审核** 自动下推
- **启用路径**:财务会计 → 应付款管理 → 日常处理 → 应付单
- **典型误判**:"按合同分期付" → 标准 **付款计划** 已经支持,不要在 BOS 加自定义子表
- **来源**:[应付单 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%BA%94%E4%BB%98%E5%8D%95) · fetched 2026-04-23

### 暂估应付 / 财务应付
- **标准功能**:同应收对称——**业务应付 / 暂估应付**;暂估应付走 `应付暂估` 科目,收到发票后红冲暂估 + 转应付
- **启用路径**:基础设置 → 应收应付参数 → 立账模式
- **典型误判**:**存货暂估** vs **应付暂估** 别混 → 前者在存货核算模块,后者在应付管理;两套独立但要勾稽
- **来源**:[应付款管理 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%BA%94%E4%BB%98%E6%AC%BE%E7%AE%A1%E7%90%86) · fetched 2026-04-23

### 应付单跟踪表(订单 → 入库 → 发票 → 付款)
- **标准功能**:**应付单跟踪表**——一张表看订单 / 入库 / 开票 / 付款全状态
- **启用路径**:应付款管理 → 报表 → 应付单跟踪表
- **典型误判**:用户要"自定义跟踪报表" → 先看跟踪表满不满足,90% 不用二开
- **来源**:[应付单跟踪表 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%BA%94%E4%BB%98%E5%8D%95%E8%B7%9F%E8%B8%AA%E8%A1%A8) · fetched 2026-04-23

## 3. 核销机制(应收 ↔ 收款 / 应付 ↔ 付款)

### 核销三种方式(优先级从高到低)
- **需求关键词**:"收款冲应收"、"付款冲应付"、"自动找单核销"
- **标准功能**:
  1. **关联关系核销**——上下游下推关系,**下游单据审核即自动核销**(应收单下推收款单,收款单审核即核销)
  2. **匹配条件核销**——向导式批量;选 **核销方案** 按 客户 / 金额 / 票号 / 业务日期 匹配,可多方案按优先级叠加
  3. **手工核销**——一对多 / 多对一勾选
- **启用路径**:应收款管理 → 核销处理;应付款管理 → 核销处理
- **典型误判**:写 Python 在收款单 BeforeSave 里"找应收单核销" → **关联关系核销** 已经做了,Python 反而绕开标准核销记录无法审计
- **来源**:[应收收款核销 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%BA%94%E6%94%B6%E6%94%B6%E6%AC%BE%E6%A0%B8%E9%94%80) · fetched 2026-04-23

### 核销方案(自动核销规则)
- **需求关键词**:"按合同号自动核销"、"按发票号匹配"
- **标准功能**:**核销方案**——按 客户 / 业务日期 / 单据号 / 备注 字段定义匹配规则 + 单据范围 + 核销凭证业务日期
- **启用路径**:基础设置 → 应收应付参数 → 核销方案
- **典型误判**:多业务部门塞进一个方案 → 应该维护 **多个方案 + 优先级**
- **来源**:[应收收款核销 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%BA%94%E6%94%B6%E6%94%B6%E6%AC%BE%E6%A0%B8%E9%94%80) · fetched 2026-04-23

### 反核销 + 开票核销
- **反核销**:**核销记录界面** 选记录 → `反核销` 按钮 → 自动生成反核销凭证;顺序必须 **先反核销 → 再反审核收款单**
- **开票核销**:**应收开票核销** / **应付开票核销**——和收付款核销并列的另一条核销线,暂估立账模式下尤其关键
- **启用路径**:应收 / 应付管理 → 核销记录 / 应收开票核销 / 应付开票核销
- **来源**:[应付开票核销 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%BA%94%E4%BB%98%E5%BC%80%E7%A5%A8%E6%A0%B8%E9%94%80) · fetched 2026-04-23

## 4. 收款单 / 付款单(出纳侧)

> 收款单 / 付款单虽然挂在 AR / AP 业务流上,但实际单据维护和银行账户落账都在 **出纳管理** 模块。详见 [`cb-cash.md` § 现金 / 银行日记账](./cb-cash.md)。本字典只提一点:**"应收应付侧的核销"** 和 **"出纳侧的银行流水落账"** 是 **同一张单据的两个视角**,常规流程是 **AR/AP 侧先核销 → 出纳侧确认收 / 付款**。

## 5. 凭证生成 + 凭证模板(智能会计平台)

### 智能会计平台
- **需求关键词**:"业务单据自动做账"、"出库就出凭证"、"定时生成凭证"
- **标准功能**:**智能会计平台**——基于 **凭证模板** 把业务单据转成 **业务凭证 + 总账凭证**;两种触发:**单据审核自动生成**(实时)+ **定时任务批量生成**(夜间)
- **启用路径**:财务会计 → 智能会计平台 → 凭证模板 / 凭证生成 / 自动生成方案
- **典型误判**:Python 在 AfterSave 里 `BillObj.GenerateVoucher()` → 不需要,**配凭证模板** 即可;Python 自己拼凭证会丢业务单据 → 凭证联查链路
- **来源**:[智能会计平台 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E6%99%BA%E8%83%BD%E4%BC%9A%E8%AE%A1%E5%B9%B3%E5%8F%B0) / [凭证生成 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%87%AD%E8%AF%81%E7%94%9F%E6%88%90) · fetched 2026-04-23

### 凭证模板设计
- **需求关键词**:"自定义摘要"、"借贷取数公式"、"按部门分录"
- **标准功能**:**凭证日期 / 凭证字 / 摘要 / 科目 / 借贷方向 / 金额 / 辅助核算** 全部可按业务单据字段取数 + 公式;系统预置常见业务模板
- **启用路径**:财务会计 → 智能会计平台 → 凭证模板
- **典型误判**:摘要要"客户简称 + 物料名 + 单号" → **公式字段拼接** 即可,不需要插件
- **来源**:[凭证模板 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%87%AD%E8%AF%81%E6%A8%A1%E6%9D%BF) · fetched 2026-04-23

### 凭证云模板 + 生成情况查询 + 联查
- **云模板**:**下载凭证云模板**——按科目表 + 记账模式提供配套模板,**在线更新不受版本限制**,会计准则变了不用等产品升级
- **生成情况查询**:**凭证生成情况查询**——展现每张单据的成功 / 失败 / 未生成状态
- **联查**:凭证 ↔ 源单据 **双向联查**
- **启用路径**:财务会计 → 智能会计平台 → 下载凭证云模板 / 凭证生成情况查询
- **来源**:[下载凭证云模板 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E4%B8%8B%E8%BD%BD%E5%87%AD%E8%AF%81%E4%BA%91%E6%A8%A1%E6%9D%BF) · fetched 2026-04-23

### 自动转账(月末计提 / 摊销 / 结转)
- **需求关键词**:"月末计提折旧 / 工资"、"摊销待摊费用"、"结转损益"
- **标准功能**:**自动转账模板**——公式 + 借贷规则,月末一键生成转账凭证(折旧 / 摊销 / 结转损益 / 期间损益结转)
- **启用路径**:财务会计 → 总账 → 凭证处理 → 自动转账
- **典型误判**:Python 算月底计提 → **自动转账** 已经做这事,实在不够再考虑二开
- **来源**:[自动转账 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E8%87%AA%E5%8A%A8%E8%BD%AC%E8%B4%A6) · fetched 2026-04-23

## 6. 科目 / 辅助核算(核算项目)

### 科目(账簿核心)
- **需求关键词**:"启用账簿"、"会计科目表"、"挂往来科目"
- **标准功能**:**科目 → 会计要素 → 科目类别** 三层;科目上可设 **币别 / 数量 / 辅助核算 / 现金流量类型 / 受控系统**
- **启用路径**:基础资料 → 总账 → 科目;**快速启用向导** 一站式启用(基础参数 + 科目 + 启用日期)
- **典型误判**:启用账簿后想改科目结构 → 启用日期 + 期间已定死,**结构调整必须反结账到启用前**;UAT 后期才发现要改是噩梦
- **来源**:[科目 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E7%A7%91%E7%9B%AE) / [快速启用向导 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%BF%AB%E9%80%9F%E5%90%AF%E7%94%A8%E5%90%91%E5%AF%BC) · fetched 2026-04-23

### 辅助核算(6 类核算维度)
- **需求关键词**:"按部门看费用"、"按客户看应收"、"按项目分摊"、"按物料统计"
- **标准功能**:科目下挂的多维分析维度,**主流 6 类**:**部门 / 客户 / 供应商 / 物料 / 员工 / 自定义核算项目**;科目勾选后凭证录入强制指定
- **启用路径**:基础资料 → 总账 → 科目 → "辅助核算" 页签;基础资料 → 公共 → 核算项目类别(自定义维度)
- **典型误判 1**:"按项目核算成本" → **不是二开是配置**;新建"项目"维度挂到费用科目即可
- **典型误判 2**:已发生数据后加辅助核算 → 历史凭证不会补维度,**只能从启用期间开始统计**;实施前期就要设计
- **报表透视**:**ACCT 取数公式** 按辅助核算维度过滤;总账账龄 / 明细账 / 科目余额表都支持维度组合
- **来源**:[科目 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E7%A7%91%E7%9B%AE) / [报表取数公式 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%8A%A5%E8%A1%A8%E5%8F%96%E6%95%B0%E5%85%AC%E5%BC%8F) · fetched 2026-04-23

## 7. 多币别 / 期末调汇

### 币别 + 汇率类型 + 折算方法
- **需求关键词**:"美元结算"、"欧元采购"、"按期末 / 交易日汇率折算"
- **标准功能**:
  - **币别** 基础资料 + **汇率类型**(预置:**公司汇率 / 即期汇率 / 预算汇率**,支持直接 / 间接汇率)
  - **外币折算方法**(预置:**期末汇率 / 平均汇率 / 历史汇率 / 本年累计折算**)
  - 支持 **核算组织级本位币**(多组织可不同)
- **启用路径**:基础资料 → 公共 → 币别;基础资料 → 财务 → 汇率 / 外币折算方法
- **来源**:[币别 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%B8%81%E5%88%AB) / [外币折算方法 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%A4%96%E5%B8%81%E6%8A%98%E7%AE%97%E6%96%B9%E6%B3%95) · fetched 2026-04-23

### 期末调汇(汇兑损益)
- **需求关键词**:"月末外币余额按期末汇率折一遍"、"汇兑损益自动出凭证"
- **标准功能**:**期末调汇方案**——按期末汇率折算外币科目余额,**差额自动入汇兑损益 + 生成记账凭证**;**应收 / 应付 / 总账三套独立调汇**;**多方案** 可指定不同科目走不同汇兑损益科目 / 汇率
- **启用路径**:财务会计 → 总账 → 期末处理 → 期末调汇;应收款管理 → 应收期末调汇;应付款管理 → 应付期末调汇
- **典型误判**:Python 在月底跑脚本算汇兑差 → **期末调汇方案** 已经做 + 输出可审计凭证;Python 拼的凭证没方案痕迹,审计难解释
- **来源**:[期末调汇 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%9C%9F%E6%9C%AB%E8%B0%83%E6%B1%87) / [应收期末调汇 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%BA%94%E6%94%B6%E6%9C%9F%E6%9C%AB%E8%B0%83%E6%B1%87) · fetched 2026-04-23

## 8. 税金管理

### 税种 / 税率 + 价内价外
- **需求关键词**:"增值税自动算"、"含税价 / 不含税价"、"税额对不上"
- **标准功能**:
  - **税种**(预置增值税)+ **税额精确度** + 舍入规则(向上 / 四舍五入)+ **税率** 关联税种
  - **价内 / 价外逻辑**:增值税专用发票(销项 / 进项)→ **价外**;普通发票(销售 / 采购)→ **价内 / 价外可选**;进项费用普通发票 → 强制 **价内**
- **启用路径**:基础资料 → 公共 → 税种 / 税率;单据上 **计税方式字段**
- **典型误判**:"税额小数老对不上" → **税额精确度 / 舍入规则** 没配对,不是二开;"要自定义算税逻辑" → 改 **计税方式 / 税率** 即可
- **来源**:[税种 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%A8%8E%E7%A7%8D) / [采购增值税专用发票 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E9%87%87%E8%B4%AD%E5%A2%9E%E5%80%BC%E7%A8%8E%E4%B8%93%E7%94%A8%E5%8F%91%E7%A5%A8) · fetched 2026-04-23

## 9. 月结 / 年结(关账顺序)

### 关账顺序硬约束
- **需求关键词**:"月底结账"、"关账"、"月结"
- **标准功能**:**存货核算 → 应收 → 应付 → 总账**,**前置没关后置不能关**
  1. 存货核算期末关账(核算成本传递到总账)
  2. 应收期末关账(应收凭证 + 期末调汇)
  3. 应付期末关账(应付凭证 + 期末调汇)
  4. 总账期末结账(汇总所有上游)
- **启用路径**:各模块 → 期末处理 → 结账
- **典型误判**:"总账关不了" → 99% 是上游 **存货 / 应收 / 应付** 没关账,不是 bug
- **来源**:[总账期末结账 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%80%BB%E8%B4%A6%E6%9C%9F%E6%9C%AB%E7%BB%93%E8%B4%A6) · fetched 2026-04-23

### 关账 vs 结账 + 反结账
- **关账**:**锁单据期间**(任意日期),给年终一个稳定边界防补录;**比结账更刚性**,关账后必须 **反关账** 才能录单
- **结账**:**结转余额**(月度),结账后仍可 **反结账**
- **反结账**:必须 **从最新已结期间逐期向前反**,不能跳期。当前结到 6 月要反到 3 月 → **6 → 5 → 4 → 3** 逐期来
- **启用路径**:财务会计 → 总账 → 期末处理 → 关账 / 结账 / 反结账
- **来源**:[关账 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%85%B3%E8%B4%A6) / [反结账 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%8F%8D%E7%BB%93%E8%B4%A6) · fetched 2026-04-23

### 总账自动结账方案 + 财务结账中心
- **自动结账方案**:定时任务自动跑 关账 → 自动转账 → 期末调汇 → 结账
- **财务结账中心**:一个面板看所有核算组织 + 所有模块的关账 / 结账状态
- **启用路径**:财务会计 → 总账 → 期末处理 → 总账自动结账方案;财务会计 → 财务结账中心
- **来源**:[总账自动结账方案 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%80%BB%E8%B4%A6%E8%87%AA%E5%8A%A8%E7%BB%93%E8%B4%A6%E6%96%B9%E6%A1%88) / [财务结账中心 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E8%B4%A2%E5%8A%A1%E7%BB%93%E8%B4%A6%E4%B8%AD%E5%BF%83) · fetched 2026-04-23

## 10. 总账报表 + 账表勾稽

### 三大表
- **需求关键词**:"资产负债表"、"利润表"、"现金流量表"
- **标准功能**:系统内置三大表模板——
  - **资产负债表** 勾稽:`资产 = 负债 + 所有者权益`
  - **利润表** 99% 数据来自总账,通过 **ACCT 取数公式** 取期间发生额 / 余额
  - **现金流量表** 走 **凭证现金流量项目标签** + 内置直接法 / 附表(详见 [`cb-cash.md` § 6 现金流量项目设置](./cb-cash.md))
- **启用路径**:财务会计 → 总账 → 报表
- **典型误判**:报表和明细账对不上 → 90% 是 **科目辅助核算** / **现金流量项目** 没勾对,不是公式错
- **来源**:[报表系统 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%8A%A5%E8%A1%A8%E7%B3%BB%E7%BB%9F) / [报表取数公式 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%8A%A5%E8%A1%A8%E5%8F%96%E6%95%B0%E5%85%AC%E5%BC%8F) · fetched 2026-04-23

### 账表设计 / 自定义账表
- **需求关键词**:"做一张管理报表"、"老板要个分析表"
- **标准功能**:**账表设计器**——6 种类型 **简单账表 / 树形账表 / 分页账表 / 直接 SQL 账表 / 透视表 / 简单报表**;**直接 SQL** 类型可写白名单查询
- **启用路径**:财务会计 → 总账 → 账表设计
- **典型误判**:"做个管理报表" → 先用 **账表设计器** 试,Python 插件做报表是反模式
- **来源**:[账表设计 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E8%B4%A6%E8%A1%A8%E8%AE%BE%E8%AE%A1) · fetched 2026-04-23

### 账龄分析(总账侧 vs AR/AP 侧)
- **需求关键词**:"看客户欠款多久"、"往来科目账龄"、"AR / AP 到期债务"
- **标准功能**:
  - **总账账龄分析表**——按辅助核算维度组合看往来科目余额时间分布,自定义账龄分组天数 / 名称,支持多账簿合并
  - **应收款账龄分析表**——数据来源 **核销未完毕的应收单 + 收款单 + 收款退款单**
  - **应付款账龄分析表**——账龄计算可按 **业务日期** 或 **到期日**
- **启用路径**:总账 → 报表 → 总账账龄分析表;应收 / 应付管理 → 报表 → 账龄分析表
- **典型误判**:业务部门用总账账龄、财务用 AR/AP 账龄 → 数据可能差,**前者按科目余额、后者按单据核销状态**;实施前就统一口径
- **来源**:[总账账龄分析表 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E6%80%BB%E8%B4%A6%E8%B4%A6%E9%BE%84%E5%88%86%E6%9E%90%E8%A1%A8) / [应收款账龄分析表 · help.open.kingdee.com](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E5%BA%94%E6%94%B6%E6%AC%BE%E8%B4%A6%E9%BE%84%E5%88%86%E6%9E%90%E8%A1%A8) · fetched 2026-04-23
