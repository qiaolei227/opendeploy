> 🟡 **主流程级别**。本文以金蝶官方公开手册(`help.open.kingdee.com/dokuwiki` 与 `dokuwiki_std`)为线索源,从**实施顾问视角**重写为"误判提醒+功能反查"字典。
>
> **未在客户环境逐条实测**——具体子参数、字段位置、版本差异(V8 / V9 各版次)以现场为准。带 `<!-- 待实证 -->` 的条目表示官方页面没明说,**不要凭印象补**。

## 1. BOM / 物料清单

### 多版本 BOM
- **需求关键词**:"同一个产品有几套配方"、"夏季版 / 冬季版"、"老款新款共存"
- **标准功能**:**BOM 版本** 是物料清单的核心唯一键,同一物料可维护多版本 BOM,生产订单按 `BOM 版本` 字段挑选
- **启用路径**:生产制造 → 工程数据 → 物料清单 → 物料清单
- **典型误判**:顾问看到客户"两套配方" 想用扩展字段区分 — **直接建第二个 BOM 版本就行**
- **OpenDeploy 工具覆盖**:✅ 元数据可读(`kingdee_list_objects` / `kingdee_describe_object`),BOM 主对象 `ENG_BOM`
- **来源**:[help.open.kingdee.com/物料清单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%89%A9%E6%96%99%E6%B8%85%E5%8D%95) · fetched 2026-04-23

### 替代料(替代件)
- **需求关键词**:"A 料没了用 B"、"主料缺货换替代"、"按比例混用替代料"
- **标准功能**:子项类型 = **替代件** + **替代策略**(混用替代 / 手工替代 / 整批替代 / 整批+混用替代);同一**项次**的子项归为一组替代关系
- **启用路径**:生产制造 → 工程数据 → 物料清单 → 物料清单 → 子项明细行 `子项类型` + `项次` + `替代策略`
- **典型误判**:顾问写 Python 在领料时拦截"如果库存不足就换 B" — **BOM 替代料 + 整批/混用策略原生支持**,做插件等于重写引擎
- **OpenDeploy 工具覆盖**:✅ 元数据可读
- **来源**:[help.open.kingdee.com/物料清单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%89%A9%E6%96%99%E6%B8%85%E5%8D%95) · fetched 2026-04-23

### 用量分子分母
- **需求关键词**:"3 个零件做 2 个产品"、"非整数用量"、"按损耗率折算"
- **标准功能**:`用量:分子` / `用量:分母` 表示生成单位父项物料所需子项数量,精确表达分数用量;额外支持**阶梯损耗**
- **启用路径**:子项明细行 `用量:分子` + `用量:分母` 字段;阶梯损耗在子项设置里
- **典型误判**:写 Python 算"按订单数量动态算损耗" — 阶梯损耗就是这个
- **来源**:[help.open.kingdee.com/物料清单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%89%A9%E6%96%99%E6%B8%85%E5%8D%95) · fetched 2026-04-23

### 工程变更(ECN)
- **需求关键词**:"BOM 改了要追溯"、"工程变更单"
- **标准功能**:BOM 主表有 `是否变更中` 字段记录工程变更单状态;变更后可同步**在制生产/委外用料清单**和**计划 BOM**
- **启用路径**:物料清单上有"同步更新"操作触发同步在制单据 <!-- 工程变更单具体菜单待实证 -->
- **典型误判**:被问"BOM 改了在制订单怎么办" → 标准有同步操作,不用手刷
- **来源**:[help.open.kingdee.com/物料清单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%89%A9%E6%96%99%E6%B8%85%E5%8D%95) · fetched 2026-04-23

### BOM 用途分类
- **需求关键词**:"自制件 BOM 和委外 BOM 不一样"、"组装报价用一个简化 BOM"
- **标准功能**:BOM `用途` 字段:**自制 / 委外 / 通用 / 组装 / 报价**,不同用途走不同业务流
- **启用路径**:物料清单 → `用途` 字段
- **典型误判**:顾问建俩物料区分"组装 vs 报价" → 一个物料多个用途的 BOM 即可
- **来源**:[help.open.kingdee.com/物料清单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%89%A9%E6%96%99%E6%B8%85%E5%8D%95) · fetched 2026-04-23

## 2. 工艺路线 / 工序

### 工艺路线(工序+工作中心+工时)
- **需求关键词**:"按工序排产"、"每道工序在哪个车间做"、"工时定额"
- **标准功能**:工艺路线定义工序顺序、每个工序绑定 **工作中心**、**6 个基本活动** 维护标准工时定额
- **启用路径**:生产制造 → 车间管理 → 工艺建模(或 智慧车间MES → 智慧车间 → 工艺建模)
- **典型误判**:顾问被问"工时怎么算" 想做插件 — **工艺路线 6 个活动数量 + 基本批量**就是工时模型
- **OpenDeploy 工具覆盖**:✅ 元数据可读
- **来源**:[help.open.kingdee.com/工艺路线](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%B7%A5%E8%89%BA%E8%B7%AF%E7%BA%BF) · fetched 2026-04-23

### 并行/串行/网状工序
- **需求关键词**:"两道工序同时做"、"工序 A 没完工序 B 就开"、"重叠生产"
- **标准功能**:工艺路线支持 **线状 / 重叠 / 网状 / 分割 / 合并** 多种工序关系;通过 `参照序列` + `转出工序` / `转入工序` + "同时开始/同时结束" 对齐方式表达
- **启用路径**:工艺建模 → 工序明细 → 工序序列页签(主干序列 / 并行序列 / 替代序列 / 返工序列)
- **典型误判**:认为"K/3 Cloud 只能串行工序" 想自己写排程算法 — 标准支持四种序列
- **来源**:[help.open.kingdee.com/工艺路线](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%B7%A5%E8%89%BA%E8%B7%AF%E7%BA%BF) · fetched 2026-04-23

### 替代/返工序列
- **需求关键词**:"工序 A 失败可以走 A'"、"返工怎么走"
- **标准功能**:工艺路线内置 **替代序列**(同一工位备选)和 **返工序列**(质检失败回炉)两种特殊序列类型
- **启用路径**:工艺建模 → 工序序列 → 序列类型
- **典型误判**:顾问做返工业务想新建一张单 — 标准就有返工序列 + 工序汇报里报返修数量
- **来源**:[help.open.kingdee.com/工艺路线](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%B7%A5%E8%89%BA%E8%B7%AF%E7%BA%BF) · fetched 2026-04-23

## 3. 生产订单

### 订单来源(MRP / 销售订单 / 手工)
- **需求关键词**:"销售订单直接转生产"、"自动生成生产订单"
- **标准功能**:生产订单四种来源 — **手工录入 / MRP 生成 / MPS 生成 / 拆分**;来源单据可以为销售订单或空
- **启用路径**:生产制造 → 生产管理 → 生产订单 → 生产订单
- **典型误判**:顾问写 Python "销售订单审核后下推生产订单" — 走 MRP 或销售订单标准下推路径即可,没必要插件
- **OpenDeploy 工具覆盖**:✅ 元数据可读,生产订单对象 `PRD_MO`
- **来源**:[help.open.kingdee.com/生产订单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%94%9F%E4%BA%A7%E8%AE%A2%E5%8D%95) · fetched 2026-04-23

### 状态机(计划→下达→开工→完工→结案)
- **需求关键词**:"订单到哪一步了"、"开工不让改"、"结案后锁定"
- **标准功能**:分录业务状态正向序列 **计划 → 计划确认 → 下达 → 开工 → 完工 → 结案**;支持正向跳跃和逆向操作
- **启用路径**:生产订单分录 `业务状态` 字段;单据类型上配 `自动下达 / 自动开工` 等参数
- **典型误判**:顾问要求做"开工后字段锁死" 想 BOS 锁字段 — 标准状态机已经控制了大部分操作权限,**先看默认行为**再决定要不要 BOS
- **来源**:[help.open.kingdee.com/生产订单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%94%9F%E4%BA%A7%E8%AE%A2%E5%8D%95) · fetched 2026-04-23

### 拆单
- **需求关键词**:"一张订单拆几张做"、"分批投产"
- **标准功能**:**拆分行** 操作 — 计划/计划确认/下达/开工状态且未挂起的分录可拆分成多行
- **启用路径**:生产订单 → 选中分录 → 拆分行
- **典型误判**:被问"能不能把一个订单分两批" → 不需要二开,标准操作
- **来源**:[help.open.kingdee.com/生产订单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%94%9F%E4%BA%A7%E8%AE%A2%E5%8D%95) · fetched 2026-04-23

### 单据类型差异化管控
- **需求关键词**:"自制订单和返修订单流程不一样"、"领料检查规则不同"
- **标准功能**:**单据类型** 维度配置不同业务规则(产品入库材料领用检查 / 结案规则 / 自动下达开工等)
- **启用路径**:生产订单单据类型维护 <!-- 具体菜单路径待实证 -->
- **典型误判**:顾问被问"两种订单不同规则" 想 BOS 拆两个单据 — 单据类型已经区分,**不用复制对象**
- **来源**:[help.open.kingdee.com/生产订单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%94%9F%E4%BA%A7%E8%AE%A2%E5%8D%95) · fetched 2026-04-23

## 4. 工序汇报 / 报工

### 工序报工
- **需求关键词**:"工人报工"、"完工汇报"、"按工序记数量"
- **标准功能**:工序汇报记录 **合格数 / 工废 / 料废 / 损耗 / 待返修** 数量,以及实际开/完工时间、班次班组、资源、设备
- **启用路径**:生产制造 → 车间管理 → 工序汇报 → 工序汇报
- **典型误判**:顾问被问"想要个报工界面" 上来就做表单 — 标准工序汇报单字段齐全;复杂界面再考虑 BOS 改版
- **OpenDeploy 工具覆盖**:✅ 元数据可读
- **来源**:[help.open.kingdee.com/工序汇报](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%B7%A5%E5%BA%8F%E6%B1%87%E6%8A%A5) · fetched 2026-04-23

### 完工数量公式
- **需求关键词**:"合格率"、"良品率"、"废品分类"
- **标准功能**:**完工数量 = 合格数量 + 废品数量 + 待返修数量**;废品再分 **工废 / 料废 / 损耗** 三类
- **启用路径**:工序汇报单 → 数量字段
- **典型误判**:顾问被问"统计良品率" 想做报表 SQL — 标准字段已分类,普通报表/BI 直接出
- **来源**:[help.open.kingdee.com/工序汇报](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%B7%A5%E5%BA%8F%E6%B1%87%E6%8A%A5) · fetched 2026-04-23

### 工序间下推(移转/检验/不良品)
- **需求关键词**:"工序 A 报完自动转工序 B"、"质检流程"、"不良品记录"
- **标准功能**:工序汇报支持下推 **生产入库单 / 检验单 / 工序不良品明细**
- **启用路径**:工序汇报 → 操作 → 下推
- **典型误判**:顾问被要求"报工后自动质检" 想 Python 做下推 — 标准下推菜单就有
- **来源**:[help.open.kingdee.com/工序汇报](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%B7%A5%E5%BA%8F%E6%B1%87%E6%8A%A5) · fetched 2026-04-23

- **标准功能**:手册提及"标准程序提供开放接口,允许通过 BOS 定制工序汇报界面"

## 5. 委外加工

### 整单委外(产品级)
- **需求关键词**:"找外协厂做整个产品"、"委外加工订单"
- **标准功能**:委外管理覆盖 **委外订单 / 委外用料清单 / 委外领料 / 委外退料 / 采购入库**(供应商交货)全过程
- **启用路径**:生产制造 → 委外管理
- **典型误判**:顾问被问"委外做整批产品" 想用销售/采购拼流程 — 委外管理是独立模块,有自己的订单/领料/入库闭环
- **OpenDeploy 工具覆盖**:✅ 元数据可读,委外订单对象 `SUB_SUBREQORDER` <!-- 对象代码待实证 -->
- **来源**:[help.open.kingdee.com/委外管理](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%A7%94%E5%A4%96%E7%AE%A1%E7%90%86) · fetched 2026-04-23

### 委外用料 + 委外领料/退料
- **需求关键词**:"我提供原料给外协"、"外协退还剩料"
- **标准功能**:**委外用料清单** + **委外领料单** + **委外退料单** 三张单据闭环;材料由企业提供
- **启用路径**:生产制造 → 委外管理 → 委外领料 / 委外退料
- **典型误判**:被问"外协退料怎么处理" → 标准退料单,不是仓库退库二开
- **来源**:[help.open.kingdee.com/委外管理](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%A7%94%E5%A4%96%E7%AE%A1%E7%90%86) · fetched 2026-04-23

### 工序委外(单工序级)
- **需求关键词**:"只有这道电镀外发"、"中间工序找外协"、"做完发回来下道工序"
- **标准功能**:**工序委外** 与整单委外不同,针对单个或连续几个工序;支持 5 种转移类型(主组织↔供应商 / 协同组织↔供应商 / 供应商↔供应商连续委外)
- **启用路径**:生产制造 → 车间管理 → 工序委外 → 工序转移单委外发出列表 / 接收列表
- **典型误判**:顾问把工序委外做成"两次整单委外" → 错,工序委外是 K/3 Cloud 独立流程,有质量/结算字段闭环
- **来源**:[help.open.kingdee.com/工序委外](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%B7%A5%E5%BA%8F%E5%A7%94%E5%A4%96) · fetched 2026-04-23

### 末序委外直接入库
- **需求关键词**:"最后一道电镀完直接入库"、"省一步工序汇报"
- **标准功能**:工序委外的**末序工序**为入库点时,系统显示 `入库状态` 字段(未入库 / 部分入库 / 全部入库)直接闭环到产品入库
- **启用路径**:工序转移单 → `入库点` 标志 + `入库状态` 字段
- **典型误判**:顾问让客户"再开一张工序汇报+入库" → 末序入库点配置后省了
- **来源**:[help.open.kingdee.com/工序委外](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E5%B7%A5%E5%BA%8F%E5%A7%94%E5%A4%96) · fetched 2026-04-23

## 7. 计划管理(MRP/MPS)

### MRP 运算 + 计划订单
- **需求关键词**:"自动算缺料"、"按销售订单算生产计划"、"MRP 跑批"
