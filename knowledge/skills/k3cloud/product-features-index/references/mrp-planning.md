# 物料需求计划(MRP)标准功能字典 🟡

> **定位**:供应链 / 计划经理最常误判的模块。MRP 在 K/3 Cloud 是一套**重配置、轻代码**的引擎 —— 90% 的"需求"靠"计划方案 + 物料档案计划属性"两层组合就能落,真要动到二开的少。
>
> 顾问拿到需求第一反应"这要写算法吧",九成情况下答案是"先看计划方案勾了什么 + 物料档案计划页签填了什么"。

---

## 0. 核心概念速览(读其他章节前先扫一眼)

K/3 Cloud 计划管理三层数据骨架:

| 层 | 在哪配 | 决定什么 |
|---|---|---|
| **计划方案** | 生产制造 → 计划管理 → 物料需求计划 → **计划方案** | MRP 这一次跑**取哪些需求 / 哪些供给 / 用哪个公式算净需求** |
| **物料档案 · 计划页签** | 基础资料 → 物料 → 打开物料 → "计划"页签 | **每个物料的策略**(MPS / MRP / 无)、批量、安全库存、提前期 |
| **MRP 运算 / 周期计划** | 计划管理 → 物料需求计划 → 计划运算向导 / MRP 周期计划 | **触发**:手工 / 周期跑 |

净需求公式(系统硬编码,不可改):

```
净需求 = 毛需求 × (1 + 损耗率) − 现有库存 + 安全库存 − 预计入库 + 已分配
```

**判断顾问需求是不是二开的口诀**:能映射到上面任一字段 → 标准功能;不能映射 → 再看是不是"计划方案勾选项"(几十个开关);都不行 → 才考虑 BOS / Python。

## 1. MRP 运算触发

### 手工运算(计划运算向导)
- **需求关键词**:"手动跑一次 MRP"、"算一下未来一个月的需求"
- **标准功能**:**计划运算向导** / **批量计划运算向导**——选定计划方案 + 计划期间 + 物料范围,一次完整运算;支持净改变模式(只重算受影响物料)
- **启用路径**:生产制造 → 计划管理 → 物料需求计划 → **计划运算向导**(单方案) / **批量计划运算向导**(多方案串跑)
- **典型误判**:用户说"我要个按钮一键跑 MRP" — **就是这个向导**,不用做按钮插件
- **OpenDeploy 工具覆盖**:❌(MRP 无相关工具)
- **来源**:[计划管理](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E8%AE%A1%E5%88%92%E7%AE%A1%E7%90%86) · fetched 2026-04-23

### 定时 / 周期触发(MRP 周期计划)
- **需求关键词**:"每天凌晨自动跑 MRP"、"按周自动算"
- **标准功能**:**MRP 周期计划**报表按月/周/日/自定义周期展示运算结果。**注意**:官方手册中"周期计划"主要指**展示维度**(报表口径)而非定时调度器;真正的定时调度需走金蝶后台服务计划任务(BOS 后台 → 计划任务 / Job Scheduler)
- **启用路径**:计划管理 → 报表分析 → **MRP 周计划**;定时调度走 BOS Designer → 后台计划任务
- **典型误判**:顾问要写 Python 写定时器跑 MRP — **后台计划任务调"批量计划运算向导"服务**就行,不用插件

- **OpenDeploy 工具覆盖**:❌(MRP 无相关工具)
- **来源**:[mrp 周期计划](https://help.open.kingdee.com/dokuwiki/doku.php?id=mrp%E5%91%A8%E6%9C%9F%E8%AE%A1%E5%88%92) · fetched 2026-04-23

### 销售订单 / 物料触发(净改变)
- **需求关键词**:"销售订单一审核就重算"、"改了 BOM 自动重算"
- **标准功能**:K/3 Cloud MRP 支持**净改变运算**——单据级别的供需变更进入待算队列,下次运算只算变化的部分。**不是事件驱动实时算**——还是要"运算"动作触发,只是范围小
- **典型误判**:用户说"销售订单审核要立刻看到 MRP 结果" — 标准是"运算后看",真要实时只能 BOS 联动 + 异步触发后台运算服务,**慎答应**(性能黑洞)
- **OpenDeploy 工具覆盖**:❌(MRP 无相关工具)
- **来源**:[计划管理](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E8%AE%A1%E5%88%92%E7%AE%A1%E7%90%86) · fetched 2026-04-23

## 2. 主生产计划(MPS)与 MRP 的关系

### MPS 物料 vs MRP 物料
- **需求关键词**:"成品要不要进 MRP?"、"哪些料走 MPS 哪些走 MRP?"
- **标准功能**:**物料档案 · 计划页签 · 计划策略**字段三选一:
  - **MPS**(主生产计划)— 关键成品 / 重要半成品 / 长提前期物料,先编 MPS 再展开 MRP
  - **MRP**(物料需求计划)— 普通物料,直接被 MPS 或独立需求驱动
  - **无计划**— 不参与运算
- **启用路径**:基础资料 → 物料 → 打开物料 → **计划**页签 → "计划策略"
- **典型误判**:用户问"为啥这个料 MRP 没出计划订单" — 9 成是计划策略选了"无计划"或父料没进 MPS
- **OpenDeploy 工具覆盖**:❌(MRP 无相关工具)
- **来源**:[物料](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%89%A9%E6%96%99) · fetched 2026-04-23

### MPS 编制 / 核准流程
- **需求关键词**:"主计划员要拍板的那张表"、"MPS 流程"
- **标准功能**:计划管理 → **主计划编制向导** → 生成 MPS 计划订单 → **主计划核准** → 核准后展 MRP
- **启用路径**:计划管理 → 主生产计划 → 主计划编制向导 / 主计划核准
- **典型误判**:中小客户其实不用 MPS(直接销售订单驱 MRP)——别上来就推 MPS 复杂流程
- **OpenDeploy 工具覆盖**:❌(MRP 无相关工具)
- **来源**:[计划管理](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E8%AE%A1%E5%88%92%E7%AE%A1%E7%90%86) · fetched 2026-04-23

## 3. 物料供需平衡 / 净需求

### 供需来源勾选(计划方案核心)
- **需求关键词**:"为啥预测没进 MRP"、"采购订单不算供给吗"、"在途要不要算"
- **标准功能**:计划方案上有两组复选框
  - **需求来源**:预测单 / 销售订单 / 冲销结果 / 计划订单(可勾)
  - **供给来源**:计划订单 / 生产订单 / 委外订单 / 采购申请 / 采购订单 / 库存等(可勾)
- **启用路径**:生产制造 → 计划管理 → 物料需求计划 → 计划方案 → "运算范围"页签
- **典型误判**:用户说"我预测都录了为啥 MRP 不认" — **去计划方案看"需求来源"勾了预测没**,90% 是没勾
- **OpenDeploy 工具覆盖**:❌(MRP 无相关工具)
- **来源**:[计划方案](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E8%AE%A1%E5%88%92%E6%96%B9%E6%A1%88) · fetched 2026-04-23

### 物料供需查询(实时供需视图)
- **需求关键词**:"看一眼这个料现在缺多少"、"还需要采购多少"
- **标准功能**:**物料供需汇总表** / **物料供需明细表** / **物料供需状态查询**——按物料展示在手 + 在途 + 需求 + 净缺口
- **启用路径**:计划管理 → 物料供需状态 → 物料供需状态查询(或汇总 / 明细表)
- **典型误判**:用户说"做个看板看缺料" — **标准报表已经有**,真要可视化导出 Excel 二次加工
- **OpenDeploy 工具覆盖**:❌(MRP 无相关工具)
- **来源**:[计划管理](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E8%AE%A1%E5%88%92%E7%AE%A1%E7%90%86) · fetched 2026-04-23

### 配套材料采购申请
- **需求关键词**:"按订单算缺料 + 直接生采购申请"、"接单即采购"
- **标准功能**:**配套材料采购申请**功能可基于销售订单 / 生产订单 / 委外订单 / 预测 / 计划订单**直接计算缺料并生成采购申请单**(不走完整 MRP 运算)。考虑安全库存 / 在手 / 损耗 / 仓库 MRP 控制
- **启用路径**:采购管理 → 采购申请 → 配套材料采购申请
- **典型误判**:用户"按订单一键算缺料采购" — **不需要二开,标准功能**;轻量场景甚至不用跑 MRP
- **OpenDeploy 工具覆盖**:❌(MRP 无相关工具)
- **来源**:[配套材料采购申请](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E9%85%8D%E5%A5%97%E6%9D%90%E6%96%99%E9%87%87%E8%B4%AD%E7%94%B3%E8%AF%B7) · fetched 2026-04-23

## 4. 物料档案 · 计划属性(批量 / 安全库存 / 提前期)

> 这一栏决定了 MRP 给出的计划订单"什么时候要 / 一次要多少 / 起订量"。改字段 = 改逻辑,**别去改 MRP 引擎**。

### 订货策略(批量策略)
- **需求关键词**:"每次至少要订 100"、"按周凑单"、"再订货点法"
- **标准功能**:四种订货策略
  - **LFL**(批对批法)— 需多少订多少
  - **FOQ**(固定批量法)— 每次订固定量
  - **POQ**(期间订货量法)— 周/月凑单一起订
  - **ROP**(再订货点法)— 库存降到再订货点就触发
- **启用路径**:基础资料 → 物料 → 计划页签 → "订货策略"
- **典型误判**:用户"我们每次必须订 1000 起" — **FOQ + 最小订货量**,不用插件;"我们一周一汇总" → POQ
- **OpenDeploy 工具覆盖**:❌(MRP 无相关工具)
- **来源**:[物料](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%89%A9%E6%96%99) · fetched 2026-04-23

### 安全库存 / 最大最小订货量 / 批量增量 / 再订货点
- **需求关键词**:"留一点安全库存"、"最少 50 起订"、"100 一箱必须整箱订"、"低于 200 触发采购"
- **标准功能**:对应字段
  - **安全库存** — 净需求公式直接用
  - **最小订货量 / 最大订货量** — 控制单次订单上下限
  - **批量增量** — 起订量上的整数包装单位(50 起订、10 一档,生成 50/60/70...)
  - **再订货点** — ROP 策略的触发点
- **启用路径**:基础资料 → 物料 → 计划页签
- **典型误判**:这四个字段几乎涵盖 90% "起订 / 整箱 / 安全线"需求,**进客户先问填了没**
- **OpenDeploy 工具覆盖**:❌(MRP 无相关工具)
- **来源**:[物料](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%89%A9%E6%96%99) · fetched 2026-04-23

### 提前期(固定 / 变动 / 检验)
- **需求关键词**:"采购周期 7 天"、"自制 5 天"、"到货后还要 3 天检验"
- **标准功能**:三种提前期
  - **固定提前期** — 采购 / 生产固定耗时
  - **变动提前期** — 按数量线性增长(单位时间 × 数量)
  - **检验提前期** — 入库前的检验缓冲
- **启用路径**:基础资料 → 物料 → 计划页签
- **典型误判**:用户"采购到货后还要质检 3 天才能用" — **检验提前期字段**,不要改 MRP 算法
- **OpenDeploy 工具覆盖**:❌(MRP 无相关工具)
- **来源**:[物料](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E7%89%A9%E6%96%99) · fetched 2026-04-23

### 损耗率(工程数据)
- **需求关键词**:"投料要多 5%"、"出成率 95%"
- **标准功能**:净需求公式里 `毛需求 × (1 + 损耗率)`,损耗率取自工程数据(BOM 子项 / 物料档案)
- **启用路径**:工程数据 → BOM(子项行)/ 物料档案
- **典型误判**:用户"算 MRP 要考虑损耗" — **填了就自动算**,无需插件
- **OpenDeploy 工具覆盖**:❌(MRP 无相关工具)
- **来源**:[计划方案](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E8%AE%A1%E5%88%92%E6%96%B9%E6%A1%88) · fetched 2026-04-23

## 5. 计划订单 → 采购申请 / 生产订单转换

### 计划订单审核 + 投放
- **需求关键词**:"MRP 出来的单子要批一下才下发"、"集中投放"
- **标准功能**:MRP 运算输出**计划订单**(中间单据);计划员通过**计划订单批量维护** / **计划员工作台**审核 → 投放(下推)→ 生成采购申请 / 生产订单 / 委外订单
- **启用路径**:计划管理 → 计划订单 → 计划订单 / 计划订单批量维护 / 计划员工作台
- **典型误判**:用户"想要 MRP 出来直接生 PO" — **可以下推但不建议跳过审核**,跟客户确认审核责任归属
- **OpenDeploy 工具覆盖**:❌(MRP 无相关工具)
- **来源**:[计划管理](https://help.open.kingdee.com/dokuwiki_std/doku.php?id=%E8%AE%A1%E5%88%92%E7%AE%A1%E7%90%86) · fetched 2026-04-23

### 转换路径(物料属性决定)
- **需求关键词**:"自制件应该出生产订单,采购件出采购申请"
- **标准功能**:计划订单下推时按物料属性自动分流
  - **自制** → 生产订单
  - **采购** → 采购申请单(再核准转采购订单)
  - **委外** → 委外订单
- **启用路径**:基础资料 → 物料 → "基本信息" / "采购" / "生产"页签的属性
- **典型误判**:同一个料既可以自制也可以采购 → **物料档案上有"默认"属性**,特殊情况让计划员手工改下推目标
- **OpenDeploy 工具覆盖**:❌(MRP 无相关工具)
- **来源**:[采购申请单](https://help.open.kingdee.com/dokuwiki/doku.php?id=%E9%87%87%E8%B4%AD%E7%94%B3%E8%AF%B7%E5%8D%95) · fetched 2026-04-23

## 6. 反向追溯(为什么这单要建?)

### MRP 日志 / 运算结果明细
- **需求关键词**:"这张计划订单为啥生出来的"、"追到销售订单"
- **标准功能**:
  - **MRP 日志查询** — 记录每次运算的参数 / 过程 / 例外信息(每次运算唯一日志编码)
  - **MRP 运算结果明细表** — 物料级毛需求 / 预计入库 / 计划订单量 / 单据类型 / 单据编号 / 单据行号 / 计划跟踪号
  - **计划跟踪号(FPlanTrackNumber)** — MRP 自动生成的链条 ID,**反向追溯的关键字段**
- **启用路径**:计划管理 → 报表查询 → MRP 日志查询 / MRP 运算结果明细表 / MRP 运算结果汇总表
- **典型误判**:用户"想要个报表知道每张计划订单源头" — **运算结果明细表 + 计划跟踪号已经能拼出整条链**,不用 SQL 二开

| "采购到货后还要 3 天检验才能用" | **检验提前期** | 物料档案计划页签 |
| "想看每张计划订单为啥建" | **MRP 运算结果明细 + 计划跟踪号** | 标准报表 |
| "预测应该进 MRP" | **计划方案勾"预测单"为需求来源** | 计划方案勾选 |
| "在途采购订单要算供给" | **计划方案勾"采购订单"为供给来源** | 计划方案勾选 |
| "为啥这个料没出计划订单" | **物料档案计划策略选了"无计划"** | 物料档案计划页签 |
| "要把库存锁给重要客户" | **销售订单锁库 / 预留网络** | 标准功能 |
| "想要每天凌晨自动跑 MRP" | **后台计划任务调批量计划运算服务** | BOS 后台 |
| "给我一个跑 MRP 的按钮" | **计划运算向导本身就是入口** | 标准菜单 |
| "MRP 算损耗" | **物料档案 / BOM 子项的损耗率** | 工程数据 |
| "成品和半成品分两层算" | **MPS 物料 + MPS 编制 / 核准** | 物料档案 + 主计划菜单 |

## 8. 什么时候**真的**需要二开

凭经验筛掉以上 14 类后,剩下还是要 BOS / Python 的常见场景:

- **算法变更**:净需求公式不能改(系统硬编码),客户真要换算法 → BOS 加自定义计划方案 + Python 接管运算
- **跨账套 MRP**:K/3 Cloud MRP 在单账套内,集团多账套合并需求要二开
- **需求来源不在标准列表**:比如"WMS 库存预警当需求""第三方预测系统对接" → BOS 接口 + Python 拉数 + 转预测单
- **复杂分配规则**:超出"按重要性 / 时间"两种,如按客户分级 + 利润率综合分配 → Python 计划员工作台插件
- **审核 / 投放复杂工作流**:超出审批流能力 → BOS 工作流 + Python

> 进入这五类前,**先把上面 7 个章节的字段 / 勾选项跟客户对一遍**。
