# Plan 5.10(非正式)— 段批量化文字 + 实时 token 计数 + 工具耗时显示

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to execute task-by-task.

**Goal:** 把当前"逐字流式"换成 Claude Code 风格——LLM 生成文字时不打字机,**显示 "思考中… (~N tokens)" 实时计数**;一段文字段结束(碰到 tool_call,或整轮结束)就一次性把整段 commit 渲染出来。**工具调用期间** 显示 "执行 X 中… (Ns)" 带 elapsed 计数。token 数从 provider 的原生 usage 抽精确值(Anthropic 中途吐 / OpenAI+Ollama 终态吐),没拿到时按 delta 事件数估计加 `~` 前缀。

**Architecture:**
- **共 2 个数据流改动**:① stream event 增加 `usage` 事件类型,3 个 LLM client 把 provider 的 usage 数据吐成 stream 事件(Anthropic 的 message_delta.usage / OpenAI 开 stream_options.include_usage / Ollama 终 response 的 eval_count);② chat-store 改 delta handler 不立即渲染文字,缓冲到 `pendingText` + 计数,在 tool_call / done 时 flush。
- **2 个 UI 改动**:① ToolCall 加 `startedAt` + Message 组件用 500ms ticker 渲染 elapsed;② `derivePendingActivity` 扩 3 个分支(`thinking` / `generating{tokens,exact}` / `awaiting-tool{toolName,elapsedMs}`)+ 4 个新 i18n key。
- 测试:llm-types 类型变更 + Anthropic/OpenAI/Ollama 各加一个"emits usage event" 测试 + chat-store batching 测试 + Message.tsx derive 测试扩展。

**Tech Stack:** TypeScript 6 strict / React 19 + Zustand / Vitest 4 / 现有 SSE pipeline + 3 个 LLM client。无新增依赖(无 tokenizer)。

- [ ] **Step 2: 改 derivePendingActivity 类型 + 实现**

`src/renderer/components/Message.tsx`:

```ts
export type PendingActivity =
  | { kind: 'thinking' }
  | { kind: 'generating'; tokens: number; exact: boolean }
  | { kind: 'awaiting-tool'; toolName: string; startedAt: number };

export function derivePendingActivity(message: ChatMessage): PendingActivity | null {
  if (!message.isStreaming) return null;
  // pendingText 里有内容就是正在生成文字段 → 显示 token 计数
  if (message.pendingText !== undefined && message.pendingText.length > 0) {
    return {
      kind: 'generating',
      tokens: message.pendingTokens ?? 0,
      exact: message.tokensExact === true
    };
  }
  const blocks = message.blocks ?? [];
  if (blocks.length === 0) return { kind: 'thinking' };
  const last = blocks[blocks.length - 1];
  if (last.type === 'text') return null;
  // tool_use 块
  const call = (message.toolCalls ?? []).find((c) => c.id === last.callId);
  if (call && !call.result) {
    return {
      kind: 'awaiting-tool',
      toolName: call.name,
      startedAt: call.startedAt ?? Date.now()
    };
  }
  return { kind: 'thinking' };
}
```

注意:`pendingText` / `pendingTokens` / `tokensExact` 字段需在 `ChatMessage` interface 里加上(Task 5 已做)。

Message 组件渲染处:用 `useState` + `useEffect` 跑 500ms ticker(只在有 `awaiting-tool` 或 `generating` 时启动),触发重渲染让 elapsed / tokens 实时更新:

```tsx
const [tick, setTick] = useState(0);
useEffect(() => {
  if (!message.isStreaming) return;
  const id = setInterval(() => setTick((t) => t + 1), 500);
  return () => clearInterval(id);
}, [message.isStreaming]);

// ... 在 turn-body 末尾:
const pending = derivePendingActivity(message);
if (pending) {
  let label: string;
  if (pending.kind === 'thinking') {
    label = t('messages.thinking');
  } else if (pending.kind === 'generating') {
    label = pending.exact
      ? t('messages.generatingExact', { tokens: pending.tokens })
      : t('messages.generating', { tokens: pending.tokens });
  } else {
    const seconds = Math.max(1, Math.round((Date.now() - pending.startedAt) / 1000));
    label = seconds === 0
      ? t('messages.toolRunning', { name: pending.toolName })
      : t('messages.toolRunningSeconds', { name: pending.toolName, seconds });
  }
  return <div className="turn-thinking">{label}</div>;
}
```
