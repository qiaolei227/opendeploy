# OpenDeploy MVP-0.1 · Plan 1: 项目引导与应用基础

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 从零搭建 OpenDeploy 的 Electron + React + TypeScript 桌面应用基础设施，产出一个可运行、可切换中英双语、可切换主题、可持久化配置的空壳应用，为后续 LLM 集成和金蝶功能做地基。

**Architecture:** Electron 多进程（main + preload + renderer）+ React 18 渲染层 + electron-vite 构建 + react-i18next 国际化 + Zustand 状态管理 + Vitest 测试 + 本地 JSON 持久化（`%USERPROFILE%/.opendeploy/settings.json`）。严格遵循 Electron 安全最佳实践（contextIsolation + sandbox + nodeIntegration=false）。

**Tech Stack:** Electron 28+, Node.js 20 LTS, pnpm, TypeScript 5+, React 18+, Vite 7 (peer-compat with electron-vite 5), electron-vite, react-i18next, Zustand, Radix UI + Tailwind CSS, Vitest, ESLint 9, Prettier.

## 执行过程重要决策记录

**Task 2 修复**：原 Plan 未指定 vite/eslint 版本，pnpm 解析到 vite 8 和 eslint 10，与 electron-vite 和 eslint-plugin-react 有 peer 冲突。修复：pin `vite@^7`、`eslint@^9`。

**Task 3 修复**：原 Plan 的 tsconfig 包含 `baseUrl: "."`，在 TypeScript 6.x/7+ 已弃用（会导致 typecheck 报错）。修复：移除 `baseUrl`，将 paths 改为相对路径 `./src/xxx/*`。

**`pnpm typecheck` 预期行为**：在 src/ 为空时会报 `TS18003: No inputs were found`。这是 TypeScript 的设计，不是 config 错误。Task 5+ 创建 src/ 文件后会自动消失。
